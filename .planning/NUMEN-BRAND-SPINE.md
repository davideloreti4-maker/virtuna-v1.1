## Naming layers

One logic per layer — never mix philosophies within a layer.

```
Numen Machines    company / house        (the Anthropic, the Cognition)
  └─ Numen        the intelligence        (the Claude, the Devin) — web, iOS
       ├─ audience  your personas — build & tune        ← HERO OBJECT
       ├─ skills    Hooks · … — every one returns a Read
       ├─ the Read  generate → simulate → ranked verdict + why   ← SIGNATURE MECHANIC
       └─ SIM-1     the model — Flash / Max, auto-selected, shown as badge
```

- **Numen Machines** — company. Footer, legal, "made by." Industrial/abstract is fine here.
- **Numen** — the ONE name users learn. The humanized creator intelligence you talk to. "I asked Numen / Numen read my video."
- **your audience** — user-built, finetuned personas. The thing you test *against*. Strongest brand asset: flywheel intake + switching cost + the visible cheatcode. (Word "audience" chosen over "SIMs" — intuitive, warm, zero teaching, frees SIM for the model.)
- **skills** — descriptive capabilities Numen wields (Hooks, …).
- **SIM-1** — the model/engine, **visible** as a badge in composer + a small mark on generated content. Tiers **Flash** (no-video skills + chat) / **Max** (video test), auto-selected by skill — never a user picker. Keep the `-1` (versioned, signals a real model). Badge reinforces the *simulation* = the differentiator (vs ChatGPT's "GPT-4o"). You talk to Numen; it runs on SIM-1.

## The Read — signature mechanic

Every skill, same spine:

```
GENERATE  →  SIMULATE  →  READ
(options)    (vs your SIMs)  (ranked verdict + the why)
```

General LLMs stop at GENERATE — they hand you options, you guess. Numen returns a **Read**: which wins, by how much, why the rest fall flat. Grounded in expert knowledge + what worked with your SIMs.

The signature UX and the data flywheel are the **same click**: pick winner → post → outcome returns → model + SIMs sharpen.

## Positioning

- **Category:** foresight / prediction — NOT generation. (Generation is commoditized; unwinnable vs the labs.)
- **Metaphor:** a wind tunnel for content. Test before you ship.
- **Enemy word:** slop.
- **Promise:** see how it lands before you post.
- **Tagline seed:** *"ChatGPT writes it. Numen knows if it'll land."*

## Moat

model (content-intelligence base + tool skills + live creator data) **+** SIM personas — both grounded in what actually earns attention, not slop. The **data flywheel** (building soon) is the unreplicatable accelerant: nobody else holds your creators' personas + outcomes.
