# Sketch 003: AudienceLens — Clean Language

## Why this exists
v2 (002) proved the *interaction* but the visual language was wrong: a row of coral
dots = candy, too much color, and it crammed per-person data into a thumbnail.

## The rethink
- **New IA:** inline card = aggregate signal + ONE human teaser. Per-person detail
  moves into the drill where it can breathe.
- **Monochrome base, coral once per surface** (primary number / CTA / active).
- **Faces, not abstraction** — small monochrome avatars (initials) = the room.
- **Words as texture** — one pulled verbatim beats any inline chart.
- **Hairline meters (2px), not fills.**

## How to View
```
open .planning/sketches/003-audience-lens-clean/index.html
```

## The decision: 3 inline treatments
- **A — Editorial / verbatim-led:** pulled quote + "Name + N kept watching". Most differentiated, most premium.
- **B — Avatar stack + hairline meter:** who reacted + one 2px bar + clean metric.
- **C — Quiet data:** big score + 3 calm distribution bars. Most "tool".

## Redesigned drill (shared)
- **Replay = real timeline** — avatar marks placed by reaction time, playhead sweeps, caption narrates who reacted when. (No candy scrubber.)
- **The room = readable list** — avatar + name + role + reaction label; tap a row → verbatim + why inline; "Ask [name] directly" → in-voice node chat.
- **Chat = audience-scoped, persists across cards** (subject chip changes, conversation doesn't).

## To decide
1. Which inline treatment — A, B, or C? (Hypothesis: A.)
2. Does the timeline replay feel premium / legible now?
3. Room-as-list vs the old grid — better?
4. Is coral-once-per-surface enough accent, or too austere?
