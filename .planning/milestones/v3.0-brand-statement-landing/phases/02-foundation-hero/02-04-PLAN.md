Critical implementation notes:

1. **Server component (Pitfall 6):** No `'use client'` at top. The component server-renders the section + gradient + text + buttons as static HTML. Only `BehavioralCanvas` is a client island that hydrates post-paint. This minimizes JS shipped per LCP per Phase 5 BUILD-04.

2. **DOM chip overlay (orchestrator decision #6):** The chip is a real `<div>` with semantic `role="status"`, `aria-live="off"` (since the value never changes), and an explicit `aria-label` for screen readers. The styles come from `CONFIDENCE_CHIP` constants so the value source-of-truth is the constants file. `pointer-events-none` makes it decorative (not clickable) per CONTEXT.md deferred ideas. The chip remains visible in reduced-motion mode because it's pure DOM — Canvas particles are clustered around its position in the reduced-motion path.

3. **Chip positioning:** The chip sits at `top: 45%` (calculated from `(0.5 + PARTICLE_MOTION.targetOffsetY) * 100` to match the canvas target). `left-1/2 -translate-x-1/2 -translate-y-1/2` centers it both horizontally and vertically relative to that top anchor — so the chip's center aligns precisely with where the canvas particles converge.

4. **Locked copy verbatim:** All five strings (pre-headline, H1, sub-headline, subline, primary CTA, secondary CTA) come from REQUIREMENTS.md HERO-01..05. Use `&middot;` HTML entity in the pre-headline (Pitfall 11) — literal `·` would also work but `&middot;` is more portable across editors.

5. **CTAs use existing Button + asChild composition:** Radix Slot pattern lets `<Button asChild><Link>...</Link></Button>` produce a styled anchor without breaking Next.js client-side navigation. Primary uses `variant="primary"` (coral background per Button.tsx); secondary uses `variant="secondary"` (transparent + 6% border per Button.tsx).

6. **Typography clamp() values are inline style (not Tailwind utilities):** RESEARCH.md §2 D-35 specifies `clamp(2.75rem, 6.5vw, 5rem)` for the H1 — Tailwind v4 has no built-in arbitrary-clamp utility (`text-[clamp(2.75rem,6.5vw,5rem)]` works but is harder to read; inline `style={{ fontSize: "..." }}` is clearer for these one-off oversized values). The `font-light`, `font-medium`, `font-normal` Tailwind classes still apply the weight; `font-sans` ensures Inter (via `--font-sans` token).

7. **text-wrap: balance + explicit `<br />` (D-39 / Pitfall 8):** `textWrap: "balance"` is supported by 89% of browsers as of May 2026. The `<br />` enforces the locked two-line break in the 11% without support. Both work together (CSS WG spec confirms balance respects author breaks).

8. **No external imports:** Imports are only `next/link`, `@/components/ui` (Button), `@/lib/utils` (cn), `./BehavioralCanvas`, `./behavioral-hero-constants`. Matches Plan 03's policy `phase2_policy` allowed sources list.

9. **NO `'use client'`, NO `'framer-motion'`, NO `motion/react` imports:** Server component, no motion library needed (canvas drives all motion).

10. **NO banned strings:** "viral", "AI" (whole-word), "go viral", "framer-motion" do NOT appear in the file. Run `pnpm lint:vocab` to confirm.
