### Integration Issues (Non-blocking)

1. **GlassCard naming collision** — `ui/card.tsx` and `primitives/GlassCard.tsx` both export `GlassCard` with different implementations (3 vs 7 blur levels). Showcase uses both in different contexts without conflict, but naming is confusing.

2. **Blur implementation inconsistency** — Phase 40 GlassCard uses inline blur mapping, Phase 42 GlassPanel uses CSS classes from globals.css. Both work but divergent patterns.
