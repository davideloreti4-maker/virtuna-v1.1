<objective>
Stand up the **generalized thread/message persistence** — the substrate every studio thread saves to and re-hydrates from. Add a new `threads` table (`type` discriminator + nullable `reading_id` UUID FK; the Reading stays the untouched artifact, the thread WRAPS it — D-12), a `messages` table (one atomic row per message, body = typed-blocks JSONB array — D-13), RLS mirroring the `analysis_chats` pattern, and the helper layer (lazy idempotent grounded-thread creation D-15, insert/load with rehydration re-validation D-14). Then **apply the migration to the live DB and regenerate types** so types come from the database, not config.

Purpose: THREAD-07 persistence is the floor under every later tool's output; the Flash output (Plan 03) and any thread message persist through here. The migration is intentionally additive — `analysis_results` is byte-untouched (protects the regression-gated Max path, D-12).
Output: the migration, the applied schema, regenerated `database.types.ts`, and `src/lib/threads/` helpers.
</objective>
