<objective>
Build the **SIM-1 Flash text-mode** engine path (ENGINE-01) — the personas react to TEXT (no video). It is a new, ISOLATED fork of the proven `runFold` pattern that NEVER imports into `pipeline.ts` / `aggregator.ts` (the protected Max path). One bounded Qwen `json_object` call returns all 10 archetypes, each emitting a **stop/scroll verdict + a one-line first-person quote** (D-03); a pure deterministic aggregator rolls the verdicts into a **band + audience-fraction** — qualitative only, never a fabricated number (D-02/D-11, the engine-side honesty spine ENGINE-03). Mode framing (hook/idea/chat) is a call parameter (D-04). The result maps to renderer-ready `band` + `personas` blocks (Plan 01) tagged `sim1-flash`, exposed as a `ToolRunner`.

Purpose: this is the engine the studio's generators (Ideas P3, Hooks P4) will call; built here, calibrated here (ENGINE-01's "winning framing" spec). Isolation is the hard constraint — text-mode is additive, so `ENGINE_VERSION` does NOT bump and the Max regression gate stays green.
Output: `src/lib/engine/flash/` (schema, prompts, aggregate, runner) + the Flash `ToolRunner`.
</objective>
