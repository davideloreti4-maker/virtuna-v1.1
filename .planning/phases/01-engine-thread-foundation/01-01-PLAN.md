<objective>
Author the **renderer registry (SSOT)** and the **tool-runner contract** — the shared spine the whole studio renders through. One module maps `blockType → {component, schema}`; the tool-runner is the single contract every tool (`{promptTemplate, knowledgeBundle, outputSchema, renderer}`) flows through. Structured output validates against the *same* registry the message renderer re-validates on rehydration. This is the **structural** enforcement of THREAD-04 "no model-generated UI" (D-14): the model can only emit block `type`s already in the registry; everything else degrades to a static `<UnsupportedBlock>`.

Purpose: Plans 03 (Flash engine) and 04 (composer) plug into these contracts; building them first hands executors the contracts instead of forcing a scavenger hunt. This is the cross-cutting seam CONTEXT.md mandates authoring once.
Output: `src/lib/tools/` (blocks, registry, tool-runner) + `src/components/thread/` (block host + the three block renderers + unsupported placeholder).
</objective>
