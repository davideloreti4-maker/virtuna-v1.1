## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering / Information disclosure | `parent_id` request field on POST /api/analyze | mitigate | parent_id is a STORED pointer only — it is never used this plan to READ another row's data. It is validated as `z.string().optional()` by AnalysisInputSchema (input bounded to a string). The child row is owned by the submitting `user.id` (existing auth). The chip-side read of the parent (05-03) re-checks `user_id` ownership on `/api/analysis/[id]?summary`, so a forged parent_id cannot leak another user's data even if persisted. |
| T-05-02 | Integrity | parent_id FK on analysis_results | mitigate | FK `REFERENCES analysis_results(id) ON DELETE SET NULL` — a parent_id that does not match a real row is rejected by the DB at insert; deleting a source nulls the child pointer instead of cascading. Referential integrity is DB-enforced (D-08). |
