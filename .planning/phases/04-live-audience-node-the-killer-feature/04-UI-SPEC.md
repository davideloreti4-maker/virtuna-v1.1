## Spacing Scale

Scale: 4px base unit. All values multiples of 4.

| Token | Value | Phase 4 Usage |
|-------|-------|---------------|
| `--spacing-1` | 4px | Chip icon-gap, heatmap cell inner padding, marker superscript offset |
| `--spacing-2` | 8px | Chip horizontal padding, row label padding, popover internal gap |
| `--spacing-3` | 12px | Heatmap cell height, filmstrip segment gap — **see Exception A** |
| `--spacing-4` | 16px | Section padding inside AudienceNode, chip row gap, drawer section padding |
| `--spacing-5` | 20px | Popover padding, inspector section padding — **see Exception B** |
| `--spacing-6` | 24px | AudienceNode internal section gap (chips → filmstrip → curve → drawer) |
| `--spacing-8` | 32px | Drawer header height, weight slider row height |
| `--spacing-12` | 48px | Inspector panel padding (desktop right-side) |
| `--spacing-16` | 64px | Bottom sheet header height (mobile) |

**Standard exceptions (off-scale, required for legibility and touch):**

- **Touch targets** (tap cells, row labels, dropoff markers): minimum 44px hit area regardless of visual size (marker dot is 8px visual, 44px tap area via transparent padding)
- **Heatmap cell minimum height:** 32px on mobile portrait (ensures tap accuracy)
- **Weight slider:** 4px track, 20px thumb (44px hit area with padding)

**Off-scale values — explicit justifications required:**

- **Exception A — `--spacing-3` (12px) as heatmap cell height minimum on desktop:** 12px is the minimum row height that keeps the coral-intensity gradient readable at desktop density (10 rows visible simultaneously). Stepping up to 16px (`--spacing-4`) would push the full heatmap grid height above the visible viewport on 13" screens without scroll. Justified by legibility at data density — CONTEXT.md O-2 + NF2. _12px is a visual floor, not a tap target; the tap area is always padded to 32px on mobile and 44px minimum via transparent padding._
- **Exception B — `--spacing-5` (20px) as popover and inspector section padding:** TapPopover and PersonaInspector use 20px interior padding to meet the thumb affordance standard on mobile (touch region needs breathing room around interactive elements within the panel). Stepping down to 16px (`--spacing-4`) collapses touch clearance below WCAG 2.5.8 for tappable items near the popover border. Justified by touch ergonomics on the 44px hit-area standard — CONTEXT.md D-13, D-04.

## Color

### 60/30/10 Split

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#07080a` | `--color-background` | Board body, canvas background, behind heatmap grid |
| Secondary (30%) | `#18191a` | `--color-surface` | AudienceNode body, GlassPanel backgrounds, inspector panel, drawer backgrounds |
| Elevated surface | `#222326` | `--color-surface-elevated` | Popovers, dropdown menus, weight override drawer inner |
| Accent (10%) | `#FF7F50` | `--color-coral-500` | **See reserved list below** |
| Anti-virality / warning | `oklch(0.75 0.15 85)` | `--color-warning` | Orange band on curve in affected segments only — QUIET treatment |
| Error / semantic negative | `#E05252` | `--color-error` | vs-Niche negative badge; destructive action confirmation state — **see Destructive Actions below** |

**`--color-error` WCAG AA verification:** `#E05252` on `#18191a` (node surface) → contrast ratio 4.8:1, passes AA (4.5:1 threshold). Hex used, not oklch, because node surface L < 0.15 (Tailwind v4 oklch caveat applies).

### Accent Reserved-For List (exclusive — use coral nowhere else)

Coral `#FF7F50` is reserved for:
1. Retention curve line (2px stroke)
2. Retention curve gradient fill (coral/15 → transparent)
3. Hook zone warm band behind curve (coral/8)
4. Heatmap cell fill intensity scale (coral/5 → coral/80, proportional to attention value)
5. Dropoff marker dots (8px solid fill)
6. Filmstrip placeholder band background (coral/20 tint)
7. "Show personas ▾" affordance text + chevron when hovered/active
8. Weights badge text ("Weighted: 65/20/10/5")
9. Anti-virality fix-chip border (NOT fill — coral border, dark fill)
10. Focus rings (`focus-visible:ring-accent`)

Coral is NOT used for: heatmap cells in anti-virality state (they stay coral-intensity, orange is only on the curve band), row labels, inspector body text, slider tracks.

### Archetype Ring Colors (dropoff markers, within coral palette)

| Archetype | Ring Color | Value |
|-----------|-----------|-------|
| FYP non-followers | coral-500 | `#FF7F50` (warm coral) |
| Niche-aligned | coral-300 | `oklch(0.87 0.10 40)` |
| Loyalist | coral-100 | `oklch(0.97 0.03 40)` |
| Cross-niche | white/30 | `rgba(255,255,255,0.30)` |

### Anti-Virality Color Behavior

- **Timeline-pattern trigger:** Curve gradient fill transitions coral → `--color-warning` (`oklch(0.75 0.15 85)`) ONLY within the dropoff range band on x-axis. ⚠ tag below filmstrip in affected segments uses `--color-warning` fill.
- **Confidence-gate only (no timeline pattern):** Subtle `--color-warning` border on AudienceNode outer frame + header label change. No per-segment orange.
- **Heatmap cells:** ALWAYS coral-intensity regardless of anti-virality state. Never tinted orange.

### Destructive Actions

Phase 4 has two destructive actions: "Reset to defaults" (wipes custom slider weights) and "Discard Changes" (closes drawer without applying edits).

**Button color rule:** Both destructive buttons use **neutral styling by default** — `bg-transparent`, `--color-foreground-muted` text, `rgba(255,255,255,0.06)` border. On hover, text transitions to `--color-error` (`#E05252`) with no background change. Rationale: these actions occur within a routine creative flow (iterating on audience mixes); alarming red buttons would interrupt flow for a reversible operation. The hover-only error color provides a clear signal without anxiety.

**"Reset to defaults" confirmation pattern:** On tap, fires an inline toast at the bottom of the WeightOverrideDrawer: `"Weights reset · Undo"` — with an [Undo] text link (coral, 14px, 400 weight). The toast auto-dismisses after 4s. If the user taps [Undo] within 4s, sliders return to their pre-reset values. No modal dialog — the action is reversible; a modal would be disproportionate.

### Borders

- Default: `rgba(255,255,255,0.06)` (`--color-border`)
- Hover: `rgba(255,255,255,0.10)` (`--color-border-hover`)
- Glass: `rgba(255,255,255,0.06)` (`--color-border-glass`)

## Component Contracts

### AudienceNode Shell (plan 4.1)

**Pattern:** Extends `NodeOverlay.tsx` — DOM overlay fills the Audience group container frame (Konva handles spatial position + frame border, DOM overlay handles all content).

**Desktop layout (vertical stack, D-01):**
```
┌─ AudienceNode (NodeOverlay wraps GlassPanel) ──────────────────┐
│ HeadlineChips (plan 4.2)                     [WeightsBadge]   │
│ Filmstrip (plan 4.3)                                           │
│ RetentionCurve + DropoffMarkers (plans 4.4, 4.5)              │
│ [AntiViralityOverlay chips if triggered] (plan 4.10)           │
│ HeatmapDrawer affordance (plan 4.6 — collapsed default)        │
└────────────────────────────────────────────────────────────────┘
```

**Sizing:** Height = dynamic (grows when heatmap drawer opens). Min-height 320px on desktop, 240px on mobile portrait.

**Background:** GlassPanel (`linear-gradient(137deg, rgba(17,18,20,0.75) 4.87%, rgba(12,13,15,0.9) 75.88%)`, `blur(5px)` via inline style, 12px radius, `rgba(255,255,255,0.06)` border, `rgba(255,255,255,0.15) 0 1px 1px 0 inset` shadow).

**States:**

| State | Visual |
|-------|--------|
| `idle` | Skeleton shimmer, all layers greyed |
| `streaming` | Skeleton rows visible, cells fill left→right as Pass 2 streams |
| `complete` | Full render — chips, filmstrip, curve, markers, drawer affordance |
| `anti-virality` | Orange band on curve, ⚠ tags below filmstrip, node header/border warning treatment |
| `reduced-motion` | Jump-cut to `complete` state — no L→R wave, no curve morph, opacity transitions only |
| `low-perf-tier` | Skip curve morph, skip L→R wave entirely — jump to final values |
| `medium-perf-tier` | Halve L→R wave duration to 90ms, retain curve morph |

**ARIA:**
```html
<section aria-label="Audience analysis" aria-live="polite" aria-busy={isStreaming}>
```
Announces: "Audience reactions arriving" (streaming start), "Audience analysis complete" (complete), "Anti-virality pattern detected at segments X–Y" (if triggered).

### HeadlineChips (plan 4.2)

**Content (D-02):** 5 chips + 1 weights badge.

| Chip | Data source | Format |
|------|------------|--------|
| Avg watch % | `weighted_completion_pct` (Pass 2 ready) else `persona_behavioral_aggregate.completion_pct` | `{N}%` |
| Loop % | `weighted_loop_pct` else `persona_behavioral_aggregate.loop_rate` | `{N}%` |
| Top dropoff | `weighted_top_dropoff_t` | `{N}s` |
| Hook score | `weighted_hook_score` | `{N}/3` |
| vs Niche | `weighted_completion_pct - niche_baseline_pct` | `+{N}%` or `−{N}%` (diff badge) |

**Weights badge (D-19):** Positioned inline at chip-row end (right-aligned). Tappable — opens WeightOverrideDrawer. Copy: `"Weighted: 65/20/10/5"` (actual values). Small coral text, 12px, weight 400.

**Chip anatomy:**
- Background: `rgba(255,255,255,0.05)` (card pattern)
- Border: `rgba(255,255,255,0.06)` 1px
- Radius: 8px (`--radius-md`)
- Padding: 4px 8px (`--spacing-1` `--spacing-2`)
- Value: 14px semibold, `--color-foreground`
- Label: 12px regular, `--color-foreground-muted`
- vs-Niche badge: positive = `--color-success`, negative = `--color-error` (`#E05252`) — semantic tokens only; `--color-warning` is reserved for anti-virality orange, not vs-Niche negative

**Skeleton state:** 5 grey shimmer blocks, same dimensions as chips, `animate-pulse`.

**ARIA:** `<dl>` list pattern — chip is `<div role="term">` + value is `<div role="definition">`.

### Filmstrip (plan 4.3)

**Default state:** Coral-tinted segment band (`coral/20` background) + `heatmap.segments[].label` text (12px, `--color-foreground-secondary`). Visually occupies same horizontal width as curve below.

**Loaded state:** `<img>` swaps in when `filmstrip_segment_ready` SSE event delivers `keyframe_uri`. Swap animation: 150ms opacity cross-fade from band → image.

**Layout:** Horizontal flex row. Each segment cell width = proportional to `(t_end - t_start) / total_duration * container_width`. Minimum cell width: 48px (merge sub-1s scenes at display time per R2.6).

**Cell sizing:** Height fixed 48px on desktop, 36px on mobile portrait.

**Cell border:** `rgba(255,255,255,0.06)` between cells (1px right border on all but last).

**No separate x-axis timestamp labels** — filmstrip IS the x-axis (D-09). Timestamps visible on cell tap only (popover).

**Anti-virality annotation area:** 20px strip BELOW filmstrip. In `anti-virality` state, affected segments get a small chip: `"seg N: [brief fix summary]"` — 11px, `--color-warning` border, `--color-surface-elevated` fill, 4px radius. Tap → TapPopover with full fix text + counterfactual reference.

**Scrollable on long videos:** Horizontal scroll when content overflows. Single-finger scroll (touch), scroll wheel (desktop). No scrollbar shown (overflow hidden, scroll via gesture).

**ARIA:** `<figure aria-label="Video keyframe filmstrip">`. Each cell: `aria-label="Segment N: {label}, {t_start}s to {t_end}s"` (from `heatmap.segments[].label`).

### RetentionCurve (plan 4.4)

**Rendering:** Dedicated `<canvas>` element in DOM overlay. NOT Konva (D-10, O-1).

**DPR-aware setup:**
```ts
const dpr = window.devicePixelRatio ?? 1;
canvas.width = containerWidth * dpr;
canvas.height = containerHeight * dpr;
ctx.scale(dpr, dpr);
canvas.style.width = containerWidth + 'px';
canvas.style.height = containerHeight + 'px';
```

**Interpolation:** Catmull-Rom α=0.5 (hand-rolled, no d3). Bias: hand-rolled unless catmull-rom requires >60 LOC, then import `d3-shape` for `curveCatmullRom`.

**Curve style (D-08):**
- Line: 2px stroke, `#FF7F50` (coral-500 hex — no oklch in canvas ctx)
- Gradient fill: `createLinearGradient` from curve top → bottom, stops: `rgba(255,127,80,0.15)` at top → `rgba(255,127,80,0)` at bottom
- Hook zone (0–3s): `rgba(255,127,80,0.08)` rect fill behind curve, same 0–3s x range
- Grid lines: 4 horizontal lines at 25/50/75/100% y — `rgba(255,255,255,0.04)` 1px
- Y-axis labels: `0`, `25`, `50`, `75`, `100` — 10px monospace, `rgba(255,255,255,0.3)`, left-aligned with 4px padding
- No x-axis labels (filmstrip handles temporal axis, D-09)

**Morph animation (D-07, O-3):**
1. Pass 1 baseline renders on `complete` SSE event (flat curve from `persona_behavioral_aggregate.completion_pct`). Duration: 300ms ease-out-cubic.
2. Single 800ms morph (`--ease-out-cubic`) when `weighted_curve` arrives (≥7-of-10 Pass 2 complete). All dropoff markers fade in during this 800ms window.
3. RAF loop drives morph: interpolate each data point `current[i] = lerp(baseline[i], target[i], progress)`.

**Reduced-motion:** Jump-cut to final `weighted_curve` state. No morph animation. Opacity transition on markers: 150ms.

**Low perf tier:** Skip morph entirely — jump to weighted curve at `complete`.

**Canvas tap handling:** `pointermove` / `pointerdown` events on canvas → calculate nearest data point → trigger TapPopover at pointer position.

**ARIA:** Canvas is `role="img" aria-label="Audience retention curve. Weighted average: {weighted_completion_pct}%"`. Screen reader does not read canvas internals; headline chips provide numeric values.

### DropoffMarkers (plan 4.5)

**Rendered inside the RetentionCurve canvas** (not separate DOM elements).

**Marker anatomy (D-11):**
- Dot: 8px diameter, solid `#FF7F50` fill
- Ring: 2px stroke, color per archetype (see Color §Archetype Ring Colors)
- Hit area: 44px × 44px transparent pointer region centered on dot

**Cluster collapse (D-12):**
- Collapse trigger: ≥3 markers within 12px OR ≥2 markers within 6px
- Collapsed state: single coral dot (10px diameter) + superscript counter badge in top-right (8px, semibold, `--color-surface` bg, `--color-accent` text)
- Tap cluster → TapPopover with list of all collapsed personas (mini-rows with archetype tag + attention %)

**Fade-in timing (D-07):** All markers fade in during the 800ms curve morph window. Each marker opacity: `easeOutCubic(progress * 1.2)` clamped to 1, creating staggered natural feel.

**Reduced-motion:** Instant opacity 1 on `complete` (no fade-in animation).

### HeatmapDrawer (plan 4.6)

**Default state:** Closed. Only "Show personas ▾" affordance visible — 16px semibold text + chevron icon, coral on hover (D-03).

**Desktop expand:** Audience node grows vertically — inline CSS Grid expands below curve with `grid-template-rows: 0fr → 1fr` transition, 250ms `--ease-out-cubic`.

**Mobile portrait:** Affordance tap → bottom sheet opens (NOT inline expansion, O-6). Bottom sheet contains full 10-row heatmap grid. Affordance stays visible; sheet dismiss closes the grid.

**Grid layout:**
- CSS Grid: 10 rows (persona rows) × N columns (segment columns, N = `heatmap.segments.length`)
- Row height: 32px (mobile min) / 36px (desktop)
- Column width: proportional to segment duration, same formula as filmstrip
- Cell fill: `rgba(255,127,80, attention_value)` where `attention_value ∈ [0.05, 0.80]` — clamp to avoid invisible-at-0 / saturated-at-1

**Row reveal animation (D-05, D-06):**
- At `wave_0_complete`, skeleton rows pre-render in fixed archetype order: FYP×6, niche×2, loyalist×1, cross-niche×1
- Skeleton cells: `--color-surface-elevated` with `animate-pulse`
- When persona's Pass 2 starts (`pass2_persona_start`): row label becomes readable (opacity: 0.4 → 1, 150ms)
- When persona's attentions stream (`partial.personas[i].attentions[j]`): cells fill left→right, 18ms stagger per cell, 180ms total wave per row (D-06)
- When persona's Pass 2 ends (`pass2_persona_end`): cells reach full saturation
- Swipe marker appears on curve simultaneously with last cell fill (sync point)

**Reduced-motion row fill:** Cells appear at full saturation instantly on `pass2_persona_end`. No L→R wave. 150ms opacity transition only.

**Medium perf tier:** L→R wave duration reduced from 180ms → 90ms.

**Low perf tier:** Skip wave — jump to full saturation on `pass2_persona_end`.

**Cell interactivity:** Tap → TapPopover (D-13). Focus via keyboard: Tab navigates rows, Arrow keys navigate cells, Enter/Space opens TapPopover.

**Row label:** 12px regular, `--color-foreground-secondary`. Tap → PersonaInspector opens.

**Swipe markers:** White vertical tick marks (1px wide, full row height) at `swipe_predicted_at` column. `rgba(255,255,255,0.70)` color.

**Hook zone column highlight:** Columns where `is_hook_zone = true` get `rgba(255,127,80,0.04)` background behind ALL cells in that column.

**Color-blind mode:** CSS class swap on `<body>` or `<HeatmapDrawer>`. Pattern: diagonal stripes (45° hatch, 2px stroke, `rgba(255,255,255,0.15)`) overlaid on cell fill, density proportional to attention value. Activated via `prefers-color-scheme` query or settings toggle. Tested against deuteranopia + protanopia simulators.

**ARIA:**
- Grid: `role="grid" aria-label="Persona attention heatmap" aria-rowcount="10" aria-colcount="{N}"`
- Row: `role="row" aria-label="{persona_name}, {archetype}"`
- Cell: `role="gridcell" aria-label="{attention_pct}% attention at segment {N}, {t_start}s to {t_end}s{reason if present}"`
- Swipe marker: `aria-label="Predicted swipe at {t}s"` on adjacent cell

### TapPopover (plan 4.8)

**Trigger sources:** Heatmap cell tap, dropoff marker tap, curve point tap (non-marker).

**Positioning:** Appears at tap/click point, auto-repositions if near viewport edge (collision detection: if within 16px of edge, flip side). GlassPanel background.

**Content variants:**

| Trigger | Popover content |
|---------|----------------|
| Heatmap cell | Archetype tag (12px, colored per archetype), persona name (14px semibold), timestamp range, attention % (14px bold), 1-line `segment_reasons[idx]` if present, "See full →" link |
| Dropoff marker | Same as cell but for `swipe_predicted_at` context. "Swipe at {t}s" header |
| Curve point (aggregate) | Weighted attention % at t, list of contributing personas with their attention at t (3 shown, "N more" if overflow), filmstrip frame glows for 600ms (film frame border: 2px coral) |
| Cluster marker | List of all collapsed personas, each row: archetype ring dot + name + archetype + attention at swipe |

**Dismiss:** Tap outside, Escape key, scroll >40px.

**Animation:** 150ms fade-in + 4px translateY up (`--ease-out-cubic`). Reduced-motion: instant show, no translate.

**Surface:** GlassPanel — `blur(5px)`, 12px radius, `rgba(255,255,255,0.06)` border, `rgba(255,255,255,0.15) 0 1px 1px inset`. Background explicitly via `style={{ backdropFilter: 'blur(5px)' }}` (Lightning CSS workaround).

**ARIA:** `role="tooltip" aria-live="polite"`. Focus managed: popover receives focus on open if triggered by keyboard.

### PersonaInspector (plan 4.8 / 4.9)

**Trigger:** Tap heatmap row label OR "See full →" link in TapPopover.

**Desktop:** Right-side panel, 360px wide, slides in from right (300ms `--ease-out-cubic`). Board canvas remains interactive. Z-index: `--z-modal` (400).

**Mobile portrait:** Bottom sheet — slides up from bottom (300ms `--ease-out-cubic`), handles drag-to-dismiss. Fixed top handle (32px × 4px, `rgba(255,255,255,0.20)` pill), max-height 85dvh, scrollable body.

**Content:**
1. Persona header: archetype tag (coral border, dark fill, 12px), persona name (16px semibold), audience-share weight label (12px muted)
2. Pass 1 summary: verdict + confidence (14px regular)
3. Swipe prediction: "Predicted swipe at {t}s — {reason}" (14px regular)
4. All `segment_reasons` (inflection points only): rendered as timeline list — segment timestamp chip + reason text (14px, 1.5 line-height)

**Mobile:** Bottom sheet, max-height 70dvh.

**Drawer anatomy:**

┌─ Audience Mix ─────────────────────────────── [×] ┐

**Close button:** X button top-right with `aria-label="Close audience mix override"` (declared on the `<button>` element, not on the dialog).

**Preset chips (D-18):**

| Preset | FYP | Niche | Loyalist | Cross-niche |
|--------|-----|-------|----------|-------------|
| Default mix | 65% | 20% | 10% | 5% |
| Established creator | 40% | 20% | 30% | 10% |
| Niche-heavy | 30% | 55% | 10% | 5% |
| New creator | 75% | 15% | 5% | 5% |

"Custom…" chip shows slider panel (always shown if no preset matches current values).

**Sliders:** Range inputs. Track: 4px height, `rgba(255,255,255,0.10)` background, coral fill for active portion. Thumb: 20px × 20px, `#FF7F50`, 9999px radius (pill). 44px hit area via padding.

**Auto-renormalize (D-18):** Proportional. When slider A changes by Δ, distribute −Δ across the other 3 sliders proportionally to their current values. Lock-pin gesture (hold modifier) not shipped in Phase 4.

**Client-side recompute (O-5, D-20):** 16ms RAF debounce on slider drag. Recomputes:
- `weighted_curve[j] = Σ (weights[persona_type] × personas[i].attentions[j])` (mirrors `weighted-aggregator.ts` server logic — import `lib/engine/persona-weights.ts` client-safe helper)
- All 5 headline chip values
- Anti-virality re-evaluation (`isTimelinePatternTriggered()` — client-safe version)
- Curve morph: 300ms ease-out-cubic to new weighted curve values during drag

**Persistence (D-21):**
- "Apply Audience Mix" button: writes override to `analyses.analysis_override` JSON column
- "Save as my default" checkbox (unchecked by default): writes to `creator_persona_weights` table on Apply
- "Reset to defaults" link: restores to engine default (65/20/10/5), clears draft state, shows inline undo toast (see Color §Destructive Actions)
- Drawer close without Apply: discards draft (inline confirmation if values changed)

**Confirmation on discard:** If slider values changed and user closes without Apply, inline toast at bottom of drawer: "Apply changes before closing?" with [Apply Audience Mix] [Discard Changes] actions. Not a modal. Both actions are verb+noun pairs.

**ARIA:** `role="dialog" aria-modal="true" aria-label="Audience weight override"`. Each slider: `aria-label="{archetype} weight" aria-valuemin="0" aria-valuemax="100" aria-valuenow="{value}"`. Sum display: `aria-live="polite"`.

## Motion Budget

All animations must respect performance tier. Budget per Phase 4 component:

| Component | Animation | Duration | Easing | Perf guard |
|-----------|-----------|----------|--------|------------|
| Curve morph | RAF loop lerp, all segments | 800ms | `--ease-out-cubic` | Skip on Low tier |
| Curve baseline | RAF loop lerp | 300ms | `--ease-out-cubic` | Always |
| Cell fill wave (per row) | Staggered opacity 0→1 | 180ms (18ms/cell stagger) | linear | Halve on Medium (90ms), skip on Low |
| Marker fade-in | Canvas opacity 0→1 | 800ms (during morph) | `easeOutCubic(t*1.2)` | 150ms on reduced-motion |
| Heatmap drawer open | `grid-template-rows` transition | 250ms | `--ease-out-cubic` | Skip on Low |
| Inspector/drawer slide | `translateX` or `translateY` | 300ms | `--ease-out-cubic` | 150ms opacity on reduced-motion |
| Popover appear | opacity + 4px translateY | 150ms | `--ease-out-cubic` | Skip translate on reduced-motion |
| Filmstrip keyframe swap | cross-fade opacity | 150ms | linear | Instant on reduced-motion |
| Anti-virality enter | curve band opacity + color shift | 400ms | `--ease-out-cubic` | 150ms on reduced-motion |
| Anti-virality exit | reverse | 200ms | `--ease-in-out` | 150ms on reduced-motion |
| Weight slider recompute | RAF debounced curve update | 16ms (RAF) | immediate | Always — this is the killer UX |

**FPS targets (NF1):**
- High tier (iPhone 13+, modern desktop): 60fps — all animations
- Medium tier (iPhone 11-12): 45fps — cell wave halved, no parallax
- Low tier (older/throttled): 30fps — morph skipped, wave skipped, no parallax

**RAF strategy:** Curve canvas uses single RAF loop (shared between morph, marker fade, and weight recompute). Cancel RAF on component unmount.
