| Role | Value | Usage |

**Accent (`#FF7F50`) reserved exclusively for:**
- "Continue" / "Save Profile" primary CTA buttons (`variant="primary"`)
- Active progress dot on the step indicator (current card position)
- Focus ring on inputs (`focus:ring-accent/50`)

**Accent must NOT appear on:**
- Card option tiles (selected state uses `bg-white/[0.08]` border, not coral fill)
- Multi-select chips (selected state: `border-white/[0.12]` + `bg-white/[0.06]`)
- The truthfulness callout box
- Skip links / secondary actions

**Card background inside modal:** `bg-transparent` with `border border-white/[0.06]` + `border-radius: 12px` + inset shadow `rgba(255,255,255,0.05) 0 1px 0 0 inset` — matches existing `card.tsx` pattern.

**Modal overlay:** `bg-black/60` + `blur(4px)` — matches existing `DialogOverlay`.

## Interaction Contracts

### 9-Card Interview Modal

**Trigger:** User clicks the upload button in `video-upload.tsx` and has no `creator_profiles` row. Modal intercepts upload action — upload does NOT proceed until modal is dismissed (completed or fully skipped).

**Dismissal rules:**
- Clicking the overlay backdrop does NOT close the modal (mandatory flow)
- The X/close button is NOT shown (no accidental exit)
- "Skip this question" skips the current card's data but advances to the next card
- "Skip interview" (ghost text link in footer) skips all remaining cards and closes the modal — creates a `creator_profiles` row with only the completed-so-far fields populated
- After card 8 (last), the primary CTA changes from "Continue" to "Save Profile" and submitting closes the modal and resumes the upload

**Card navigation:**
- Forward: primary CTA "Continue" (coral, full-width, lg size)
- Back: secondary button "Back" (transparent border, left-aligned) — only shown on card 1+; card 0 has no back button
- Skip current card: ghost text link "Skip this question" — 12px, `text-foreground-muted`, centered below footer buttons
- Skip all: ghost text link "I'll do this later" — same styling as skip, only shown on card 0

**Progress indicator:** 9 dots in a horizontal row, centered above the card heading. Active dot: coral (`bg-accent`, 8px diameter). Completed dot: `bg-white/[0.3]` (6px diameter). Upcoming dot: `bg-white/[0.12]` (6px diameter). Transition: 150ms ease.

**Card size:** `DialogContent size="lg"` (max-w-lg, 512px). Height is auto-fit to content — no fixed height. Cards with more content will be taller; the modal does NOT scroll internally. On mobile: full-width with 16px horizontal margin.

**Keyboard:** Tab through form elements, Enter submits current card (same as Continue), Escape does NOT close (mandatory flow — suppress default Radix close-on-escape behavior).

### Card 0 — Platform Picker

- Three tiles in a row: TikTok, Instagram Reels, YouTube Shorts
- Each tile: 100% width / 3-column grid, 80px height, icon (24px) centered above label (14px), border `border-white/[0.06]`, radius 12px
- Selected: `bg-white/[0.08]` + `border-white/[0.12]`
- At least one must be selected to enable "Continue" (validate on attempt, not on blur)
- TruthfulnessCallout renders below tiles on Card 0

### Card 1 — Niche (Hierarchical)

- Level 1: grid of primary niche tiles (same tile pattern as Card 0)
- Selecting primary reveals level 2 sub-niches (animated slide-down, 200ms ease-out)
- Selecting sub-niche reveals level 3 micro-niche (optional)
- Breadcrumb trail above tiles: "Fitness > Strength Training" in 12px `text-foreground-muted`
- Skippable — no validation required

### Card 2 — Target Audience

- Age range: `Select` component, options: 13-17, 18-24, 25-34, 35-44, 45+
- Gender skew: three-button toggle (Female-skewed / Balanced / Male-skewed)
- Geo: `Input` text field, placeholder "Country or region (e.g. United States)"
- Language: `Input` text field, placeholder "Primary language (e.g. English)"
- All fields optional — card is always skippable

### Card 3 — Goal + Stage

- Goal: 4 option tiles in 2×2 grid (Growth, Engagement, Brand Deals, Conversion)
- Stage: 3 option tiles in a row (New creator, Growing, Established)
- Both sections have labels above the tile group
- No minimum selection required (skippable)

### Card 4 — Content Style

- 6 style tiles in 2×3 grid (Talking Head, B-Roll, Educational, Comedy, Tutorial, Vlog)
- Below tiles: "Cuts per second preference" with a 3-option toggle (Slow <1/s, Medium 1-3/s, Fast 3+/s)
- No minimum selection required (skippable)

### Card 5 — Reference Creators

- Up to 3 URL inputs with an "+ Add creator" ghost button below
- Each input row: URL InputField + remove button on the right
- Remove button: ghost icon (X), `aria-label="Remove creator {index + 1}"` (e.g. "Remove creator 1", "Remove creator 2", "Remove creator 3")
- Placeholder: "@handle or TikTok profile URL"
- No validation on URL format — any string accepted
- Skippable

### Card 6 — Past Wins + Flops

- Two column layout (Past wins / Past flops), each column with up to 2 URL inputs
- Column heading: "Past wins" + "Past flops" in 14px medium weight
- Same input pattern as Card 5
- Each input row: URL InputField + remove button on the right
- Remove button (wins column): ghost icon (X), `aria-label="Remove win {index + 1}"` (e.g. "Remove win 1", "Remove win 2")
- Remove button (flops column): ghost icon (X), `aria-label="Remove flop {index + 1}"` (e.g. "Remove flop 1", "Remove flop 2")
- Placeholder: "TikTok video URL"
- TruthfulnessCallout renders above both columns on this card (second appearance of callout)
- Skippable

### Card 7 — Posting Cadence

- Frequency: `Select` component (Daily, 3-4×/week, 1-2×/week, A few times/month, Rarely)
- Time-of-day awareness: toggle switch (label: "I pay attention to optimal posting times")
- Skippable

### Card 8 — Pain Points

- Single `Textarea` (4 rows), placeholder: "What's your biggest challenge as a creator right now?"
- Character count display bottom-right: "0 / 500" in 12px `text-foreground-muted`
- Max 500 characters enforced
- Skippable
- Primary CTA on this card is "Save Profile" (not "Continue")

### Settings `/settings/profile` Route

- Renders all 9 card inputs as a single scrollable form (no wizard stepping)
- Section headers per card: same heading text as modal card titles
- "Save changes" primary CTA button at bottom, full-width on mobile
- Toast confirmation on success: "Profile updated" (uses existing `toast.tsx`)
- No re-prompt for mandatory flow — all fields always optional in edit mode
- Re-prompt micro-card (PROFILE-16, 10 analyses) is NOT in scope for Phase 2's UI — it is backend-triggered and will surface as a Toast + small inline card above the upload trigger; deferred to Phase 11 integration

## States

Each card must handle these states:

| State | Treatment |
|-------|-----------|
| Empty (no input yet) | Neutral tile borders, placeholder text in inputs, Continue CTA enabled (cards are skippable) |
| Filled (has value) | Selected tiles show `bg-white/[0.08]` + brighter border; inputs show value |
| Saving (submit in progress) | Continue / Save Profile button shows `loading={true}` spinner, disabled |
| Error (Card 0 validation) | Error message appears below tile group: 14px `text-error`, role="alert" |
| Saved (settings form) | Toast notification, button returns to idle state |

## Accessibility

- All form inputs have associated labels (enforced by `InputField` component)
- Modal traps focus (handled by Radix `DialogPrimitive.Content`)
- Escape key suppressed on this modal (mandatory flow — override Radix default via `onEscapeKeyDown={(e) => e.preventDefault()}`)
- `DialogOverlay` click suppressed (`onPointerDownOutside={(e) => e.preventDefault()}`)
- Progress dots: `aria-label="Step {n} of 9"` on the active dot, `role="tablist"` on container
- Platform tiles: rendered as `<button>` with `aria-pressed={selected}`
- Multi-select tiles: `aria-pressed` pattern throughout
- Card 5 remove buttons: `aria-label="Remove creator {index + 1}"` (see Card 5 interaction spec above)
- Card 6 remove buttons: `aria-label="Remove win {index + 1}"` / `aria-label="Remove flop {index + 1}"` (see Card 6 interaction spec above)
- Card heading is the `DialogTitle` (already handles aria labeling via Radix)
- `role="alert"` on all validation error messages
- WCAG AA minimum on all text: all color pairs from existing verified token set (5.4:1+ for muted, 7.2:1 AAA for primary)
