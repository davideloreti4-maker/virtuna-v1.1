## Color

Read from `globals.css` flat-warm SSOT. 60/30/10 split:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-charcoal-app` #262624 | Page/app background, library surface |
| Secondary (30%) | `--color-charcoal-sidebar` #1a1a18 (nav, via `--color-background-elevated`) · `--color-charcoal-composer` #1e1d1b (cards/surface) · `--color-charcoal-chip` #2f2e2b (active chips / lifted matte) | Sidebar, library item cards, audience rows, edit-form surface, type tags, active filter chip |
| Accent (10%) | `--color-coral-500` (oklch 0.68 0.13 33, terracotta clay ≈ #d97757) | See reserved list below |
| Destructive | `--color-error` (oklch 0.60 0.20 25) | "Remove from shelf" + "Delete audience" confirmations only — distinct from coral (hue ~25 vs ~33) |

**Text:** `--color-cream-primary` #ece7de (titles/primary), `--color-cream-secondary` #c2bdb4 (default chrome/body/nav), `--color-cream-muted` #8a857c (labels, timestamps, placeholders, helper, em-dash empty). **Never pure white.**

**Borders:** `--color-border` rgba(255,255,255,0.06); hover `--color-border-hover` rgba(255,255,255,0.1). Card/row hover = `bg-white/[0.02]` (no translate, no border change — CLAUDE.md). Sidebar matte: solid `bg-background-elevated` + hairline; **NO glass / blur / inset-shine** on the sidebar (THEME-02 Layer B).

**Accent reserved for (explicit — one accent per surface, honesty/one-accent discipline; never "all interactive elements"):**
- The **"New Thread"** primary CTA in the sidebar (the shipped `accent` `NavItem` — the ONE accent in the nav).
- Active-nav indicator is **NOT coral** — it is `bg-white/[0.06]` + `text-foreground` (shipped active state). The Library item's active state matches Thread/Audience/Settings exactly.
- Primary CTA buttons: "Use in thread →" item launch, "Compare" launch, "Save changes" on the persona-edit form, "Create audience" (shipped). One per surface.
- Focus rings (`--color-accent/50` ring, shipped `focusRing`).
- Links inside copy.

**Accent must NOT touch:**
- The Library nav item's icon/label or its active state (matte white/[0.06], identical to the other three nouns — coral here would break the one-accent rule, since "New Thread" already owns the nav's accent).
- Filter chips / item-type tags (active chip = `--color-charcoal-chip` + cream text; mirrors STATE 05-04 "checkmark uses cream, never coral").
- Multi-select **checkboxes / selected-row state** on the audience surface (use neutral `--color-charcoal-chip` / `bg-white/[0.06]` + a cream check; selection is not an "action accent"). The audience **type-segmented** control's selected state is the only audience-surface element allowed coral (shipped `border-accent/60 bg-accent/10 text-accent` — pre-existing, in the create/edit form, not the compare selection).
- Persona-display badges (temperature/disposition) — shipped semantic/neutral variants (`info`/`success`/`default` + `bg-white/[0.06]`), NOT coral.
- The General-audience badge stays the shipped `Badge variant="accent"` (it is the pre-existing protected-baseline marker, not new).

**Honesty / degraded states:** thin-audience / not-enough-data / read-only-General use warning-toned `--color-warning` or neutral cream-muted, **NEVER `--color-error`, NEVER coral** (mirrors STATE 07-05). An empty Library or a read-only General persona table is a calm informational state, not an error.

## Accessibility & States

- **Loading:** Library reuses the shipped `Loading your library…` text state; the sidebar history reuses its `Skeleton` rows. Compare uses the P8 Read's own loading.
- **Empty:** Library empty + filtered-empty states (copy above); audience compare needs ≥2 audiences (degrade calmly if the user has fewer); read-only General persona table (informational, not empty-error).
- **Error vs honest-degradation:** network/launch/save failure = error tone (toast). Thin-audience / read-only General / not-calibrated-enough = **warning or muted tone, never error, never coral**.
- **Focus:** accent focus ring (`--color-accent/50`, shipped `focusRing`) on all interactive controls — nav items, filter chips, checkboxes, edit-form inputs, dialog actions. Keyboard-operable selection mode + dialogs (Radix handles dialog focus-trap + Escape; the shipped account/menu pattern handles outside-click + Escape).
- **ARIA:** Library nav item `aria-current="page"` when active (shipped); filter chips `role="tab"`/`aria-selected` (shipped); compare checkboxes labeled per audience; edit form fields label-associated; the Library item collapsed-rail `Tooltip` provides the accessible name.
- **Touch targets:** ≥44px effective via the shipped `pointer-coarse:` bumps (audience rows, icon buttons, primary CTAs). Nav keeps `min-h-[34px]` (the shipped baseline; matches the other three nouns).
- **Contrast:** cream-on-charcoal and accent-foreground (#1a0f0a on terracotta coral) verified ≥ AA per SSOT. Never introduce pure white.
- **Reduced motion:** inherit the shipped `usePrefersReducedMotion` / global reduced-motion CSS — no new load-bearing animation.
