<objective>
Build the calibration pipeline (AUD-02) and the audience CRUD + calibrate routes (AUD-01 route surface). Calibration: scrape the creator's OWN handle (D-06) via the shipped `ApifyScrapingProvider`, derive an Audience Profile (temperature mix / dispositions / follower tier) using `getFollowerTier`, repaint the 10 byte-stable personas with stored deterministic descriptions, bake the goal-intent bias (07-01 `biasForGoalIntent`) into `persona_weights` ONCE at calibration, and persist via `audience-repo`. Apify latency (Pitfall 4) is handled by an SSE route with honest staged status.

Purpose: turn a handle or a description into a persisted, reusable, cache-stable Audience without ever mutating the engine or breaking the honesty spine.
Output: `calibration.ts` + `persona-repaint.ts` + 3 routes + tests.
