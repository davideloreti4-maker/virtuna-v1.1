# Phase 7 — UI Design Contract

> Visual and interaction contract for the Audience Manager, calibration flow, read-only Audience Profile + persona display, and the composer audience chip.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> SSOT: **THEME-06 flat-warm visual system** (NOT Raycast glass) layered on the Raycast-derived token set. Where CLAUDE.md "Raycast Design Language Rules" and THEME-06 conflict on surface treatment, **THEME-06 flat-warm wins** (solid charcoal surfaces, no glass/blur/inset-shine on app chrome). Borders/radius/typography tokens are shared.

## Spacing Scale

Declared values (multiples of 4 — matches `globals.css` `--spacing-*`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, chip inner padding |
| sm | 8px | Compact element spacing, badge/chip gaps |
| md | 16px | Default gap/padding (form fields, card body) |
| lg | 24px | Section padding, Profile region gaps |
| xl | 32px | Manager page section breaks |
| 2xl | 48px | Major Profile/list separation |
| 3xl | 64px | Page-level top spacing |

Exceptions:
- **44px minimum touch target** for the audience-chip trigger, dropdown/bottom-sheet rows, persona table rows on touch, and all Manager action buttons (`pointer-coarse:h-11`, mirrors composer `+` button). Mobile-first is non-negotiable (v5.0 numen-rework).
- Sidebar nav rows: `min-h-[34px]` — **inherited from the existing `NavItem` (`Sidebar.tsx`); this phase reuses it and does not introduce or modify it. 34px is not a 4-multiple — exception acknowledged, not a phase-introduced violation.** Manager list rows `min-h-[44px]` on touch.

## Typography

Pulled from `globals.css` type scale. This phase declares **4 sizes** and **2 weights**:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px (`--text-sm`) | 400 (`--font-regular`) | 1.4 |
| Body | 16px (`--text-base`) | 400 (`--font-regular`) | 1.5 |
| Heading (section / card title) | 20px (`--text-xl`) | 600 (`--font-semibold`) | 1.25 |
| Display (Manager page title) | 24px (`--text-2xl`) | 600 (`--font-semibold`) | 1.2 |

**Declared sizes:** 14 / 16 / 20 / 24. **Declared weights:** 400 (regular) + 600 (semibold). No 700 in this phase.

**Inherited (NOT declared/introduced by this phase) — present only on reused, shipped components:**
- **10px** uppercase tracked caps — inherited from the existing `SectionLabel` (`Sidebar.tsx`) and chip prefix words ("on" / "for"). Reused as-is; this phase introduces no 10px role.
- **12px** (`--text-xs`) — inherited micro-text on existing `_kit` primitives, chips, and sidebar rows (share %, timestamps, chip sub-labels). Reused as-is.
- **500 (`--font-medium`)** — inherited label weight on reused components (`NavItem`, chip labels). Not introduced by this phase; new surfaces use 400/600 only.

## Color

Flat-warm matte. 60/30/10 split:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#07080a` (`--color-background`) | Page body behind the Manager |
| Secondary (30%) | `#1a1b1e` (`--color-background-elevated`) sidebar / panels; `#18191a` (`--color-surface`) cards; `#222326` (`--color-surface-elevated`) dropdowns/popovers/dialogs | Manager shell, audience cards, persona detail card, dropdown surface |
| Accent (10%) | `#FF7F50` (`--color-accent`) | See reserved list below ONLY |
| Destructive | `--color-error` (`oklch(0.60 0.20 25)`) | Delete-audience confirmation action + invalid-input border |

**Accent (coral) reserved for — explicit, exhaustive:**
1. Primary CTA on the Manager (the single "Create audience" / "Save" primary button — `Button variant="primary"`).
2. The **active** state of the composer audience chip (selected audience indicator — coral border `border-accent/60` + `bg-accent/10` + `text-accent`, matching `PlatformChip` active).
3. The **active**-row marker + selection check in the audience picker dropdown/sheet.
4. Focus rings (`ring-accent` / `ring-accent/50`) on all interactive elements.
5. The **General (locked default) badge** — coral indicates the protected baseline audience.

**Accent is NOT used for:**
- Persona temperature/disposition labels — use **semantic + neutral** tones, never coral as a category color (would collide with the "one thing to act on" rule).
- The calibration "couldn't read enough — using General" fallback notice — that uses `--color-warning` (honest, cautionary, not alarming), NOT coral and NOT error.
- The PersonaGraph nodes generally — coral is reserved inside the graph for the single weakest/worst cluster only (the existing `tone:'accent'` contract). In v1 read-only display with untuned values, default to **no coral cluster** unless a clear weak signal exists.

**Temperature palette (the creator-facing lens, D-02) — non-coral, semantic-mapped:**
| Temperature | Token / treatment |
|-------------|-------------------|
| cold | `--color-info` family (cool blue) at low chroma — "needs warming" |
| warm | `--color-foreground-secondary` neutral (`#9c9c9d`) — the middle/default |
| hot | `--color-success` family (green) at low chroma — "ready to convert/share" |

Render temperature as a `Badge` dot + word, NOT a saturated fill. Disposition (scanner/skeptic/collector/connector/converter/lurker) renders as a neutral text `Badge` (`bg-white/[0.06]`, `text-foreground-secondary`) — a label, not a color signal.

**Status tokens (reuse existing):** success/warning/error/info from `globals.css` for calibration progress, fallback notice, and validation.

## Component & Interaction Contracts (phase-specific surfaces)

### 1. Audience Manager (CRUD)
- **Entry:** new sidebar `NavItem` ("Audience", Phosphor `UsersThree` or `Users`), placed in the Settings group region above "Simulations", collapses to icon + tooltip in the rail (mirror existing `NavItem` collapsed behavior). Routes to `/audience`.
- **Page shell:** `mx-auto max-w-4xl p-6` flat-warm page. Title 24px/600 "Your audiences". Subtitle 14px muted: "Who Numen writes and tests for."
- **List:** vertical stack of audience cards (`bg-surface`, `border-white/[0.06]`, `rounded-xl` 12px, hover `bg-white/[0.02]` only — no lift). Each card row: name (16px/500) · `platform · type` sub-label (12px muted) · trailing **General badge** (coral) on the locked default, or a `⋯` overflow menu (edit/delete) on user audiences — the `⋯` trigger is icon-only and MUST carry `aria-label="Audience options"`. The 3 presets (General + growth-leaning + conversion-leaning templates) appear as cards; **General has no delete affordance** (locked default — D-04).
- **Create/Edit:** routes to `/audience/new` and `/audience/[id]` (full page on mobile, not a cramped modal — mobile-first). Fields: name (`Input`, required), type (segmented `personal | target` — `Toggle`-group or 2-chip segment), platform (`Select`: TikTok / Instagram / YouTube / Custom), goal (free-text `Input` label + intent surfacing per §Goal below). **No persona-edit affordances in v1** (D-03).
- **Delete:** `Dialog` (solid opaque) confirmation. See Copywriting.

### 2. Calibration flow
- **Personal path (D-06):** single field "Your @handle" → "Calibrate audience" primary button → progress state. Progress = `Spinner` + staged status text (reuse the streaming-stages pattern the thread views already render). Honesty spine: while scraping, show honest stage labels ("Reading your followers…"), never fabricated counts.
- **Target path:** free-text describe textarea → "Calibrate audience".
- **Fallback state (MANDATORY, D-06):** if scrape is empty/thin/fails → render a `--color-warning` inline notice banner (same banner shape as composer test-brief, but warning-toned): heading + body + a "Continue with General" affordance. **Never fabricate** a calibrated audience. The notice is calm, not an error toast.
- **Success:** transition to the read-only Audience Profile (§3).

### 3. Audience Profile + persona display (READ-ONLY v1, D-03)
- **Hero:** `PersonaGraph` node-cloud renders the 10 calibrated personas (the "see your audience" moat surface). Thread `reducedMotion` through. Map persona `weight` ← share %, `label` ← persona name. Keep the a11y sr-only mirror. In v1, **no coral cluster** by default (values untuned) — pass `tone:'default'` unless a real weak signal exists.
- **Profile header tiles:** `StatTileRow` — platform, goal (the free-text label), temperature mix (e.g. "40% warm · 35% cold · 25% hot"), top dispositions. Labels wrap, never truncate.
- **Persona list:** `DataTable` — columns: Name · Temperature (`Badge` dot+word) · Disposition (neutral `Badge`) · Share %. **No edit/row-action affordances** (read-only). Build the markup so an edit column can be added later without restructure (D-03 "design so editing can be added later").
- **Read-only affordance honesty:** a small muted caption — "Read-only for now. Editing arrives once values are tuned." — so the absence of edit is intentional, not broken.

### 4. Composer audience chip (D-04) — reconcile with sketch 001
- **Constraint discovered:** the composer chip row is already full — 6 tool chips (`Test/Idea/Hooks/Chat/Script/Remix`) + `PlatformChip` (3 platforms) + active-model label, wrapping (`flex-wrap`) on `app/home/composer.tsx` L719. Adding a 9th always-expanded inline chip-group overflows on mobile.
- **Recommended pattern (sketch 001 Variant A on desktop + Variant B on mobile, reconciled):** a **single collapsed audience pill** in the composer chip row showing the active audience as `platform · name` (e.g. "TikTok · My audience"), prefixed by a `for` micro-label (mirrors `PlatformChip`'s "on" prefix). Tapping the pill opens:
  - **Desktop:** a compact inline **dropdown** (`bg-surface-elevated`, 12px radius, `shadow-float`) listing audiences (General + presets + user audiences) with the active one coral-checked, plus a "Manage audiences →" footer link to `/audience`.
  - **Mobile:** a native-feeling **bottom sheet** with full-width 44px audience rows. Do NOT use the desktop dropdown on touch.
  - This deliberately matches sketch 001's pill→dropdown / pill→bottom-sheet direction so the audience selector and the (future) skill selector share ONE interaction grammar. Do **not** introduce a conflicting pattern. Command-palette (Variant C) is out of scope for v1.
- **Active state:** the pill uses the coral active treatment ONLY to mark "an audience is selected" — General (default) shows neutral, a calibrated audience shows coral border/tint (signals the moat is engaged).
- **Per-thread pin (D-04):** selection pins per-thread; new threads pick fresh; switching mid-thread re-grounds future turns only. The pill reflects the current thread's pinned audience. No global single-active toggle.
- **Default visible:** the chip is present whenever a generation/test tool is active (same `showPlatformChip` gating set), defaulting to "General".

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (Manager) | **Create audience** |
| Primary CTA (calibration, personal) | **Calibrate audience** |
| Manager page title / subtitle | "Your audiences" / "Who Numen writes and tests for." |
| Empty state heading | "No custom audiences yet" |
| Empty state body | "You're using **General** — Numen's universal audience. Calibrate a personal audience from your own @handle, or start from a template, to test against the people who actually watch you." (CTA: Create audience) |
| Calibration progress | "Reading your followers…" → "Building your audience profile…" (honest stages, no fabricated counts) |
| Calibration fallback (D-06) | Heading: "Couldn't read enough yet" · Body: "We couldn't read enough from @{handle} to calibrate a personal audience, so Numen is using **General** for now. Try again when your account has more public activity." · Action: "Continue with General" (warning-toned, never error, never fabricated) |
| Error state (scrape/network) | "Calibration failed. Check the handle and try again." (with Retry) — distinct from the honest "thin data" fallback above |
| Read-only persona caption | "Read-only for now. Editing arrives once values are tuned." |
| General badge tooltip | "Numen's universal audience — the protected baseline. Can't be edited or deleted." |
| Composer chip prefix | "for" (e.g. "for TikTok · My audience") |
| Picker footer | "Manage audiences →" |
| Destructive confirmation | **Delete audience** — "Delete "{name}"? This removes the calibrated audience and its personas. Threads already generated under it keep their results. This can't be undone." Buttons: "Keep audience" (secondary) / "Delete" (destructive). |

Voice: honesty spine throughout — never imply a calibrated audience exists when it doesn't; never fabricate follower/persona data; the fallback is calm and factual.
