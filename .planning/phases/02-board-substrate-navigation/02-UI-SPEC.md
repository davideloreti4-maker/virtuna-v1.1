## Typography

Declared subset for this phase. Full scale in BRAND-BIBLE.md. Using existing tokens only (NF3).

| Role | Size | CSS var | Weight | Weight value | Line Height | Usage |
|------|------|---------|--------|--------------|-------------|-------|
| Board label / node title | 12px | `--text-xs` | semibold | 600 | 1.2 | Group frame title bar, Engine stage glyphs, sidebar item labels (icon-only mode), camera preset button labels |
| Body / sidebar text | 14px | `--text-sm` | regular | 400 | 1.5 | Sidebar section items (Recent, Pinned), command bar placeholder text, Input node snippet, drawer form labels, tooltip copy |
| Node heading / sidebar section header | 16px | `--text-base` | semibold | 600 | 1.2 | Sidebar section headers ("New analysis", "Running", "Recent"), node card headings, command bar chip labels |
| Stage label / streaming text | 14px | `--text-sm` | medium | 500 | 1.5 | Live stage text above Engine group during streaming ("Reading the hook…"), drawer "Recent inputs" section header |

**Weight discipline:** Use only `regular (400)` and `semibold (600)` for all Phase 2 UI text. `medium (500)` reserved exclusively for the streaming stage label (transient, animated text where medium reduces visual jitter vs semibold). No other weights.

**Letter spacing:** `0.2px` globally applied (globals.css body). No per-component overrides.

### Konva Layer Architecture

- **Layer 0:** Background fills + wires (Konva `Rect` + `Line`)
- **Layer 1:** Group frame shapes (Konva `Rect` + title bar `Text`)
- **Layer 2:** Hit-test areas (Konva transparent shapes)
- **DOM Overlay (z above Konva):** All readable text, charts, accessibility, interactive form elements

All text content (labels, metrics, reasoning) lives in DOM overlay, not Konva canvas text. Konva renders spatial primitives only.

## Interaction Contract

### Pan / Zoom (Camera)

| Interaction | Desktop | Mobile |
|-------------|---------|--------|
| Pan | Click + drag on canvas | Two-finger drag |
| Zoom | Scroll wheel | Pinch gesture |
| Fit to content | `0` key, or "Overview" button | "Overview" button |
| Camera presets | `1`/`2`/`3` keys, or preset buttons | Preset buttons |
| Reset | `R` key, or "Reset" button | "Reset" button |
| Auto-follow | During streaming (cancels on user touch) | During streaming (cancels on user touch) |

Reduced-motion (`prefers-reduced-motion: reduce`): pan/zoom instant (no easing animation), auto-follow disabled, no shimmer animations, board layout and camera presets remain fully functional.

### State Machine Transitions

| From | To | Trigger | Visual Change |
|------|----|---------|---------------|
| `idle` | `streaming` | Analysis submitted | Command bar disables, Engine frames activate, camera auto-follows |
| `streaming` | `complete` | SSE `complete` event | Engine collapses to badge, camera glides to Audience+Verdict, command bar re-enables |
| `complete` | `anti-virality` | SSE `complete` + threshold triggered | Verdict + Audience frames get orange border treatment, Actions promotes reshoot |
| `complete` | `edit-input` | Tap Input node | Input drawer opens from left (desktop) or bottom (mobile) |
| `edit-input` | `streaming` | Re-run submitted | Drawer closes, board updates in place |

### Focus Management (Accessibility — D-24)

- Tab order: Sidebar → Command bar → Group frames (spatial reading order: Input → Engine → Audience → Verdict → Actions → Content Analysis)
- Focus ring: `focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2` (existing button.tsx pattern)
- Focus rings: visible, never suppressed
- `aria-live="polite"` region: Engine group (announces stage transitions during streaming)
- `aria-live="assertive"` region: Error states (connection drop, cancel confirmation)
- Drawer focus trap: standard Sheet/radix behavior (existing component)
- Keyboard nav within board: Arrow keys move between group frames when frame is focused

## Performance Tier Visual Degradations

Derived from CONTEXT.md D-20, D-21.

| Tier | Target FPS | Visual changes vs High |
|------|-----------|------------------------|
| High | 60fps | Full effects: shimmer, camera easing, frame transitions |
| Medium | 45-60fps | Reduced: no shimmer animation on frames, simpler camera easing (linear instead of quart) |
| Low | 30fps min | Auto-engage `prefers-reduced-motion` subset: no animations, instant camera, no shimmer |

Auto-downgrade: sustained <40fps for 3s → drop tier + "Optimized for your device" toast.
Manual override: `settings?tab=performance` (existing settings pattern; force tier selector deferred to Phase 8 polish).
