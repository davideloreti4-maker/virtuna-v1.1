# Phase 13 — UI Design Contract (Ambient Numen)

> Visual and interaction contract for the living, always-present audience (AMBIENT-01). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase EXTENDS the shipped P9 AudienceLens + THEME-06 flat-warm system. It does NOT introduce a new design language, new tokens, or a new Lens.** Every value below is a real shipped token from `src/app/globals.css` or a real shipped component in `src/components/audience-lens/`. The contract's job is to specify the *new surfaces* (persistent presence, per-card reaction-at-rest, type-to-room input) using those existing primitives — nothing invented.

## Typography

All sizes are real `--text-*` / inline `text-[Npx]` values already used across the shipped Lens and cards. **Hold to ≤4 sizes + ≤2 weights on the new Ambient surfaces** (the shipped components already obey this).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / reaction line | 14px (`--text-sm` / `text-[14px]`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) |
| Label / subject ("reacting to: X", roster name, honesty caption) | 13px (`text-[13px]`) | 500 (`--font-medium`) | 1.25 (`--leading-snug`) |
| Heading / presence title ("Your audience") | 15px (`text-[15px]`) | 600 (`--font-semibold`) | 1.25 (`--leading-snug`) |
| Micro / sr-context + uppercase eyebrow | 11px (`text-[11px]`) | 500 (`--font-medium`) | 1.25 |

**Rules:**
- Exactly **two weights** in use on Ambient surfaces: 400 (body/reaction text) and 600 (headings/emphasis). 500 (`--font-medium`) is permitted for labels/captions ONLY (it is the shipped Lens caption weight) — do not introduce 700 on any Ambient surface.
- The "reacting to: `<concept>`" subject label: 13px / weight 500, with the **concept clause** in `--color-foreground` and the static `reacting to:` prefix in `--color-foreground-muted`. Concept text truncates with ellipsis at one line (`truncate`) — never wraps to dominate the strip.
- Live counts (stop/total fractions on the presence) use `tabular-nums` (matches `PopulationSwarm`).
- Serif (`--font-serif`) is **forbidden** on every Ambient surface — these are chrome/data, not voice moments.

## Color

All values are real semantic tokens from `globals.css`. **Coral is a SIGNAL, not decoration** — this is the load-bearing color rule for this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (~60%) | `--color-background` = `#262624` (charcoal-app) | Thread column + presence strip background at rest |
| Secondary (~30%) | `--color-surface` = `#1e1d1b` / `--color-surface-elevated` = `#2f2e2b` | Presence expanded panel, the type-to-room input field, the promoted-cue resting chip |
| Text primary | `--color-foreground` = `#ece7de` (cream-primary, NEVER `#fff`) | Reaction text, subject concept clause, roster names |
| Text secondary | `--color-foreground-secondary` = `#c2bdb4` | Verbatim quote text (italic), de-emphasized reaction detail |
| Text muted | `--color-foreground-muted` = `#8a857c` | "reacting to:" prefix, honesty captions, idle/empty copy, timestamps, the em-dash degrade |
| **Accent (~10%) — SIGNAL ONLY** | `--color-accent` = `#d97757` (terracotta, `--color-coral-500`) | See reserved-for list below. Render via `var(--color-accent)` — NEVER the legacy hardcoded `#FF7F50` |
| Borders | `--color-border` = `rgba(255,255,255,0.06)`; hover `--color-border-hover` = `rgba(255,255,255,0.1)` | Presence strip hairline, input border, expanded-panel top border |
| Hover/active surfaces | `--color-hover` = `rgba(255,255,255,0.05)`; `--color-active` = `rgba(255,255,255,0.1)` | Cue hover, input focus surface lift |
| Cream-alpha dots | `rgba(236, 231, 222, α)` where α = `0.2 + watchThrough*0.5` | Persistent-presence dot-cloud fill (verbatim from `PersonaGraph`/`PopulationSwarm`) |

**Accent (`#d97757`) reserved for — the COMPLETE list on Ambient surfaces:**
1. The **worst cluster's dots** in the presence viz (the single weak/worst slot — `tone:'accent'`, exactly as `PersonaGraph`/`PopulationSwarm` paint it). This is the only data-driven coral.
2. The Lens's inherited **"Rewrite for this audience →" CTA** + inherited Read lever (already coral inside `AudienceLens` — do not change).
3. The **`MAX` badge / active skill-pill icon** where the composer already uses it (unchanged; not new to this phase).

**Coral is explicitly FORBIDDEN on (honesty + one-accent law):**
- The persistent presence container, its border, its title, the "reacting to:" subject label, or the subject concept text.
- The per-card reaction-at-rest sentiment ribbon when the verdict is positive/neutral (a "good" reaction is **cream**, never coral — coral would falsely read as alarm). The sentiment ribbon may only go coral for the worst-cluster signal, consistent with the dot rule.
- The type-to-room input, its placeholder, its send affordance, or its loading state.
- Any "live / reacting" pulse or presence-is-active indicator. Liveness reads via **motion + cream opacity**, never a coral glow.

**Score-zone semantic colors** (`--color-success` green / `--color-warning` amber / `--color-error` red) — these are DATA colors retained for the per-card **band chip** (Strong/Mixed/Weak) that already ships on each card. The reaction-at-rest promotion (Surface 3) keeps the existing band chip semantics untouched. Do NOT use error-red for the honest-degrade or thin-signal states — those are `--color-foreground-muted` (an em-dash / quiet caption), never an error tone.

**⚠ Known inconsistency to correct, not propagate:** `hook-card-block.tsx` and `idea-card-block.tsx` hardcode the legacy `#FF7F50` coral on their archetype tag chips (`color: '#FF7F50'`, `rgba(255,127,80,…)`). THEME-06 SSOT is `--color-accent` (`#d97757`). New Ambient surfaces MUST use the token. The executor SHOULD migrate any per-card-reaction code it touches to the token while promoting the cue (do not introduce a third coral value).

| Destructive | not applicable | **No destructive actions in this phase.** Ambient adds reactions + an input; it deletes/mutates nothing. No confirmation surfaces required. |

## Copywriting Contract

All copy is glanceable, honest, and lower-case-leaning to match the shipped Lens/composer voice. **The honesty spine is a copy constraint:** never imply a reaction exists when none is in focus; always name the one real concept; degrade to quiet, not to a fabricated crowd.

| Element | Copy |
|---------|------|
| Presence title | `Your audience` |
| Presence subtitle (calibrated, has personas) | `{N} calibrated people` (e.g. "10 calibrated people"); General → `General audience · default panel` |
| **Live spotlight subject (D-02, REQUIRED)** | `reacting to: ` + the concept clause (the hook line / idea title / your typed thought, truncated one line). The prefix is muted, the concept is foreground. NEVER an aggregate — exactly one labeled concept. |
| Idle / at-rest state (D-01 — nothing in focus) | `Your people are here. Make something — or type a thought to test it.` (roster shows, NO reaction rendered) |
| Per-card reaction-at-rest (Surface 3) | The existing real signal, surfaced at rest: `{stop}/{total} stop` + the lead verbatim `"{scrollQuote}"`. NO new sentence — promote what the card already emits. |
| Per-card thin-signal degrade | Render **nothing** (the cue collapses to its plain child — `flatPersonas.length === 0 → return children`, the shipped `LensTrigger` behavior). No "no reaction" placeholder on a card; silence is the honest state. |
| Type-to-room input placeholder | `Type a thought — see how the room reacts` |
| Type-to-room loading affordance | `Reading the room…` (verbatim reuse of the shipped `PopulationSwarm` cascade label — one vocabulary for "the SIM is reacting") |
| Type-to-room result honesty caption | `A quick SIM read on your {audienceName} — not a full Test.` (sets expectation: fast Flash reaction, not the Max video Read) |
| Type-to-room empty/short input | Send disabled (≥1 non-whitespace char), no error — mirrors the shipped `PersonaChatDrawer` send-gate |
| Primary CTA (within the input) | `Test this →` (the send affordance label / aria — frames it as foresight, not chat) |
| Empty state heading (no audience calibrated — General only, presence still shows) | `Your audience` (presence never hides; it shows the General panel) |
| Empty state body | `Calibrate an audience to make the room yours.` → links to `/audience` (reuses the composer "Manage audiences" affordance pattern) |
| Error state (type-to-room SIM failed) | `Couldn't reach the audience right now. Your thought is saved — try again in a moment.` (verbatim-aligned with the shipped `PERSONA_CHAT_ERROR`) → quiet `Retry →` in `--color-cream-secondary`, never error-red |
| Destructive confirmation | none — no destructive actions in this phase |

**Voice rules:** no exclamation hype, no fabricated persona names in copy, no "the AI thinks…" framing. The audience "reacts" / "stops" / "scrolls" — it never "says" anything the SIM didn't return. "Read" (the SIM reaction) is the noun; "reacts" is the verb.

## Surface Contracts (the new/changed Ambient surfaces)

> All surfaces are **fixed typed renderers** (THREAD-04 / craft-trap rule) — NOT model-generated UI. All mobile-first. All reuse the shipped Lens + dot-cloud family.

### Surface 1 — Persistent living-audience presence (D-01)

**Form (recommended, citing the shipped family):** a **thin persona-cloud strip** — the dot-cloud family from `PersonaCloud`/`PersonaGraph`, reduced to one dot per calibrated persona, rendered in a short horizontal band. NOT a generic sentiment bar (a bar discards the "your people are individuals" read that is the moat; the dot-cloud is the shipped visual language for "your audience"). Cream-alpha dots; the worst-cluster dot reads coral (only when a concept is in focus and a worst cluster exists). At rest (idle), all dots render at a calm uniform cream — the roster, no verdict.

**Dock location (decided):** the **top of the thread column** (`max-w-[760px] mx-auto`), as a **sticky strip** (`position: sticky; top: 0`) above the card stream — NOT on the composer. Rationale grounded in the shipped layout: `composer.tsx` renders thread views in the scrollable `max-w-[760px]` column *above* the bottom-pinned composer form, and the composer is already dense (`[+] · skill pill · audience · intent · model · send` + upward popovers). Docking the presence on the composer would crowd the one control row and collide with the upward-opening popovers. The sticky thread-header strip keeps the audience "always felt" as the creator scrolls the ledger, without stealing composer space. (`z-index`: below `--z-sticky` 200 / below the composer's popovers — use a local stacking context, not a global z.)

**States:**
- **Collapsed (default, mobile):** 48px-tall strip = title `Your audience` + the thin dot-cloud + (when a concept is in focus) the `reacting to: X` subject inline. One ≥44px tap target opens the Lens for the in-focus concept (Surface 5). A chevron/affordance expands.
- **Expanded:** self-sizing panel (`max-h-[40vh]`, internal scroll) on `--color-surface` with a hairline top border — shows the larger dot-cloud + the type-to-room input (Surface 4) + the subject. Reuses the Lens's region rhythm at smaller scale. Collapses on outside-tap/Escape (reuse the `composer-controls.tsx` outside-click/Escape effect).
- **Idle / at-rest (honesty-critical):** roster dots only, NO reaction, idle copy. The presence NEVER invents a reaction to nothing.

**Motion:** reuse `PersonaGraph`'s `reducedMotion`-gated `<animate>` opacity pulse on the dots (`values="0.85;1;0.85"`, staggered `dur` by index) for the calm "alive" feel; HARD-STOP under `prefers-reduced-motion` (the dots simply rest). Always render the sr-only roster mirror (the `<ul className="sr-only">` pattern from `PersonaCloud`). No `Math.random` — deterministic layout only.

### Surface 2 — Live spotlight + "reacting to: X" subject (D-02)

**Subject label (REQUIRED affordance):** `reacting to: {concept}` (typography + color per the Color/Typography sections). Always present whenever a reaction is shown; absent only in the idle state. Exactly ONE concept — never an aggregate, never blended across cards.

**Default focus:** the **latest generation** (the most recent card in the thread). On thread load with persisted cards, focus the last card.

**Re-focus interaction model (decided — both, with scroll-spy as the ambient default):**
1. **Scroll-spy (primary, ambient):** as the creator scrolls the card ledger, the presence re-focuses to the card crossing a focus line (e.g. the card nearest the top of the viewport under the sticky strip). Throttled; updates the subject + dot toning to that card's real reaction. This is the "moving spotlight" the owner described.
2. **Explicit tap (deterministic):** tapping a card (or its promoted cue) sets focus to that card immediately and is sticky until the next tap or a deliberate scroll. Tap wins over scroll-spy momentarily so a deliberate examination isn't yanked away.
3. **Type-to-room (Surface 4):** submitting a typed thought sets the subject to that ad-hoc thought and focuses its returned reaction.

**Transition:** when focus changes, the dot toning + subject cross-fade (≤200ms, `--duration-normal`, `--ease-out-cubic`); reduced-motion → instant swap. The reaction data is the focused card's **already-emitted** `fraction` + `scrollQuote` (no new model call on re-focus — determinism-gate-safe, D-03). Each card keeps its OWN durable inline reaction (the per-card ledger, Surface 3) — the spotlight is a view onto them, never a running average.

### Surface 3 — Per-card reaction at rest (D-01/D-03) — a PROMOTION of `LensTrigger`

**This is NOT a new component.** It is the existing `LensTrigger` cue, promoted from a quiet "see the room" row into a resting-state reaction readout. Today the four card blocks already wrap their lead `scrollQuote` blockquote in `<LensTrigger flatPersonas={cardScrollQuoteReactions(fraction, scrollQuote)} …>`. The promotion adds, **inside that same wrapped resting cue** (no sibling element):

**Anatomy (resting state), top-to-bottom:**
- The real **stop fraction** `{stop}/{total} stop` — small, `tabular-nums`, `--color-foreground` (NOT the band chip; the band chip stays where it is).
- A **thin sentiment ribbon** — a 2–3px-tall horizontal bar, width-split stop(cream) vs scroll(muted track). Worst-cluster-driven coral allowed only consistent with the dot rule; a positive read is cream, never coral.
- The existing lead **verbatim** `"{scrollQuote}"` (already present — keep it, `--color-foreground-secondary`, italic).

**Spacing/typography:** fraction at 13px/500; ribbon `h-[3px] rounded-full`; quote 14px/400 italic. Whole cue ≥44px tap; hover lifts to `--color-hover`. Tapping opens the Lens scoped to that card (unchanged `LensTrigger` behavior, Surface 5).

**Degrade-to-nothing (honesty):** when `cardScrollQuoteReactions(...)` returns `[]` (unparseable fraction / no signal), the cue collapses to its plain child — the shipped `flatPersonas.length === 0 → return children`. NO ribbon, NO "0/0", NO placeholder. Silence is honest.

### Surface 4 — Type-to-room input (D-04)

**Where it sits:** inside the **expanded presence** (Surface 1) — a single-line input docked at the bottom of the expanded panel. On collapsed mobile, tapping the strip's input affordance expands the presence and focuses the input. It is NOT the main composer (the main composer routes to skills; this is the ad-hoc "test a thought" reaction).

**States:**
- **Resting:** input on `--color-surface`, border `--color-border`, 42px tall (matches the shipped input height idiom), placeholder `Type a thought — see how the room reacts`. Send affordance (`Test this →`) disabled until ≥1 non-whitespace char.
- **Active/focus:** border → `--color-border-hover`, surface → subtle `--color-active` lift. Enter submits, Shift+Enter newline (reuse `PersonaChatDrawer` keydown).
- **Loading (latency affordance):** on submit, show `Reading the room…` (the shipped cascade vocabulary) with an `aria-live="polite"` region; **debounce is not the model** — the SIM only fires on explicit submit (no keystroke-fired calls; the live-composer-as-you-type stretch is explicitly DEFERRED per CONTEXT). The Flash reaction is genuinely fast but not instant — the label is honest about a real (sub-few-second) read.
- **Result:** the returned reaction (`fraction` + verbatim quotes) flows into the **spotlight** (sets subject = your typed thought, tones the dots) AND auto-opens the **Lens** scoped to that thought (Surface 5). Show the honesty caption `A quick SIM read on your {audienceName} — not a full Test.`
- **Error:** the verbatim error copy + quiet `Retry →` (Copywriting section). Never error-red.

**Data path (for the planner — this is NOT a new engine):** type-to-room reuses the **Flash text-mode reaction primitive** every card already uses — `runFlashTextMode(text, mode, panel)` → `aggregateFlash(personas)` → `{band, fraction}` + `selectLeadScrollQuote(personas)` → `scrollQuote`, then `cardScrollQuoteReactions(fraction, scrollQuote)` for the Lens. (Source: `ideas-runner.ts` L327–382 / `src/lib/engine/flash/run-flash-text-mode.ts`.) NOTE: the `/api/tools/chat` route streams **markdown**, not a stop/scroll reaction — type-to-room needs the **reaction** route/runner (Flash text-mode), NOT the chat markdown route. This is a reaction (foresight), not conversational Q&A (that stays one tap deeper in the Lens's PersonaChatDrawer). No `ENGINE_VERSION` bump (text path, Qwen-only, reuses the shipped primitive).

### Surface 5 — One-Lens-many-doors continuity (D-05/D-06)

**Contract:** the per-card tap (Surface 3), the presence spotlight open (Surface 1/2), and the type-to-room result (Surface 4) ALL open the **same shipped `<AudienceLens>`**, scoped to their concept. Reuse `LensTrigger`'s mount pattern (it already constructs the Lens in cascade mode from `flatPersonas` + `conceptText`). Do NOT restyle, re-color, or fork the Lens — it is P9, already built (drill · chat-one-persona · cluster · Population · Rewrite). The only new thing is the **set of entry points**, all converging on the one component.

**Open transition + scoping:** identical across all three doors — the shipped bottom `Sheet` (`side="bottom"`, `max-h-[90vh]`, `rounded-t-[20px]`), `conceptText` set to the door's concept, `flatPersonas` = that concept's real reaction. Consistent so the creator learns one gesture → one depth.

## Cross-Cutting Contract

**Motion / reduced-motion (reuse, do not reinvent):**
- All presence/dot motion gates on a `reducedMotion` prop sourced from the OS preference (the shipped `<MotionConfig reducedMotion="user">` layer-1 + the `@media (prefers-reduced-motion: reduce)` layer-2 in `globals.css`).
- The presence dot pulse, the spotlight cross-fade, and the type-to-room loading shimmer all HARD-STOP under reduce — content/first-paint never depends on motion.
- Every motion surface ships an **always-present sr-only mirror** (the `PersonaCloud`/`PopulationSwarm` `<ul className="sr-only">` + `role="status"` pattern): the roster, the in-focus subject + its stop/total, and the type-to-room result are all readable by assistive tech regardless of motion state.
- Deterministic only — `mulberry32` seeded layout, NO `Math.random`, NO `Date.now` in render (SSR-hydration + engine-determinism-gate safe).

**Accessibility:**
- ≥44px tap targets on every interactive Ambient element (presence cue, expand toggle, input, send, promoted per-card cue) — `minHeight: 44` + `pointer-coarse:h-11`.
- Roles: the presence open-cue uses `role="button"` + `tabIndex={0}` + Enter/Space (the shipped `LensTrigger` pattern); the expand toggle uses `aria-expanded`; the type-to-room input is a labeled `<textarea>`/`<input>` with an accessible name; the loading + result + error use `aria-live` (`polite` for loading/result, `alert` for error).
- Subject label is real text (not an aria-only attribute) so the spotlight is never ambiguous (D-02).
- Focus is visible (reuse `focus-visible:ring-1 focus-visible:ring-accent` idiom; note: the ring is the one place accent-on-focus is allowed, consistent with the shipped CTAs).

**Honesty spine (binding — DESIGN constraint, not just engineering):**
- The presence **idles** (roster, no reaction) when no concept is in focus.
- It renders **exactly one labeled concept's** reaction at a time — never an aggregate/blend across cards.
- It **never fabricates** a reaction or a per-persona quote — only the real `fraction` + the real lead `scrollQuote` (via `cardScrollQuoteReactions`); other personas stay quote-empty.
- It **degrades to quiet** (cream em-dash / silence / collapsed cue) on thin signal — never to error-red, never to a fabricated crowd.
- The `reacting to: X` subject is a **required** affordance so the spotlight is never ambiguous.

**Engine posture (carried):** UI / text-path work only. No `ENGINE_VERSION` bump. Keep the engine + KC regression suites green. Preserve same-video SIM-1 Max score-identity. Qwen-only. Per-card reactions reuse already-emitted data (zero new model calls); type-to-room reuses the shipped Flash text-mode reaction primitive (one fast call per explicit submit).
