# Phase 10 — UI Design Contract

> Visual and interaction contract for Phase 10: Account Read, Saved Shelf & Recalibration Flywheel.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Design authority:** flat-warm SSOT (THEME-06, v5.0) is the design authority for ALL net-new UI in this phase. Tokens below are READ from `src/app/globals.css` (the locked SSOT) — they are NOT re-decided here. The Account Read card REUSES the fixed typed `reading/` renderers (treat as locked); this contract specifies how Account Read content maps onto them, not a new visual design. `BRAND-BIBLE.md` describes the older cold Raycast palette; where it conflicts with globals.css flat-warm tokens, **globals.css wins** (STATE Hard Constraint).

## Color

Read from globals.css flat-warm SSOT. 60/30/10 split:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-charcoal-app` #262624 | Page/app background, shelf surface |
| Secondary (30%) | `--color-charcoal-composer` #1e1d1b (cards) · `--color-charcoal-sidebar` #1a1a18 (nav) · `--color-charcoal-chip` #2f2e2b (chips/lifted) | Typed-item cards, nudge container, form surface, item-type chips |
| Accent (10%) | `--color-coral-500` (oklch 0.68 0.13 33, terracotta) | See reserved list below |
| Destructive | `--color-error` (oklch 0.60 0.20 25) | "Remove from shelf" confirmation only — distinct from coral (hue ~33 vs ~25) |

**Text:** `--color-cream-primary` #ece7de (titles/primary), `--color-cream-secondary` #c2bdb4 (default chrome/body), `--color-cream-muted` #8a857c (labels, timestamps, placeholders, em-dash empty). Never pure white.

**Borders:** `--color-border` rgba(255,255,255,0.06); hover `--color-border-hover` rgba(255,255,255,0.1). Card hover = `bg-white/[0.02]` only (no translate, no border change — CLAUDE.md).

**Accent reserved for (explicit — never "all interactive elements"):**
- Primary CTA buttons only (e.g. "Recalibrate", "Capture outcome" primary submit, shelf-item primary "Use in thread →" launch).
- Active nav-item indicator for the "Saved" sidebar entry.
- Focus rings on inputs (`--color-accent` focus ring).
- Links inside copy.

**Accent must NOT touch:**
- Item-type tags / chips (use `--color-charcoal-chip` + cream text — neutral, NOT coral; mirrors STATE 05-04 "checkmark uses cream-secondary, never coral").
- Provenance tags (public_scrape vs creator_supplied) — neutral cream/muted.
- The accuracy track-record number — uses cream-primary, NOT coral (it's data, not a CTA).
- Score-zone semantics inside Account Read — those are the existing `reading/` score-zone DATA colors (success/warning/error raw), governed by the fixed renderers, not chrome accent.

**Honesty / thin-history & drift states:** warning-toned `--color-warning`, NEVER `--color-error`, NEVER coral (mirrors STATE 07-05 "calibration fallback uses --color-warning, never coral, never error"). A thin-history empty state is a calm informational state, not an error.

## Copywriting Contract

> Honesty spine governs every string: NEVER fabricate, NEVER promise a view-count, NEVER imply auto-apply. The pull-score is a concept ceiling, not a prediction. Provenance is always disclosed (public vs creator-supplied).

### Recalibration PROPOSE nudge (FLYWHEEL-04) + Drift nudge (FLYWHEEL-06)

| Element | Copy |
|---------|------|
| Outcome-driven headline | `Your buyers ran warmer than we modeled.` |
| Outcome-driven body | `Across your last {N} posts, {disposition} showed up more than your audience predicted. Want to recalibrate "{audienceName}" to match?` |
| Drift-driven headline | `Your audience has shifted.` |
| Drift-driven body | `A fresh look at your account shows your audience mix has moved since we calibrated "{audienceName}". Recalibrate to match?` |
| Confirm CTA (primary, accent) | `Recalibrate` |
| Decline CTA (secondary) | `Not now` |
| Honesty footnote | `This only adjusts this audience — it never changes your scores or the General audience.` |
| Post-confirm success | `Recalibrated. Future Reads use the updated audience.` |
| Below-gate state | (no nudge rendered — never surface a proposal before the ≥N consistent-posts confidence gate passes) |

**Hard interaction rule (D-05, regression gate):** the nudge is ALWAYS propose→confirm. NO auto-apply, NO silent mutation, NO single-post trigger. The headline is the *surfaced* human-readable form; the underlying signal is the per-disposition signature delta. Decline = log `declined`, dismiss, do not re-nag same proposal.

### Primary CTA (phase-level)

`Recalibrate` (the moat moment — confidence-gated, creator-confirmed).

### Destructive actions in this phase

- **Remove from shelf** — confirmation dialog (copy above), destructive-colored confirm. The ONLY destructive action. (Recalibration is NOT destructive — it's an additive override the creator confirms.)

## Interaction Contracts

### Acts/State IA (D-07/D-08)
- **Acts run in the thread; State lives on surfaces.** Account Read is an Act (runs in thread) whose output is State (savable to shelf). Every thread output is savable-to-shelf; every shelf item is actionable-into-thread.
- **Save (Act→State):** thread card "Save" → `POST /api/saved` with `{item_type, ref_id, thread_id, snapshot}`. `snapshot` = the block's props so the shelf renders the SAME typed renderer without re-fetch. Affordance flips to `Saved ✓` (cream-secondary check, never coral).
- **Use (State→Act):** shelf item "Use in thread →" launches via existing `CHAIN_HANDOFFS` SSOT — per-type label (hook→Test full, idea→Develop). P12-extendable; P10 wires the minimal set.

### Shelf surface (flat, typed)
- Flat list/grid of typed-item cards. NO folders, NO tags, NO collection grouping (ROADMAP guard). Optional type-filter is a flat client-side filter chip row — NOT a folder structure.
- Each card: item-type tag (chip, neutral) + title + timestamp (muted) + `Use in thread →` (accent) + overflow `Remove`.
- Built P12-extendable: the flat `saved_items` shape must NOT bake in throwaway assumptions; P12 EXTENDS (separate watchlist/collection tables/columns), never reworks.

### Account Read → reading/ renderer mapping (NO new render)
- Compose `account-read-block` from existing `reading/` components (ReadingHero for account summary, accordions for pattern sections). Register in `BLOCK_REGISTRY`/`block-registry.ts` like `multi-audience-read` (closest precedent — static composed card). Fixed typed renderer; NO model-generated UI.
- Stream via the `calibrate/route.ts` SSE pattern (`status` → `done` / `fallback` / `error`).

### Recalibration / Drift nudge (one component, two triggers)
- Single nudge component; `source` discriminates outcome-driven vs drift-driven copy. Both run the SAME propose→confirm path (D-01 "folded in, not built separately").
- Surfaces only AFTER the confidence gate passes server-side (≥N consistent posts). Never renders speculatively.

### Outcome capture
- Low-friction: paste URL is the primary path; private signals are a collapsed optional section. Each metric carries a provenance tag. Blank field = excluded (never zero-filled).

## Accessibility & States

- **Loading:** SSE staged status text (`Reading your account…` / `Reading the post…`). Account Read may reuse `reading-skeleton.tsx`.
- **Empty:** shelf empty state + track-record empty state + thin-history fallback (all defined in Copywriting).
- **Error vs honest-degradation:** scrape/network failure = error tone; thin-history / not-enough-data = **warning tone, never error** (it's an honest limit, not a fault).
- **Focus:** accent focus ring on all inputs; keyboard-operable nudge confirm/decline.
- **Touch targets:** interactive controls ≥ 44px effective hit area (icon buttons may use padding to reach this — declared exception-free since spacing scale supports it).
- **Contrast:** cream-on-charcoal and accent-foreground (#1a0f0a on coral) verified ≥ AA per SSOT; do not introduce pure white.
