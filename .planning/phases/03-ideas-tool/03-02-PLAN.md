## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model output → persisted/rendered block | The Ideas model emits `idea-card` props that are persisted and rendered; unvalidated props would be a model-generated-UI / injection vector. |
| profile data → card copy | Profile fields flow to a user-visible grounding line; LLM-facing caveat prose or fabricated claims would breach the honesty spine. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | model-generated UI via idea-card | mitigate | Fixed registry + double `validateBlock` (write + rehydration); `IdeaCardRenderer` owns layout, model emits validated props only (D-10/THREAD-04). Invalid → unsupported sentinel, never executed. |
| T-03-05 | Spoofing/Repudiation | honesty spine on the grounding line | mitigate | `grounding-line.ts` renders only data-supported fragments; cold-start is an honest flag, never a fabricated "because your audience…" (D-09/D-14). Test asserts no caveat-prose leak. |
| T-03-06 | Repudiation | output provenance | mitigate | `KC_GEN_VERSION` stamp on persisted output gives reproducible provenance, decoupled from `ENGINE_VERSION` (D-06/kc-version.ts). |
