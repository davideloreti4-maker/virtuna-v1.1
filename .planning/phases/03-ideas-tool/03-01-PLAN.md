<objective>
Niche-instantiate the SIM-1 Flash TEXT path so the Ideas gate can actually say "no" (lever #10, D-05), recalibrate the band thresholds, and prove the gate works with a slop-vs-strong calibration test (D-06) — all as a TEXT-PATH-ONLY change that leaves the protected SIM-1 Max video path bit-identical (D-07).

This is the single highest-impact engine task in the phase: today the text path fires ONE generic call (`STABLE_FLASH_SYSTEM_PROMPT`, all 10 archetypes flat, no niche), which produced the flat 6/6/6/6/5 "all Mixed" distribution — a blindfolded judge. The video path already uses richer machinery (`selectPersonaSlots` → niche-instantiated personas + FYP/tough_crowd weighting). D-05 routes the text path through that same data WITHOUT fanning out to N calls: fold the 10 niche-instantiated slots into ONE Flash system prompt's archetype block (repetition of weighted slots encodes the ~30% tough_crowd weighting for free). The gate (Plan 03) is inert without this.

Purpose: make the Ideas viability gate trustworthy — the prerequisite for ENGINE-02.
Output: a niche-aware `runFlashTextMode` (back-compat default = today's generic behavior), recalibrated `STRONG`/`MIXED` thresholds, and the slop-vs-strong acceptance test.
</objective>
