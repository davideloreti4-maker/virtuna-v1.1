<objective>
Ship the `+ Build an audience` chooser (UX-04 / D-03 / D-08): one entry, three paths, all converging on a saved named General SIM in the library. Recommended form (UI-SPEC S3): a small centered Radix `Dialog` opened from the picker's `+ Build an audience` row (the `onBuildAudience` callback added in 07-02), hosted by the composer.

Three paths:
1. **From a description** → HOP to `/audience/new` with `mode:'general'` preset (D-08 — the deliberate one-time setup action reuses the full calibration form, not a duplicated mini-form); on save it returns to the composer with the new SIM selected.
2. **From evidence** → stays in-composer; routes to the existing P5 `profile-runner` / evidence-drop (reuse, do not rebuild).
3. **From a template** → lists `GENERAL_TEMPLATES` (analyst/hiring) → `cloneTemplateAudience` (07-03) into an owned, editable SIM, then selects it.

Purpose: fill the General library so Simulate/Predict have an audience to run against (§16.4); template = clone-and-edit (the moat object).
Output: `build-chooser.tsx` + composer host wiring + a chooser test.
</objective>
