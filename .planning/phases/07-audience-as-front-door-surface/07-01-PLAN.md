<objective>
Make the skill menu Mode-scoped (UX-02 / D-01): add a per-skill `modes[]` tag to `SkillMeta`, tag every existing skill `['socials']`, add the three General entries (Profile / Simulate / Predict) tagged `['general']`, and filter `SkillRows` by the active Audience's mode BEFORE the existing Creator/Marketing group partition. This removes the one hardcoded `creator`/`marketing` site that never read the audience, and keeps the Socials/creator render byte-identical (the #1 constraint).

This plan touches ONLY `composer-controls.tsx` (+ its test). The host wiring that passes the real `activeMode` and the per-skill submit semantics (D-07) are the separate Wave-2 plan 07-04; here the new `activeMode` prop defaults to `'socials'`, so the live composer is unchanged until 07-04 threads it.

Purpose: remove the hardcoded-Socials skill fork (the door-open seam, D-01) without a skill-registry refactor.
Output: a mode-gated `SKILLS`/`SkillRows` + the three General skill rows + updated guard test.
</objective>
