# Phase 1 — UI Design Contract

> Visual and interaction contract for the Numen Surface ground-up warm-neutral dark kit. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Direction is LOCKED** by `NUMEN-SURFACE-VISION.md` §6 and `01-CONTEXT.md` (D-01…D-14). This contract records the HOW + calibration defaults; it does NOT relitigate direction. Palette hexes below are the D-11 calibration **proposal** — the user retains final sign-off during planning; the APCA script (`scripts/check-apca.ts`, D-12) is the pass/fail gate on whatever is chosen.
>
> **Scope note:** This kit is a *parallel, namespaced layer* (`.numen-surface` scope class, D-01/D-02). It coexists with the live coral/Raycast app untouched. Nothing live is deleted in Phase 1 (D-04). This contract governs the NEW kit only — the OLD `BRAND-BIBLE.md` Raycast language is the migration-FROM reference, NOT a source for the new kit.

## Spacing Scale

8-point scale (multiples of 4). The existing repo scale (`globals.css`) is reused as-is — the warm-neutral kit changes *color/type/motion*, not the spacing geometry.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, pill chip vertical padding |
| md | 16px | Default element spacing, block padding |
| lg | 24px | Section padding, between Reading blocks |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Intermediate steps available in the existing scale (12, 20, 40, 80, 96px) may be used where a multiple-of-4 step is needed; do not author off-grid values.

**Radius scale (reused from repo):** xs 4 / sm 6 / md 8 / lg 12 / xl 16 / 2xl 20 / 3xl 24 / full 9999px. Defaults for the new vocabulary: **full-pill tool chips = `full`**, **circular icon buttons = `full`**, **soft-elevation surfaces / glass = `lg` (12px)**.

Exceptions: Circular icon buttons must meet a **44px minimum touch target** (mobile-first surface, accessibility) even when the visible icon is smaller — pad to 44px hit area.

## Typography

Sans-led (Inter) for all functional/data UI; serif (Source Serif 4) reserved for **voice moments only** — the greeting/hero and the verdict line (D-04). Two functional weights: regular (400) + semibold (600). The serif voice may use a heavier display weight (600–700) at large optical size only.

| Role | Size | Weight | Line Height | Family |
|------|------|--------|-------------|--------|
| Body | 16px | 400 | 1.5 | Inter (sans) |
| Label / UI | 14px | 600 | 1.4 | Inter (sans) |
| Heading | 20px | 600 | 1.3 | Inter (sans) |
| Voice / Display | 28px | 600 | 1.2 | Source Serif 4 (serif), `font-optical-sizing: auto` |

**Serif is reserved for:** the greeting/hero line and the verdict one-liner. It must NOT appear on data labels, buttons, metrics, or block bodies — surfacing serif on functional UI breaks the sans-led contract (D-04).

**APCA contrast gate (D-12)** — measured by `scripts/check-apca.ts` against the base `#1a1714`; final pass/fail set during calibration:
- Body / fluent text (≤16px / 400): **Lc ≥ 90** preferred, **75** floor
- Larger body (≥18px / 400): **Lc ≥ 75**
- Non-body UI labels: **Lc ≥ 60**
- Large/heavy (≥24px bold / ≥36px 400 — headlines, verdict line): **Lc ≥ 45**

## Color

**Core principle (DS-02 + DS-08):** the chrome is near-neutral warm dark so it *recedes*; the only load-bearing functional color is the **verdict scale**, and the user's video **keyframes are the imagery/chroma**. Brand accent (clay) is used **sparingly** — logo, primary action, focus only. No pure black anywhere.

> Hex values below are the **D-11 calibration proposal** (placeholders pending user sign-off + APCA validation). Direction (warm-neutral / muted verdict scale / matured-clay accent) is locked; exact hues are calibration, not direction.

| Role | Value (proposed) | Usage |
|------|------------------|-------|
| Dominant (60%) — `--numen-bg` | `#1a1714` | App background, warm near-black base (no pure black) |
| Secondary (30%) — `--numen-panel` / `--numen-panel-2` | `#211e1a` / `#2a2622` | Cards, panels, sidebar, nav — stepped warmer |
| Text — `--numen-text` / `--numen-text-muted` | `#f0ebe3` / `#a39c91` | Warm off-white body; warm-grey muted (APCA-tuned) |
| Border — `--numen-border` | `rgba(240,235,227,0.06)` | Hairline warm borders |
| Accent (10%) — `--numen-accent` | `#d98a5e` (clay, matured from `#FF7F50`) | **Reserved list below** |
| Verdict good — `--numen-verdict-good` | `#7faf7a` (muted green) | Positive band only |
| Verdict mixed — `--numen-verdict-mixed` | `#d6a85a` (amber) | "Mixed signals" band — a first-class, common verdict |
| Verdict bad — `--numen-verdict-bad` | `#c97a64` (clay-red) | Negative band only |

**Accent (clay) reserved for — and ONLY for:**
1. The logo / brand mark
2. The single primary action (one per view)
3. Focus rings / focus-visible state

Accent must NOT be used as a generic interactive color, on every button, on links in body text, on hover fills, or on decorative surfaces. Near-neutral is the default; color is the exception (DS-02).

**Verdict scale is the only load-bearing functional color.** It carries meaning (band = good / mixed / bad). It is muted, not saturated — calm, not alarming. The amber "mixed" state is first-class and common (GATE-02 principle), never styled as a degraded/error state.

**Keyframes carry the chroma (DS-08):** warm-neutral chrome recedes deliberately so the (often cool) user video stills + the verdict carry all the color and energy. This is verifiable in the showcase — chrome tokens must read near-neutral beside a sample keyframe.

**Destructive:** No destructive/data-mutating actions exist in Phase 1 (kit + audit + showcase only — see Copywriting). When destructive UI is introduced in a later phase, use `--numen-verdict-bad` (clay-red) as the destructive semantic; do not introduce a separate red.

**Anti-patterns (hard rules, D-07):** NO neon, gradient, beam, glow, or shimmer ships — including the OLD `globals.css` `--animate-shimmer` / `gradient-x` keyframes. These are the "AI-spaceship" aesthetic the vision positions against. Do not carry them into the Numen kit.

## Motion

Calm motion system (DS-07 / D-14). Semantic duration tiers + easing families; the **stage-reveal is the one key motion moment** — no "presence" theater elsewhere.

| Property | Value |
|----------|-------|
| Duration tiers | instant (~0ms), fast (~150–200ms), base (~300–400ms), slow (~500–600ms) — exact numerics are planner's discretion within these tiers |
| Opacity easing | tween cubic-bezier (e.g. `[0.215, 0.61, 0.355, 1]`) — calm, never bouncy |
| Physical-prop easing | spring ONLY, **damping ratio ≥ 1** (critically/over-damped → no overshoot, no bounce) |
| Forbidden | the OLD `--ease-spring: cubic-bezier(0.34,1.56,0.64,1)` token — the `1.56 > 1` overshoots = the bounce §6 forbids. Define a NEW calm easing token. |
| Reduced motion | **`prefers-reduced-motion` MUST be honored** (accessibility, not optional). Via Motion's `useReducedMotion()`, the stage-reveal degrades to a static opacity appear — no slide/translate when reduced. |

## Copywriting Contract

Phase 1 ships **tokens + core primitives + a kit showcase route** (D-05/D-06) — there is no end-user product surface, no data CRUD, and no destructive action in this phase. Copy below governs the **showcase route** (the deployed-build verification surface) and sets the **voice baseline** every later phase inherits: calm, plain-language, NO engine jargon, confident-mentor tone (READ-07 anticipated; VISION §5).

| Element | Copy |
|---------|------|
| Primary CTA (showcase) | "View component" / "Toggle reduced motion" — utilitarian; the showcase has no product CTA. The kit's *canonical* primary-action label pattern for later phases = **verb + noun** (e.g. "Analyze video"), never "Submit"/"Go". |
| Empty state heading | "Nothing here yet" |
| Empty state body | "This is a kit showcase — components render below under the Numen surface tokens." (later product empty states inherit the calm, no-blame voice: state the situation + the one next step) |
| Error state | "Couldn't load this." + the recovery path. Voice rule (inherited from TOOL-05): failures are voiced calmly in-persona — **NEVER a red error toast/modal**. No engine jargon, no stack detail surfaced. |
| Destructive confirmation | none in Phase 1 — no destructive actions exist. Pattern for later phases: name the action + consequence ("Delete this Reading? This can't be undone."), confirm in-context, use `--numen-verdict-bad`. |

**Voice baseline (load-bearing for all later phases):**
- Plain language, no engine jargon ever surfaced to the user (READ-07).
- Calm restraint — confident mentor with weight, never hype, never alarmist.
- No mysticism / oracle theater (explicitly cut — VISION out-of-scope).
