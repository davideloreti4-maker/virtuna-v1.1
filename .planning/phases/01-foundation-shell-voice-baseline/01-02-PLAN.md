| Boundary | Description |
|----------|-------------|
| page → external sites (X, LinkedIn social, placeholder legal) | Outbound `target="_blank"` links open attacker-controllable destinations in a new tab |
| client browser → DOM (nav menu) | Mobile menu mutates `document.body.style` + attaches a global listener |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Tampering | footer.tsx external links (X/LinkedIn/legal) | mitigate | All external `target="_blank"` links carry `rel="noopener noreferrer"` — blocks reverse-tabnabbing (the opened page cannot reach `window.opener`). Verified by grep gate. |
| T-01-04 | Denial of Service | nav.tsx body scroll-lock + global mousedown listener | mitigate | `useEffect` cleanup resets `document.body.style.overflow` and removes the `mousedown` listener on unmount/close — no leaked lock or dangling listener. |
| T-01-05 | Information Disclosure | nav/footer copy + placeholder hrefs | accept | All hrefs are placeholders (`#cta`, `#`) or public social URLs; no secrets, tokens, or PII rendered. |
| T-01-SC | Tampering | npm/pip/cargo installs | n/a | No packages installed (all deps pre-exist; RESEARCH §Package Legitimacy Audit N/A). |

V5 Input Validation: not applicable — no user input (no forms/CTAs wired this phase; placeholder links only). V2/V3/V4/V6: not applicable — no auth/session/access/crypto.
