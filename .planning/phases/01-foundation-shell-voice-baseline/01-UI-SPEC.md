# Phase 1 — UI Design Contract

> Visual and interaction contract for the Numen landing **shell + scaffold**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** top nav, footer, kero-ordered section-rhythm scaffold (EMPTY skeleton slots), page chrome. NOT the hero artifact, verdict bands, explainer, gallery, proof, or conversion UI (Phases 2–4). NOT scroll-reveal motion (Phase 4 / MOT-01). This phase's motion contract is **section rhythm/pacing only** (MOT-02).
>
> **Token posture:** consumes the EXISTING `src/app/globals.css` `.numen-surface` scope as placeholder tokens (DS-01/DS-02/D-09). All color is by **bridged token NAME** (`bg-bg`, `text-text`, …) so the Phase-4 hex swap needs zero JSX edits. NO forked or invented tokens.

## Spacing Scale

The `.numen-surface` scope declares **no custom spacing tokens** — use Tailwind's default 4px scale (RESEARCH §MOT-02). Declared subset (all multiples of 4):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | hairline inline gaps |
| sm | 8px | `gap-2` | logo↔wordmark, icon↔label |
| md | 16px | `gap-4` / `px-4` | default element spacing; **mobile horizontal gutter** |
| lg | 24px | `gap-6` / `px-6` | **desktop horizontal gutter**; nav inner padding |
| xl | 32px | `gap-8` | nav link cluster spacing; footer column gaps |
| 2xl | 48px | `py-12` | nav vertical height band; footer block padding |
| 3xl | 64px | `py-16` | minor section break |

### Vertical rhythm (the kero spine — MOT-02 core contract)

This is the load-bearing spacing contract for the scaffold. Each `<section>` carries **uniform vertical padding** so the page paces like kero regardless of inner content (which Phases 2–4 add).

| Element | Mobile | Desktop | Token |
|---------|--------|---------|-------|
| Section vertical padding (`py`) | 96px | 128px | `py-24 md:py-32` |
| Hero section vertical padding | 112px top / 96px bottom | 160px top / 128px bottom | `pt-28 pb-24 md:pt-40 md:pb-32` (clears sticky nav) |
| Section heading → body gap | 24px | 32px | `mt-6 md:mt-8` |
| Inter-section spacing | (handled by each section's own `py`) | — | no extra margin between sections; padding does the rhythm |
| Nav bar height | 56px | 64px | `h-14 md:h-16` |

### Container & gutters (page chrome)

| Property | Value | Tailwind |
|----------|-------|----------|
| Max content width | 1152px | `max-w-6xl` |
| Centering | auto margins | `mx-auto` |
| Horizontal gutter — mobile | 16px | `px-4` |
| Horizontal gutter — desktop | 24px | `px-6` |

**One shared inner container** (`mx-auto max-w-6xl px-4 md:px-6`) is reused by nav, footer, and every section so left/right edges align down the whole page. Nav/footer may run full-bleed background with the container only constraining their inner content.

Exceptions: none (no 44px-touch-target exception needed in spacing — hit targets are enforced via min-height on interactive elements, see Accessibility).

## Typography

Single typeface (Inter). **Exactly 4 sizes, 2 weights** for the shell. Mapped to the `font-[var]` wiring already in `layout.tsx`.

| Role | Size (mobile → desktop) | Weight | Line Height | Tracking | Tailwind |
|------|------------------------|--------|-------------|----------|----------|
| Section heading (`<h2>`) | 30px → 36px | 700 (bold) | 1.2 | -0.02em | `text-3xl md:text-4xl font-bold tracking-tight` |
| Subhead / lead body | 16px → 18px | 400 (regular) | 1.5 | normal | `text-base md:text-lg leading-relaxed` |
| Body / footer copy | 16px | 400 (regular) | 1.5 | normal | `text-base leading-relaxed` |
| Nav label / footer link / legal | 14px | 400 (regular; 500 on hover/active optional) | 1.4 | normal | `text-sm` |

**Weights: exactly 2 — regular (400) + bold (700).** The `NumenLogo` wordmark internally renders `font-bold tracking-[-0.02em]` at 17px — leave it as-is (locked geometry), do not restyle.

**Hero H1** (placeholder copy this phase, real treatment Phase 2): render at `text-4xl md:text-6xl font-bold tracking-tight leading-[1.1]`. The placeholder H1 string is a voice deliverable (below), but its type treatment is set here so the scaffold paces correctly. Do NOT apply the reserved serif.

Headings use `text-text`; subhead/lead and muted UI copy use `text-text-muted`; body uses `text-text`.

## Color

All values by **bridged token NAME only** (DS-01). Hexes shown are the current placeholder values from `globals.css:357–366` for reference — Phase 4 swaps them; **never hardcode the hex in JSX**.

| Role | Token (utility) | Placeholder hex | Usage |
|------|-----------------|-----------------|-------|
| Dominant (~60%) | `bg-bg` | `#1a1714` | page background, nav background, footer background |
| Secondary (~30%) | `bg-panel` / `bg-panel-2` | `#211e1a` / `#2a2622` | (sparing in P1) mobile-menu sheet surface, any raised slot container |
| Text | `text-text` | `#f0ebe3` | headings, body, active nav label, logo color |
| Muted text | `text-text-muted` | `#bab2a5` | subheads, inactive nav labels, footer links, legal, copyright |
| Border | `border-border` | `rgba(240,235,227,0.06)` | hairline under sticky nav, footer top rule, mobile-menu divider |
| Accent (~10%) | `bg-accent` / `text-accent` | `#d98a5e` (clay) | **CTA button fill only** (see reserved-for) |

**Accent reserved for:** the **primary CTA button** in the nav (right) and the **footer CTA slot** — and `focus-visible` ring color. NOTHING else. No accent on nav links, headings, borders, logo, or icons. Clay is the single 10% note on an otherwise warm-neutral page (anti-spaceship discipline).

CTA button: `bg-accent text-bg` (dark-brown-on-clay reads correctly; `text-bg` = the warm near-black, matching the in-app `accent-foreground` intent). Hover: subtle brightness via opacity, NOT a new token.

**Verdict tokens** (`bg-verdict-good/mixed/bad`): declared in scope but **OUT OF SCOPE for Phase 1** — first use is the verdict band in Phase 2/3. Do not consume them in the shell.

Destructive: none — Phase 1 has no destructive actions.

### Glass / blur posture (Lightning CSS landmine)

The minimal nav is **opaque** (`bg-bg` + hairline `border-border` bottom), so the `backdrop-filter` strip-bug does NOT apply. **If** a translucent sticky bar is ever chosen, blur MUST be applied via React inline `style={{ backdropFilter: 'blur(Xpx)', WebkitBackdropFilter: 'blur(Xpx)' }}` — NEVER a Tailwind/CSS class (Lightning CSS strips it in prod). The `Glass` primitive is the composer/tool-sheet blur and is NOT for the nav. **Recommended: opaque nav** — simplest, avoids the landmine entirely.

## Nav + Footer Layout Contract

### Top nav (`src/components/numen-landing/nav.tsx`) — `"use client"` (mobile menu needs state)

**Desktop (≥768px):**
- Layout: full-bleed bar, inner `mx-auto max-w-6xl px-6`, `h-16`, `flex items-center justify-between`.
- Left: `<NumenLogo />` (default `wordmark="numen"`, `text-text`), links to `/` / `#hero`.
- Center/left-adjacent: inline anchor links — **How it works** (`#how-it-works`) · **Honesty** (`#honesty`) · **Gallery** (`#gallery`). `text-sm text-text-muted`, hover → `text-text`. `gap-8`.
- Right: primary CTA button (`bg-accent text-bg`, `text-sm font-medium`, `h-10 px-4 rounded-lg`). Placeholder label below; placeholder `href="#cta"` this phase (live wiring = CTA-01, Phase 2).

**Mobile (<768px):**
- Layout: `<NumenLogo />` (left) + primary CTA + hamburger toggle inline. Anchor links collapse into the hamburger.
- Hamburger: Lucide `Menu` (closed) / `X` (open). Rendered in a ≥44px hit target (`IconButton` from `numen/icon-button` is valid — it enforces 44px + requires `aria-label` — OR a plain `<button>` with `min-h-11 min-w-11`). MUST have `aria-label="Open menu"` / `"Close menu"` + `aria-expanded`.
- Slide-down menu pattern (reference: stale `layout/header.tsx`, rebuilt clean with `.numen-surface` tokens + Lucide):
  - Panel surface: `bg-panel` (or `bg-bg`) with `border-border` top divider.
  - Reveal: `max-h-0 opacity-0` → `max-h-[appropriate] opacity-100`, `transition-all duration-200 ease-out` (use `--numen-ease-calm` if expressed as a custom timing; otherwise `ease-out` is fine for this phase).
  - **Body scroll-lock** on open (`document.body.style.overflow = 'hidden'`), reset on close + on unmount cleanup.
  - **Outside-click close** via a `useRef` + `mousedown` listener, OR a full-screen dimming overlay `fixed inset-0 z-40 bg-black/50` that closes on click.
  - Each menu anchor: ≥44px row height (`min-h-11 flex items-center`), `text-text`, full-width tap target.
  - Closing the menu (anchor click or `X`) MUST restore scroll.

**Sticky decision:** nav is **`sticky top-0 z-50`** with `bg-bg` + `border-b border-border`. Opaque, so no blur landmine. This is why hero top padding (`pt-28 md:pt-40`) and `scroll-margin-top` exist — to clear the sticky bar on anchor jumps.

### Anchor / smooth-scroll contract

- **Smooth scroll:** `scroll-behavior: smooth` on `<html>` (respect `prefers-reduced-motion: reduce` → `auto`; gate the smooth rule in a media query).
- **Scroll-margin:** every `<section id>` gets `scroll-mt-20 md:scroll-mt-24` (80/96px) so the sticky-nav offset doesn't hide the heading on jump.
- **Anchor `id` naming convention (locked — Phases 2–3 fill these without renaming):** kebab-case, semantic. Final set: `#hero`, `#how-it-works`, `#honesty`, `#gallery`, `#proof`, `#cta`. (Footer has no `id`; it is the `<footer>` landmark.)

### Footer (`src/components/numen-landing/footer.tsx`) — server component (static, no interactivity)

- `<footer>` landmark, full-bleed `bg-bg`, `border-t border-border`, inner `mx-auto max-w-6xl px-4 md:px-6 py-12`.
- Contents (minimal, D-07):
  1. `<NumenLogo />` + one-line positioning copy (in voice — string below), `text-text-muted text-sm`.
  2. Anchor repeat (same 3 anchors as nav) — `text-sm text-text-muted` hover `text-text`.
  3. Legal placeholders: **Privacy** · **Terms** — placeholder `href="#"` OK this phase, `text-sm text-text-muted`.
  4. Social links: **X**, **LinkedIn** — external, `target="_blank" rel="noopener noreferrer"` (reverse-tabnabbing mitigation, required). Placeholder `href` OK.
  5. Footer **CTA slot**: same accent button treatment as nav CTA, placeholder label + `href="#cta"`. (Live wiring = CTA-02, Phase 3.)
  6. Copyright line: `© 2026 Numen` (or `Numen Machines`), `text-text-muted text-sm`.
- Mobile: single column stacked; desktop: logo+positioning left, link/social columns right (`md:flex md:justify-between` or a simple grid). Keep it minimal — no newsletter, no sitemap sprawl.

## Accessibility Baseline

- **Contrast / APCA:** inherited from the `.numen-surface` token system — tokens are APCA-gated upstream (`--numen-text-muted` lifted to Lc ≥ 60, verdict tokens Lc ≥ 45 per `globals.css` D-11). Do NOT introduce off-token text colors that would break the inherited contrast guarantee. Body/heading on `bg-bg` use `text-text`; muted UI uses `text-text-muted` (the contrast-locked muted, never a lighter ad-hoc grey).
- **Focus-visible:** every interactive element (nav links, CTA, hamburger, footer links) gets a visible `focus-visible` ring using the accent token — e.g. `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-bg`. Never remove focus outlines without a replacement.
- **Hit targets:** all tappable elements ≥44×44px — enforce on the hamburger toggle and every mobile menu row (`min-h-11`). `IconButton` already guarantees this.
- **Semantic landmarks:** exactly one `<nav>` (top nav), one `<main>` (wraps the section slots, in the marketing layout), one `<footer>`. Each scaffold slot is a `<section id>` with an `<h2>`; the hero `<h1>` is the single page-level h1. Heading order: h1 (hero) → h2 (each section), no skips.
- **External links:** `target="_blank" rel="noopener noreferrer"` on X / LinkedIn (and placeholder legal if external).
- **Reduced-motion posture (Phase 1 = static-first):** Phase 1 has NO entrance/scroll animation (MOT-01 is Phase 4). The only motion is the mobile-menu slide (200ms) and anchor smooth-scroll. Gate smooth-scroll behind `@media (prefers-reduced-motion: no-preference)`; the menu slide is short and acceptable, but SHOULD collapse to an instant show/hide under `prefers-reduced-motion: reduce`. Confirm: no parallax, no auto-playing media, no presence theater this phase.
