>

## Typography

Inter throughout. Declared roles for these surfaces:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px (`text-2xl`) | semibold (600) | 1.0 (`leading-none`) | the band WORD on the panel-variant reaction card (mirrors `band-block`) |
| Heading | 15px (`text-[15px]`) | semibold (600) | 1.2 (tight) | card lead line вЂ” subject name, verdict line, section heads |
| Body | 14px (`text-sm`) | regular (400) | 1.5 (`leading-relaxed`) | reasoning, how-they-react, list items, evidence prose |
| Label | 10px (`text-[10px]`) | medium (500) | 1.2 | quiet UPPERCASE section labels, `tracking-[0.14em]`, `text-foreground-muted` |

Secondary / inherited sizes (do not introduce new ones beyond these): `13px` (`text-[13px]`) inside dense
read panels, `12px` (`text-xs`) for provenance/meta + the honesty caveat, `16px` quotes only if a quote needs
emphasis (default quotes are 14px italic).

**Weights вЂ” exactly two are load-bearing for this phase:** regular (400) for body, semibold (600) for emphasis.
`medium` (500) is the inherited interactive-chrome weight (buttons, labels, badges) carried from the design
system вЂ” it is not a new declaration, reuse as-is.

**Emphasis without color (LOCKED rule):** hierarchy comes from weight, size, italic (quotes), and spacing вЂ”
never from paint. No accent-colored headings, no accent CTA text.

## Color

60 / 30 / 10 split, mapped to the flat-warm charcoal tokens:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#262624` (`--color-background` / charcoal-app) | thread canvas behind the cards |
| Secondary (30%) | `#1e1d1b` (`--color-surface` / `--color-surface-thread`) + `white/[0.02]вЂ“[0.04]` lifts | the card surfaces, persona-drill panels, footer rows |
| Accent (10%) | `#d97757` (`--color-accent`, terracotta) | **see reserved-for вЂ” effectively ZERO on these two cards** |
| Destructive | `var(--color-error)` (`oklch(0.60 0.20 25)`, distinct from terracotta hue ~33) | error states, refusal banner, "scrolls/resistant" Weak verdict text |

**Accent reserved for (explicit):**
- **Neither card uses the terracotta accent.** Per the LOCKED near-zero-dosage rule (`DESIGN-SYSTEM.md` В§"Accent
  dosage"), a screen with zero accent is the norm. Primary CTA = neutral cream text. Save check = cream. Bands =
  semantic data tokens (below), which are **not** the accent.
- The single sanctioned exception if ever needed: ONE liveness/presence dot (e.g. a "now in your library" pulse).
  Default = omit it. Do not reach for accent to make the wow "pop" вЂ” that instinct is the thing being removed.

**Band / data colors (sanctioned, NOT accent)** вЂ” reuse the EXACT `BAND_COLOR` map from `band-block.tsx`, never re-roll, never coral:
- Strong в†’ `var(--color-success)` В· Mixed в†’ `var(--color-warning)` В· Weak в†’ `var(--color-error)`
- Deception likelihood (forensic, Max only): Low в†’ `--color-success` В· Medium в†’ `--color-warning` В· High в†’ `--color-error`. **Band WORD only, never a number.**

**Borders / depth:** rest `white/[0.06]` (`--color-border`), hover `white/[0.10]`. Matte only вЂ” no glass gradient,
no glow, no inset white-shine. Cards are flat (solid charcoal + hairline border). `--shadow-float` is for floating
surfaces only (the composer / its drop overlay), not the result cards. Radius: cards 12px (or 18px when reusing
`ReadingSection`'s `READING_CARD` shell), inputs/buttons 8px, badges/pills full-round.

**вљ пёЏ Divergence from the closest analog (recorded, intentional):** `multi-audience-read-block.tsx` paints its
`.read` panel with a LEGACY coral `rgba(255,127,80,0.30)` border (`READ_PANEL_STYLE`). That predates the LOCKED
dosage rule and is stale. **Do NOT copy it.** The `profile-read` "how they react" / Read panel uses a NEUTRAL
treatment (6% hairline + `white/[0.04]` lift + semibold lead) вЂ” zero accent. (Flag F-03.)

### Surface 1 вЂ” `profile-read` card (hero, PROF-02)

Closest analog: `account-read-block.tsx` (a per-entity Read composed of `ReadingSection` units). The card is a
**vertical stack of `ReadingSection` units** (`gap-5`), STATIC (no model-generated UI), savable, with a footer
forward-chain CTA.

Section order (top в†’ bottom):
1. **Hero / identity header** вЂ” `ReadingSection` label `A read on this person` (or `вЂ¦this group` when `subjectKind==="panel"`).
   Body: `subjectName` at Heading (15/600); a quiet `subjectKind` chip (`person` / `panel`) using `--color-surface-elevated`;
   the `SIM-1 Flash|Max` tag + `TrustBadge "Directional"` right-aligned. Traits as cream chips, `commStyle` one line,
   `drivers` as a short list. Identity is factual, neutral вЂ” no accent.
2. **Behavioral tells (evidence-quoted)** вЂ” label `What gives them away`. Each `tells[]` entry = a tell line (14/regular,
   `text-foreground-secondary`) with its verbatim `evidence` quote directly beneath, italic 14 muted (`"вЂ¦"`). The quote is
   the PROVENANCE / trust mechanism (TRUST-02) вЂ” it must be visually bound to its claim, never floated separately.
3. **How they'll react (goal-scoped)** вЂ” label `How they'll react`. `howTheyReact` prose; a quiet `goalScope` line above
   it framing the target (`On your goal: {goalScope}`). NEUTRAL panel (6% border + `white/[0.04]`), NOT the legacy coral panel.
4. **Deeper forensic read (Max/video ONLY)** вЂ” label `Deeper read (from the video)`. Rendered **only when `forensic` is
   present** (null/absent on Flash в†’ section omitted entirely; never a teaser, never "upgrade to see"). `deceptionLikelihood`
   as a band WORD (Low/Medium/High) in the `BAND_COLOR` token, never a number. `cues[]` as rows: `{timestamp}` (tabular,
   muted) В· `{observation}` в†’ `{inference}`. Recommend this section start **collapsed** behind a disclosure to keep the
   hero calm and avoid over-claim (Flag F-01).
5. **Honesty caveat** вЂ” the D-04 caveat line, quiet 12px muted, always visible (not behind a tooltip).
6. **Footer action row** вЂ” top hairline (`border-t white/[0.06]`, `px-4 py-3`, `gap-4`): `SaveAffordance` (cream) on the
   left, the primary `Simulate a message to {name} в†’` forward-chain CTA as neutral cream-secondary text (mirror the
   `idea-card` "Develop this в†’" button exactly вЂ” text button, not a filled coral button), carrying `savedAudienceId`.

### Surface 2 вЂ” `reaction-distribution` card (SIMU-02), two variants on `subjectKind`

Closest analog: `multi-audience-read-block.tsx` (per-audience read) minus the 2-audience delta; `band-block.tsx` for the
band glyph/word; `personas-block.tsx` for the collapsible drill. **Branch the WHOLE layout on `subjectKind`** вЂ” never
show a fraction or a crowd for a person.

**Person variant (`subjectKind==="person"`, `read` present, `band`/`fraction` suppressed):**
- A single read. Verdict line: `{audienceName} is likely to be {read.verdict}` (Heading 15). The verdict word may borrow
  the success/warning/error band tone (receptiveв†’success, on-the-fenceв†’warning, resistantв†’error) вЂ” data color only.
- `read.reasoning` prose (14/1.5). ONE representative `read.quote` italic 14 muted. **No `N/10`, no panel grid, no glyph
  crowd** (Pitfall 2 вЂ” "distribution of one" is forbidden).

**Panel variant (`subjectKind==="panel"`, `band`+`fraction`+`themes`+`reactions` present):**
- Verdict header: `{band}` WORD (Display 24 or Heading 15, `BAND_COLOR`) + `{fraction}` (e.g. `6/10 react`, muted) вЂ” reuse
  `band-block` styling exactly; fraction comes from `aggregateFlash`, never re-rolled.
- **Clustered themes** вЂ” `themes[]` as labeled rows: `{label}` (semibold 14) + a representative `{quote}` italic muted.
  This is the panel's proof unit.
- **Per-persona drill** вЂ” collapsible (reuse the `personas-block` / `multi-audience-read` "Audience reactions вЂ” {stop}/{total}"
  toggle pattern): each `reactions[]` row = archetype (capitalized) + stop/scroll verdict badge + first-person quote.
- Footer: provenance (`SIM-1 Flash|Max` + `TrustBadge "Directional"`) + `SaveAffordance`.

Both variants: bands/words only, ZERO 0-100 number (the `.strict()` schema enforces it; the UI must not invent a meter
or gauge either), zero accent.

### Surface 3 вЂ” additive composer affordance (D-07, PROJECT.md locked)

Closest reference: `src/components/app/home/composer.tsx` + `composer-controls.tsx`. **ADDITIVE ONLY вЂ” wrap, never
restructure.** The creator (Socials) composer stays byte-identical.

- An **attach / drop control**: a single icon-button (Phosphor `Paperclip` or `Plus`, cream-muted chrome) mounted
  alongside existing composer controls вЂ” NOT a new row, NOT a redesign. `aria-label="Attach a chat or screenshot"`.
- A **drag-and-drop overlay** over the existing composer surface that appears only on drag-over: matte `--color-surface`
  panel + `--shadow-float` + 6% dashed-ish hairline, copy `Drop a chat export, screenshot, or short clip`. Dismisses on
  drop/leave.
- Accepts `.txt`/`.md`, image, short video ONLY (D-09). Unsupported type в†’ inline muted error (copy above); never a
  blocking modal.
- Selected-attachment chip: a compact removable chip (filename + `Г—`) in the composer, `--color-surface-elevated`,
  cream text, neutral. No accent.
- Touch target в‰Ґ 44px for the icon-button (accessibility exception to the spacing scale).

## Accessibility

- All bands/verdicts carry a text WORD, never color-only (Strong/Mixed/Weak, Low/Medium/High) вЂ” colorblind-safe by construction.
- Evidence quotes use real `"вЂ¦"` and are programmatically associated with their tell (same list item).
- Forensic `cues[]` timestamps are `tabular-nums`, read as `at {timestamp}, {observation}, suggesting {inference}` for SR.
- Interactive controls: visible focus ring (`focus-visible:ring-1 ring-white/10`), в‰Ґ44px touch target on the attach button.
- Card sections use `aria-label` summarizing band + fraction + tier (mirror `band-block` `aria-label`).

## Non-Blocking Flags (grounded recommendations)

- **F-01 вЂ” Forensic layer honesty (deception likelihood).** Render as a band WORD (Low/Medium/High) in the
  success/warning/error tokens, NEVER a percentage or number (avoids false precision; matches bands-only honesty).
  Per-cue rows: `{timestamp} В· {observation} в†’ {inference}`. **Recommend collapsed-by-default** behind a disclosure so the
  hero stays calm and the heaviest claims require a deliberate open. Max-tier only; omit the section entirely on Flash.
- **F-02 вЂ” Person vs panel switch.** Branch the entire layout on `subjectKind`. Person = single read (verdict word +
  reasoning + one quote, NO fraction/crowd). Panel = band word + fraction + clustered themes + collapsible drill. This is
  the D-02 honesty line; do not soft-merge the two into one template.
- **F-03 вЂ” Reject the legacy coral Read panel.** The closest analog (`multi-audience-read-block`) uses a stale coral
  border that violates the LOCKED near-zero-dosage rule. The new `profile-read` "how they react" panel must use NEUTRAL
  emphasis (6% hairline + `white/[0.04]` + semibold lead). Keep `reskin-matte.test.ts` green; do not introduce any
  `rgba(255,127,80,*)` / `#FF7F50` / coral literals.
- **F-04 вЂ” Caveat is always visible.** The D-04 honesty caveat is product copy, not a tooltip. Render it inline, quiet,
  on every `profile-read` (and as a one-line directional note on the simulate card). It is part of the trust contract.
- **F-05 вЂ” Composer affordance stays minimal.** One icon-button + a drag overlay + a removable chip. No front-door
  redesign, no audience picker, no mode menu (all P7). The creator composer path must remain byte-identical.
