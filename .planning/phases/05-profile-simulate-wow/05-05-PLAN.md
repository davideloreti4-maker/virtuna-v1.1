<objective>
Build the Simulate verb: `runSimulate` runs a drafted message through a General audience on the existing Flash engine and emits a `reaction-distribution` card; the `/api/tools/simulate` route wires it into the SAME thread so Profile→Simulate is one continuous flow.

Purpose: SIMU-01/02/03. `runTwoAudienceRead` is the closest analog — lift its per-audience read, drop the 2-audience delta. The honesty line is D-02: a person gets a single honest read (no fabricated crowd — Pitfall 2), a panel gets the distribution + clustered themes. The person/panel branch is DETERMINISTIC — it reads the `subjectKind` that Profile (05-04) persisted on the audience's `custom_context`, never re-inferring from persona count (so a person SIM that baked >1 persona cannot mis-branch to "panel"). Bands-only + Directional-by-rule + reuse `aggregateFlash` (never re-roll). This route is the second untrusted boundary — the drafted message + the resolved audience's `success_criterion`/`custom_context` are isolated (D-08).
Output: `runSimulate` + the simulate route, person/panel-aware (from the persisted marker), bands-only, persisting to the open thread.
</objective>
