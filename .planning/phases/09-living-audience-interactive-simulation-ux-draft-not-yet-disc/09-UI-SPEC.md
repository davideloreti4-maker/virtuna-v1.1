## Spacing Scale

Declared values (read from `globals.css --spacing-*`, 8px base; all multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| spacing-1 | 4px | Dot/icon gaps in the swarm legend, chip inner padding |
| spacing-2 | 8px | Persona-node label gap, reaction-feed row gap, scale-toggle inner padding |
| spacing-3 | 12px | Chat-drawer message gap, archetype-breakdown row padding |
| spacing-4 | 16px | Default Lens-sheet content padding, card-to-card spacing |
| spacing-5 | 20px | The Read header internal block spacing |
| spacing-6 | 24px | Lens-sheet section padding (Read → cloud → feed) |
| spacing-8 | 32px | Major section break (cloud region ↔ chat region) |
| spacing-12 | 48px | Sheet top/bottom safe padding (mobile) |
| spacing-16 | 64px | Reserved — page-level only (not expected in the Lens sheet) |

Exceptions:

- **Population swarm dot geometry** (~1,000 dots, radius derived from `PersonaWeights`) is governed by the `mulberry32` deterministic layout in `PersonaGraph`, NOT the spacing scale — it is canvas/SVG geometry, exempt by nature (same exemption `PersonaGraph`'s 200-dot layout already holds).

## Typography

Read from `globals.css --text-*` / `--font-*` / `--leading-*`. The Lens uses Inter for ALL chrome; Newsreader serif is NOT used here (the Lens is an instrument, not a voice-moment). Four roles only:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (`--text-base`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) | Persona verbatims, chat-with-persona messages, reaction-feed rows |
| Label | 14px (`--text-sm`) | 500 (`--font-medium`) | 1.25 (`--leading-snug`) | Archetype names, segment-cluster labels, scale-toggle labels, live counters' captions |
| Heading | 20px (`--text-xl`) | 600 (`--font-semibold`) | 1.25 (`--leading-snug`) | The Read verdict line (sheet header), drawer persona name |
| Display | 30px (`--text-3xl`) | 600 (`--font-semibold`) | 1.1 (`--leading-tight`) | Population live aggregate counters (the big "loved / scrolled" numbers) |

Weight discipline: **400 + 600 are the two load-bearing weights** for this phase (`--font-medium` 500 is the existing chrome label weight, retained for Label role parity with shipped cards — not a new weight). Do NOT introduce 700/bold in the Lens.

Caption/timestamp/12px (`--text-xs`) inherits the existing `--color-cream-muted` treatment for sub-Read metadata ("instantiated from your 10 calibrated archetypes" honesty label) — same as shipped timestamps; not a new role.

## Color

Flat-warm THEME-06 (`globals.css`). 60/30/10 split below. **Cream-alpha, never pure white. Coral is reserved for the weak/worst cluster ONLY — never a generic accent or a "loved" state.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#262624` (`--color-charcoal-app` → `--color-background`) | Lens-sheet background, swarm canvas backdrop, drawer backdrop |
| Secondary (30%) | `#1e1d1b` (`--color-charcoal-composer` → `--color-surface`) + `#2f2e2b` (`--color-charcoal-chip` → `--color-surface-elevated`) | The Read header card, chat-drawer surface, scale-toggle track, archetype-breakdown rows, persona detail cards. Cream-alpha dots/links use `--color-cream-*` at low alpha (matches `PersonaGraph`/`PersonaCloud`). |
| Accent (10%) | `#d97757` (`--color-coral-500` → `--color-accent`) | See reserved list below |
| Destructive | `--color-error` (`oklch(0.60 0.20 25)`) | Persona-registry / cache-discipline guardrail only — NOT a UI surface this phase (no user-facing destructive action exists; see Copywriting) |

**Accent (coral) reserved EXCLUSIVELY for — explicit list:**
1. The **worst/weak cluster** in the swarm + segment-grouping (`worstBadGroupKey` coral treatment — the single below-~40% group, ≤2 coral marks per frame, per `audience-derive.ts`).
2. The **"Rewrite for this audience →" sticky primary CTA** (the one primary action; uses `--shadow-button` flat-matte + coral, mirroring shipped chain CTAs).
3. The Read **lever** emphasis token where the existing P8 Read already paints the lever (reuse P8's treatment verbatim — do NOT recolor the inherited Read).

**Coral is NEVER used for:** "loved"/positive sentiment, generic hover, links, persona-node selected state, the scale toggle's active side, live positive counters, chat-send. Positive/neutral sentiment uses cream-alpha + the score-zone `--color-success`/`--color-warning` data colors already established on score chips (data colors, not chrome). Selected/hover states use `--color-hover` / `--color-active` (white-alpha), as everywhere else.

Honesty-label and "instantiated from your 10" microcopy: `--color-cream-muted` (`#8a857c`), never coral, never error.

## Copywriting Contract

All copy honesty-spine compliant: labeled SIM-1, shows reasoning, never fabricates a crowd/timeline/focus-group. CTA verbs are concrete (matches shipped "Develop this →" / "Test full →" family).

| Element | Copy |
|---------|------|
| Primary CTA (sticky) | **"Rewrite for this audience →"** (routes back to the originating skill runner via `CHAIN_HANDOFFS`, lever-as-steering → new card + Read showing the delta). On non-regenerable surfaces (plain chat turns) the CTA is **hidden** (Claude's discretion D-05 — no concept object to regenerate). |
| Secondary CTA (per persona node) | **"Ask them why →"** (opens the in-context chat drawer scoped to this Read; one persona at a time) |
| Scale toggle labels | **"Panel · 10"** ⇄ **"Population · 1,000"** (the 10 named archetypes are the legible legend of the swarm; defaults to last-used per-Read, D-07) |
| Population honesty label | **"1,000 viewers instantiated from your 10 calibrated archetypes."** (always visible under the swarm; `--color-cream-muted`) |
| Replay control | **"Replay reactions"** (Test/Read, segment-by-segment) / **"Play"** (text concepts, staggered cascade — the room "reads" it). NEVER labeled as a real-time crowd. |
| Empty state heading | **"No audience reaction yet."** |
| Empty state body | **"Run this concept against your audience to see how the room reacts."** (degraded-signal path: where `segment_reactions`/personas are empty, the Lens omits the cloud entirely rather than rendering an empty box — matches `PersonaCloud`'s existing degrade rule; the empty *copy* shows only where a reaction is expected but absent.) |
| Cold-start / thin-audience body | **"Using the General audience — calibrate an audience to read this against your people."** (never fabricates personas; honesty spine, mirrors P7 calibration fallback) |
| Chat-drawer empty / prompt | **"Ask {archetype} why they reacted this way."** (persona answers in-voice about THIS concept) |
| Error state | **"Couldn't reach the audience right now. Your concept is saved — try again in a moment."** (problem + reassurance + retry path; reuses the shipped `SkillRunError` surface — no new error UI) |
| Reduced-motion mirror (sr-only) | **"Audience reaction summary: {n} of {total} viewers stopped; strongest in {segment}, weakest in {segment}."** (sr-only aggregate + archetype breakdown — mirrors `PersonaGraph`'s sr-only list) |

**Destructive actions this phase:** **NONE.** The Lens reads, replays, chats, and regenerates (regenerate appends a new card — additive, not destructive). No delete/clear/overwrite of user data is in scope. No destructive-confirmation copy required. (Persona-registry byte-stability is a code-cache guardrail, not a user action.)

## Interaction Contract (phase-specific — consumed by planner/executor)

> Beyond the standard token tables, this phase is interaction-heavy. These contracts are load-bearing.

| Interaction | Contract |
|-------------|----------|
| Open the Lens | `PersonaCloud onOpen` seam (already stubbed) → AudienceLens sheet. Single v1 entry across all 6 skills (D-04, degrade-by-feature). |
| Scale toggle | Per-Read state, remembers last choice (D-07). Panel·10 = constellation + feed + chat; Population·1,000 = deterministic swarm (D-02) + live counters + cascade + archetype breakdown. Same signal, two resolutions — counters = weighted rollup of the 10. |
| Node drill-down | Tap dot → archetype + its verbatim reaction to THIS concept. ≥44px hit area. |
| Chat-with-persona | In-context drawer/sheet over the cloud, sub-thread persisted within the Read (D-03), one persona at a time. Grounded on `persona-registry` prompt + reaction-to-this-concept; answers in-voice via extended `chat-runner`. |
| Reaction replay | Test/Read (`segment_reactions`) → segment-by-segment. Text concepts → staggered cascade ordered by sentiment/weight (D-06). NO synthesized pseudo-timeline. |
| Rewrite loop | Lever-as-steering → originating runner → new card + new Read in-thread, shows delta (D-05). |
| Motion / a11y | Cascade + pulse gated on `reducedMotion` (extend `PersonaGraph`'s existing gate). sr-only aggregate + archetype-breakdown mirror always present. Deterministic `mulberry32` only — no `Math.random` (SSR/hydration + engine-determinism-gate safe, D-02). |
