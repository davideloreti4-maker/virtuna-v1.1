<task type="auto">
  <name>Task 2: Animation and CLS Verification</name>
  <files>.planning/phases/10-final-qa/10-VERIFICATION.md</files>
  <action>
Verify 60fps animations and no layout shift:

1. **Enable FPS meter:**
   - DevTools > Command Palette (Cmd+Shift+P)
   - Type "Show FPS" and enable

2. **Test all animated elements:**
   - Button hover states
   - Card hover effects
   - Modal open/close transitions
   - Accordion expand/collapse
   - Page transitions (if any)
   - Network visualization animation
   - Loading spinners/phases

3. **Check for frame drops:**
   - FPS should stay green (60fps)
   - No red bars in FPS chart
   - If drops detected, identify cause with Performance panel

4. **Test for CLS (Layout Shift):**
   - Use Web Vitals extension or paste CLS debug script:
   ```javascript
   new PerformanceObserver((list) => {
     for (const entry of list.getEntries()) {
       if (entry.hadRecentInput) continue;
       console.log('CLS:', entry.value, entry.sources);
     }
   }).observe({ type: 'layout-shift', buffered: true });
   ```

5. **Test interaction-time CLS:**
   - Click dropdowns (no jump)
   - Open modals (no background shift)
   - Load async content (no jump)
   - Toggle accordions (smooth, no jump)

6. **Update verification document:**
   ```markdown
   ## Animation Performance

   | Element | FPS | Drops? | Fixed? |
   |---------|-----|--------|--------|

   ## Layout Shift Audit

   | Trigger | CLS Value | Source | Fixed? |
   |---------|-----------|--------|--------|
   ```
  </action>
  <verify>All animations at 60fps, no CLS issues detected</verify>
  <done>Animation performance verified, no layout shift</done>
</task>
