# Phase 4 — UI Design Contract

> Visual and interaction contract for the Mobile Reading Thread + PWA Shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase is presentation-only.** It renders the Phase-2 view-model (`ReadingBlock[]`) and reshapes the EXISTING `StageEvent`/SSE stream into block reveals. No `lib/engine/` edits. No new tokens — everything below is grounded in the shipped `.numen-surface` token layer (`src/app/globals.css`) and the built kit (`src/components/numen/`). Mobile-first; desktop is Phase 7 (do not break it, do not design it).
>
> **North star:** "Whoop for content" — a calm, trustworthy, neutral instrument. The words "throne / pronounces / crystallizes / climax" describe INFORMATION HIERARCHY and TIMING, never mystical decoration. Verdict = band + one-sentence why, never a naked number.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (ground-up `.numen-surface` kit — NOT shadcn; `components.json` present but unused by this kit per CLAUDE.md "ground-up kit, not a retheme") |
| Preset | not applicable |
| Component library | Radix primitives (existing) + ground-up `src/components/numen/` kit (`tailwind-variants`) |
| Icon library | Lucide only (D-09) — do NOT use the Phosphor `ui/icon.tsx` wrapper inside the Reading |
| Font | Inter (sans, all UI/body) + Source Serif 4 (`--font-serif`, voice moments only) |

**Scope class (load-bearing):** the entire Reading surface mounts under `.numen-surface`. Every warm-neutral token (`bg-numen-*`, `text-numen-*`, `border-numen-border`, verdict colors) resolves only inside this scope. The legacy coral `@theme` (`bg-accent`, `border-border`) MUST NOT appear anywhere in the thread — that is the live Raycast app, a different surface.

**Kit components this phase composes (do not re-roll):**

| Component | Role in the Reading |
|-----------|---------------------|
| `Surface` | The default block container — hairline border, `bg-numen-panel`, 12px radius, soft elevation. Every evidence block is a `Surface`. |
| `Glass` | RARE — reserved for the inert reply composer at the thread bottom ONLY. Blur via inline style (Lightning CSS). Not a card surface. |
| `StageBlock` | The ONE motion moment (DS-07). Wraps each block + the throne crystallization. Reduced-motion → static opacity. Do NOT introduce new motion. |
| `VerdictSwatch` | The throne's band color chip (`good`/`mixed`/`bad`). Muted peers, never a good→bad gradient. |
| `PillChip` | Intent chips (audience `intents[]`), and the inert follow-up tap shells. `intent="instant"` here; `agentic` is Phase 6. |
| `IconButton` | Circular 44px controls — "analyze new" action, menu affordance, dismiss-hint. Always `aria-label`. |

## Spacing Scale

Declared values (multiples of 4 — sourced verbatim from `--spacing-*` in `globals.css`, 4px base):

| Token | Value | Usage in the Reading |
|-------|-------|----------------------|
| xs | 4px | Icon-to-text gaps, chip inner gaps (`gap-1`) |
| sm | 8px | Compact intra-block spacing, chip rows (`gap-2`) |
| md | 16px | Default block inner padding, intra-block element gaps (`p-4`, `gap-4`) |
| lg | 24px | Block-to-block vertical rhythm in the thread, throne inner padding (`gap-6`, `p-6`) |
| xl | 32px | Throne-to-first-block separation; section breaks (`gap-8`) |
| 2xl | 48px | Thread top inset below the header; major rhythm break before the inert composer (`pt-12`) |
| 3xl | 64px | Bottom scroll padding so the last block clears the pinned composer (`pb-16`) |

**Exceptions:**
- **44px** minimum touch target for ALL interactive controls (`IconButton`, tappable `PillChip`) — enforced by the kit (`min-h-[44px] min-w-[44px]`). This is a hit-area floor, not a layout spacing value.
- **Thread max content width 640px** (`--breakpoint-sm`), centered, single column — the thread is a phone-width column even on wider viewports (mobile-first; desktop widening is Phase 7).
- **Safe-area insets:** the pinned header and inert composer respect `env(safe-area-inset-top/bottom)` (iOS PWA notch/home-bar). Additive to the scale, not a substitute for it.

## Typography

Inter for all UI/body/labels; **Source Serif 4 (`--font-serif`) reserved exclusively for two voice moments**: the throne's one-sentence **why** and the **expert-insight lead line**. Serif nowhere else (DS-04). Sizes/weights sourced from `--text-*` / `--font-*` tokens.

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Verdict band label (display voice) | `--text-3xl` 30px | `--font-semibold` 600 | `--leading-tight` 1.1 | The throne headline — the band word ("High potential" / "Solid contender" / "Needs work" / "Mixed signals"). Sans, not serif (it is a label, not prose). |
| Voice prose (serif) | `--text-xl` 20px | `--font-regular` 400 | `--leading-snug` 1.25 | The throne's one-sentence **why** + the expert-insight lead line. ONLY serif usage. |
| Block heading | `--text-lg` 18px | `--font-semibold` 600 | `--leading-snug` 1.25 | Each evidence block's quiet plain-language title ("The hook", "What keeps them watching", "Who responds"). |
| Body | `--text-base` 16px | `--font-regular` 400 | `--leading-normal` 1.5 | All block body copy, rewrites, fixes, summaries. |
| Label / caption | `--text-sm` 14px | `--font-medium` 500 | `--leading-normal` 1.5 | Stat captions, intent chips, the demoted `/100` chip, the confidence-language line, timestamps. |

**Rules:**
- Exactly **2 sans weights in the body of the contract**: regular (400) + semibold (600). Medium (500) is permitted ONLY for the 14px caption/label tier (chip + stat caps) — it exists as a token and the kit's `PillChip` already uses it; treat it as the label weight, not a third body weight. Never `--font-bold` (700) inside the Reading.
- The band label is a LABEL set in sans — it must never be confused with the engine's `/100` number. The number never appears in the throne headline (D-02a).
- No engine jargon in any heading or body (READ-07). "Retention heatmap" → "What keeps them watching". "behavioral_predictions" → "Who responds". "anti_virality_gated" → never surfaced; it only routes the verdict to "Mixed signals".

## Color

Warm-neutral dark, dark-only. The user's content + the verdict scale carry ALL the color; the chrome recedes (DS-08). Sourced verbatim from `.numen-surface` in `globals.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1714` (`--numen-bg`) | The thread background (warm near-black, no pure black) |
| Secondary (30%) | `#211e1a` (`--numen-panel`) / `#2a2622` (`--numen-panel-2`) | Every block `Surface`; panel-2 = the throne + elevated/glass composer |
| Accent (10%) | `#d98a5e` (`--numen-accent`, clay) | See reserved list below |
| Verdict — good | `#7faf7a` (`--numen-verdict-good`, muted green) | Throne swatch when band = `high`. A peer, not "best". |
| Verdict — mixed | `#d6a85a` (`--numen-verdict-mixed`, amber) | Throne swatch when band = "Mixed signals" — FIRST-CLASS, its own state, never a warning tint. |
| Verdict — bad | `#d4866f` (`--numen-verdict-bad`, muted clay-red) | Throne swatch when band = `needs-work`. A peer, not an error. |
| Text | `#f0ebe3` body / `#bab2a5` muted | `--numen-text` / `--numen-text-muted` |
| Border | `rgba(240,235,227,0.06)` | `--numen-border` hairline on every Surface |

**Accent (clay `#d98a5e`) reserved for — explicit list, nothing else:**
1. The single **"analyze new"** primary action (the one persistent CTA).
2. **Focus rings** on interactive controls (`focus-visible:ring-numen-accent`, already in the kit).
3. The faint ring on **agentic** follow-up chips — but those are INERT in Phase 4 (Phase 6 wires them); render them present-but-disabled, accent ring allowed as the visual seam.

Accent is NEVER used for: verdict color (the three verdict tokens own that), block headings, body text, evidence emphasis, the `/100` chip, deltas, or decorative fills. **If two things on screen are clay, one is wrong.** Verdict good/bad are NOT accent — they are their own load-bearing scale (DS-02).

**Destructive:** none in Phase 4. There are no delete/destructive actions in the Reading thread this phase (history deletion is Phase 5). The inert reply composer cannot send. No destructive-confirmation surface is contracted here.

**Verdict scale honesty (DS-02 / Phase 3):** the three verdict colors are MUTED peers, not a saturated good→bad gradient. "Mixed signals" (amber) is co-equal with the other two, not a degraded state. Any text on a verdict swatch must meet APCA Lc ≥ 60 (`scripts/check-apca.ts`).

## Interaction & Motion Contract

> This section is the load-bearing READ-* choreography. The checker validates the throne/stage-reveal behavior here against success criteria 1–4.

### Thread shell (READ-01, D-01)
- The Reading is **one rich single-column structured turn** inside a minimal thread container — NOT 10 chat bubbles, NO avatar/bubble chrome, NO "neon AI chatbot" feel.
- The thread **pronounces first**: on a new analysis the structured Reading is the first and only turn; there is never a blank prompt or an empty "type a message" first screen (READ-01).
- The thread carries: (a) a pinned header with the single **"analyze new"** `IconButton` + a quiet menu affordance; (b) at the bottom, the **inert reply composer** — a `Glass` input shell, visually present but disabled/non-sending (Phase 5/6 wires it). It reads "Ask about this Reading…" as placeholder, is not focusable-to-send, shows no error on tap.

### Throne reveal (READ-03, D-02 / D-02a) — the ONE climax
- The throne occupies a **reserved top slot from t0**, rendered as a `panel-2` `Surface`. Before `complete` it shows a **calm "forming the read…" placeholder**: a quiet, low-contrast line at `text-numen-text-muted`, NO `VerdictSwatch`, NO band word, NO number, NO `/100`. A single soft pulse via the existing skeleton-breathe rhythm (`animate-skeleton-breathe`, 1.2s) is the only motion — never a spinner, never a changing number.
- Evidence blocks **stage-reveal BELOW** the held throne as their engine stages complete (the throne stays in its "forming" state the whole time — its reserved presence + the accruing evidence below IS the "visibly forming" of READ-03; there is no fake provisional band).
- At the SSE `complete` event the throne **crystallizes** via ONE `StageBlock` motion (opacity calm-tween + high-damping translate, no overshoot): the `VerdictSwatch` (correct band color), the band label (sans 30px), and the one-sentence **why** (serif 20px) appear together as the climax. Never a slot-machine spin, never an overshoot/bounce.
- The throne's `confidenceLanguage` renders as a 14px caption line beneath the why ("Confident read" / "Reasonably confident" / "Tentative read" / "Mixed signals") — words only, never a number, never a hedge.
- **"Mixed signals" is a first-class throne state** alongside the three bands: amber swatch + "Mixed signals" label + the engine-grounded why. It is not styled as an error or an absence.

### Block order & pacing (READ-02 / READ-05, D-03)
- **Phase 4 OWNS render order** (the view-model emits pure data with NO order hints). The order is FIXED and curated; each block animates into its reserved slot as its underlying engine stage completes (as-soon-as-ready, NO artificial one-at-a-time gating).
- **Curated order (the composed argument):**
  1. **Throne** (`verdict`) — reserved top, crystallizes LAST.
  2. **Expert insight** (`expert-insight`) — Apollo, **pinned directly beneath the throne**, foregrounded, never buried by late stage completion (READ-05 — the product's interpretive value). Its lead line is serif voice prose.
  3. **Hook** (`hook`)
  4. **Retention** (`retention`, or `retention-degraded` honest block)
  5. **Audience** (`audience`)
  6. **Drivers** (`drivers`)
  7. **Persona-read** (`persona-read`)
  8. **Fixes** (`fixes`)
  9. **Content-summary** (`content-summary`)
  10. **Audio** — N/A: there is no `audio` block in the union (live-only, dropped Phase 2). Do not render an audio slot.
- The **`analysis-degraded`** block (when emitted) renders directly under the throne, ABOVE expert-insight — it is the honest "what we actually have" disclosure (D-14), first-class, never a red toast.
- **Honest omit-discipline (D-14):** an absent block is silently omitted — NEVER render an empty shell or a placeholder card for a missing block. The thread shows only blocks the view-model emitted.
- **Resting-document parity (READ-06, D-03d):** a re-opened completed Reading uses the SAME curated order and renders identically to the live render (minus the streaming delays — every block appears at once, throne already crystallized). It **opens scrolled to the verdict**. Identical render is the Phase-2 deep-equal guarantee; the UI must not branch its layout between live and resting.

### Demoted `/100` (D-02a, Claude's-discretion call → DECIDED)
- The engine `score` (the `/100`) appears ONLY as a small, quiet **supporting chip inside the verdict block body** (14px label tier, `text-numen-text-muted`, hairline border) — e.g. `71 / 100`. It is NEVER in the throne headline, NEVER accent-colored, NEVER larger than the band label. It reads as one piece of supporting evidence, not the verdict.

### PWA install + offline (SHELL-04, D-04 / D-05)
- **Install coaching = passive, post-first-Reading.** A dismissible hint appears AFTER the first completed Reading (peak felt-value moment) coaching the iOS "Share → Add to Home Screen" flow, plus an always-available install affordance in the menu. **NEVER a blocking modal.** Dismissal is remembered (localStorage). The hint is a calm `Surface` card, not a toast, not an overlay.
- **Offline scope:** app shell offline + cache-on-view of opened Readings (re-openable offline as resting documents). New analysis stays online-only. Serwist + `manifest.ts` (NOT next-pwa). The install hint and offline state surface no engine jargon.
- **Mobile-native feel:** safe-area insets respected, no horizontal overflow at 390px width, momentum scroll, the pinned header + composer feel app-native.

### Accessibility (hard requirements)
- **Reduced motion:** `StageBlock` already zeroes the translate → static opacity appear (DS-07 / D-14). The throne "forming" pulse is suppressed under reduced motion (no breathe). No parallax, no auto-playing motion beyond the single reveal.
- Every `IconButton` has a required `aria-label`. The streaming reveal must not create an aria-live storm — announce the verdict ONCE on crystallization (polite), not every block.
- Touch targets ≥ 44px. Focus rings visible (accent). Verdict color is never the SOLE carrier of meaning — the band LABEL (text) always accompanies the swatch.

## Copywriting Contract

Plain language, calm restraint, zero engine jargon (READ-07). Voice = a knowledgeable, neutral content expert — not mystical, not hype, not a chatbot.

| Element | Copy |
|---------|------|
| Primary CTA | **Analyze new** (the one persistent action; `IconButton` + label) |
| Throne — forming state | **Forming the read…** (muted, no band, no number) |
| Throne — band labels | **High potential** · **Solid contender** · **Needs work** · **Mixed signals** (from `VERDICT_BANDS.label` + the "Mixed signals" dead-band state; never invent new band words) |
| Confidence line | **Confident read** · **Reasonably confident** · **Tentative read** · **Mixed signals** (from `confidenceLanguage()` — words only) |
| Expert-insight heading | **The expert's read** (Apollo, foregrounded; the lead line itself is engine-authored serif prose) |
| Block headings (plain-language map) | Hook → **The hook** · Retention → **What keeps them watching** · Audience → **Who responds** · Drivers → **What's driving this** · Persona-read → **How your audience reacts** · Fixes → **What to change** · Content-summary → **In a sentence** |
| `/100` chip | **{score} / 100** (quiet supporting evidence, in the verdict block body only) |
| Degraded — partial | **Partial read** — "We analyzed {have list in plain words}; some signals weren't available for this one." (first-class honest block, never a red error) |
| Degraded — unavailable | **Couldn't complete this read** — "The analysis didn't return enough to call it. Try analyzing again." (calm, in-thread, never a toast) |
| Retention-degraded | **Retention data wasn't available for this video.** (honest in-block line; never a fabricated curve) |
| Empty thread (no Readings yet — the seam to Phase 5) | Not in scope this phase as a designed home; if a bare new-thread state shows before first analysis, it shows the **Analyze new** action and the pronounce-first thread — never a blank prompt (READ-01). |
| Inert composer placeholder | **Ask about this Reading…** (present, disabled, non-sending in Phase 4) |
| PWA install hint | **Keep this on your home screen** — "Tap Share, then Add to Home Screen." (dismissible, remembered; iOS-coached; never blocking) |
| Error state (analysis failed to start/stream) | **That didn't go through** — "Something interrupted the analysis. Tap Analyze new to try again." (in-thread, calm; never a red toast) |
| Destructive confirmation | none — no destructive actions in Phase 4 |

**Tone rules:** no exclamation hype, no mystical register ("the oracle has spoken" = forbidden), no engine field names, no percentages presented as the verdict, no hedging in the confidence line (the band language IS the confidence — never "we think maybe").
