From src/lib/reading/from-persisted-row.ts:
- `export function fromPersistedRow(input: unknown): CanonicalReading` (PURE, deterministic). Reads
  the RAW `analysis_results` row — hero/apollo under `row.variants.*`, score under top-level
  `row.overall_score`. Returns the narrow `CanonicalReading` (uses `overallScore`, NOT `overall_score`).

WHY NOT the hook short-circuit for the persisted row (the W2 seam): `use-analysis-stream`'s
`initialData` short-circuit CASTS the raw object to `PredictionResult` and the thread then calls
`canonicalFromLive(result)`, which reads TOP-LEVEL `hero`/`apollo_reasoning`. The persisted row keeps
those under `variants.*`, so feeding the raw persisted row as `initialData` would silently DROP
expert-insight + the hero-derived `why` → the resting render would NOT equal the live render
(breaks READ-06). Therefore the persisted route canonicalizes with `fromPersistedRow` (which DOES
read `variants.*`) and feeds the resulting `CanonicalReading` via `restingReading`. The live-shaped
`initialData` short-circuit is for live/permalink-replay of a `PredictionResult`, not the DB row.
