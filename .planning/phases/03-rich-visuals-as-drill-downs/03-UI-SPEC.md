## Typography

Inter throughout (no serif in drill-downs). The locked `@theme` `--text-*` / `--font-*` ramps. Drill-down content is dense/secondary-tier, so it leans on the small end of the ramp ‚Äî this matches every transplant component as authored.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Panel title (`SheetTitle`) | 16px (`--text-base`) | 600 semibold | 1.25 (`--leading-snug`) |
| Chart value / number (tabular) | 18px (`--text-lg`) on StatTile; chart-internal big numbers as authored | 600 semibold | 1 (`--leading-none`) |
| Body / row label inside a panel | 13px (`text-[13px]` ‚Äî the established P2 panel-body size) | 400 regular (500 medium for the lever/label emphasis) | 1.5 (`--leading-normal`) |
| Chart caption / axis / micro-label | 9.5‚Äì11px (as authored per chart: axis 9.5‚Äì10px, captions 11px) | 400 regular (500 on uppercase caps labels) | 1.25 |

**Rules:**
- **Exactly two weights in play for new text the executor writes: 400 (regular) and 600 (semibold).** Existing charts also use 500 (medium) for uppercase caps labels ‚Äî that is carried over verbatim, not introduced. Do not add new weights.
- **Tabular numerals** (`tabular-nums`) on every number, percentage, timecode, and score ‚Äî already universal in the transplant set; preserve it (prevents digit jitter on the stage-reveal in Phase 4).
- **No serif anywhere in a drill-down.** Serif is voice-only (greeting/hero line) per THEME-04; charts and panels are data ‚Üí sans.
- Uppercase micro-labels keep their existing `tracking-[0.05em]`‚Äì`tracking-[0.13em]` letter-spacing (chart identity), not the body `0.2px`.

## Color

The LOCKED flat-warm token system (THEME-06, Phase-1 D-07). Charts reskin **onto these exact tokens** ‚Äî they do not introduce new values. The single most important rule of this phase: **the matte taste bar (no glow / shine / halo / ambient lighting) extends to every chart** (D-07).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-background` = `#262624` (charcoal-app) | App canvas behind the thread |
| Secondary (30%) | `--color-surface` = `#1e1d1b` (charcoal-composer) ‚Äî the `DrillSheet` panel surface (`bg-surface`); `--color-surface-elevated` = `#2f2e2b` (charcoal-chip) for lifted tiles/cells inside a panel | Drill-down panel surface, stat tiles, keyframe cell backplates |

| Score-zone green | `--color-success` = `oklch(0.68 0.17 145)` | Score gauge fill / score-zone data only (‚â•70) |
| Score-zone amber | `--color-warning` = `oklch(0.75 0.15 85)` | Score-zone data (40‚Äì69, whole band) |
| Score-zone red | `--color-error` = `oklch(0.60 0.20 25)` | Score-zone data (<40) |

**Text colors (cream, never pure white ‚Äî D-02):** `--color-foreground` `#ece7de` (primary), `--color-foreground-secondary` `#c2bdb4` (default chart text), `--color-foreground-muted` `#8a857c` (axes, captions, timestamps, empty states). **Every `text-white/NN` and `rgba(255,255,255,‚Ä¶)` text/dot color in the transplant components must be repointed to a cream token or cream-alpha** (`rgba(236,231,222,‚Ä¶)` ‚Äî the exact value PersonaCloud already uses).

**Borders:** `--color-border` = `rgba(255,255,255,0.06)` (hairline), `--color-border-hover` = `rgba(255,255,255,0.1)`. The charts' existing `border-white/[0.06]` already equals this ‚Äî keep, but reference the token where you touch it.

### Accent reserved for (explicit ‚Äî never "all interactive elements"):
Coral is the lone color besides the score zones. In drill-downs it is reserved for **the single worst/weakest signal and the user's own position marker**, mirroring how every transplant component already uses it:
1. The **worst persona cluster** dots in PersonaGraph (`tone === 'accent'`) ‚Äî exactly one cluster, the same one PersonaCloud already corals.
2. The **biggest-drop mark + lock line + "‚àíNN%" label** in RetentionChart (one drop point).
3. The **single coral "bad" row** in SegmentTable (`badKey`, worst group below ~40%).
4. The **"you" marker** (pin + dot + chip) and the **confidence band tint** in ScoreDistribution.
5. The **weakest StatTile** in the shareability panel (`tone === 'accent'`).
6. The **no-CTA end-cap scalpel** in CraftFilmstrip (one edge, only when there's no close).
7. The Phase-2 **focus ring** on tap targets (`ring-accent/50`) ‚Äî carried over.

**Do NOT** coral-tint chart backgrounds, every bar, hover states broadly, or "active" decoration. Neutral cream is the default; coral is the scalpel.

### Score-zone color: the SSOT
The score gauge and any score-derived band color MUST come from `bandTone(score)` in `src/components/board/verdict/verdict-derive.ts` (‚â•70 `good`/green ¬∑ 40‚Äì69 `warn`/amber [whole band, never red] ¬∑ <40 `crit`/red). This is the same SSOT `ScoreGauge` and `DriverRows` already use ‚Äî the score panel's "you" reading and any zone treatment must not invent a parallel scheme.

**Destructive:** none. This phase has **no destructive actions** (no delete, no irreversible state). The `--color-error` red is **score-zone data only**, never a destructive-action affordance.

## Transplant Architecture Contract (the highest-risk landmine ‚Äî verified)

> **Mount the LEAF chart components directly; re-derive their props inside the reading panels from `data` + the PURE derive helpers. NEVER import the board *Node/Hero/Frame* wrappers.**

This is correctness, not style. Verified findings:

- **Leaf charts are CLEAN** (grep-confirmed): `RetentionChart`, `RetentionPlayer`, `SegmentTable`, `CraftFilmstrip`, `PersonaGraph`, `ScoreDistribution`, `StatTile` import **no** `useBoardStore`, `use-audience-choreography`, `use-client-weights`, or camera state. They are pure prop-driven and transplant cleanly. ‚úÖ
- **Their canonical CALLERS are deeply board-store-coupled** and must NOT be reused: `AudienceNode.tsx` (pulls `useBoardStore`, `useAudienceChoreography`, `useClientWeights`, `pendingVideo`), `VerdictNode.tsx` (pulls `useBoardStore`, `useComparisons`), `ContentAnalysisFrame.tsx`. Importing any `*Node`/`*Hero`/`*Frame` drags the retired board store back into the deliberately store-free reading cluster (Phase-2 P2-04/05 invariant) ‚Äî a hard violation.
- **The pure derive layer IS shared and safe to call from panels:** `board/audience/audience-derive.ts` (already imported by `reading.tsx`/`driver-rows.tsx`/`persona-cloud.tsx`), `board/verdict/verdict-derive.ts` (`confidenceRange`, `bandTone` ‚Äî already imported by `score-gauge.tsx`), `board/content-analysis/content-analysis-derive.ts`. These are React-free pure functions. ‚úÖ
- **`useComparisons(analysisId)` is a TanStack `useQuery`** hitting `/api/analyze/[id]/comparisons` ‚Äî NOT board-store. It is **safe to call inside the score panel** to source `niche: NicheCohort` (the histogram). Calling it lazily (only when the score panel mounts) keeps the container's single-subscription contract intact (it's a separate, panel-local query, not a second analysis subscription). The hero gauge keeps reading `overall_score` only ‚Äî no comparisons needed for the headline.

## Interaction & Responsive Contract

- **Open/close:** every drill-down opens in the ONE `DrillSheet` (bottom sheet on mobile / right drawer on desktop, via `useIsMobile`), driven by the single `panel` state in `reading.tsx`. The Reading stays mounted behind; dismiss returns the user in place. Phase 3 adds **panel TYPES** (`'score'`) to the closed union ‚Äî it does NOT touch `drill-sheet.tsx` (panel-agnostic by design).
- **Closed allow-list union (security, threat T-02-12):** `PanelId` grows `hook | retention | shareability | personas` ‚Üí **`+ score`**; `PANEL_TITLE` grows in lockstep. **Never reflect a raw key** into the title or content switch.
- **Panel width:** the thread column is `max-w-[760px]`; **rich visuals MAY break wider inside the sheet/drawer** (the drawer is `sm:max-w-[560px]` desktop / `max-h-[85vh]` mobile ‚Äî the focused home for a heavy chart). Charts are width-responsive (SVG `preserveAspectRatio`, `%`-based geometry, `@container` StatTileRow) so they fit both.

- **Mobile interactivity downgrade (discretion ‚Äî no hover-only affordances on touch):**
  - `PersonaGraph` hover card ‚Üí **tap-to-reveal** on mobile (the desktop `onMouseEnter`/`onMouseLeave` must have a touch equivalent; a tapped node shows its card, tap-elsewhere dismisses).
  - `RetentionPlayer` (if included) scrub ‚Üí already pointer-driven (`onPointerDown/Move/Up` + keyboard slider); fine on touch.
  - `SegmentTable` sort ‚Üí keep simple; sorting is optional, don't add a hover-only sort control.
- **Reduced motion:** any chart animation respects `usePrefersReducedMotion()`. Specifically the PersonaGraph node `<animate>` pulse (gate it) and any reveal. The hero gauge's stroke transition is already reduced-motion-gated.
- **Phase-4 awareness (keep the seam intact):** these visuals get stage-revealed in Phase 4 (driven by `useAnalysisStream` at the CONTAINER). Keep every transplanted visual **prop-driven and side-effect-light** ‚Äî no internal data subscription, no self-fetching (the one exception, `useComparisons` in the score panel, is a cached query keyed by id and is fine). This lets Phase 4 animate them without a rewrite.
- **Degradation (D-13, Roadmap SC-2 ‚Äî every visual):** real data ‚Üí render; null/thin data ‚Üí `PanelEmpty` ("Not available for this read."). **Never** throw, **never** a grey-cell fallback box, **never** a fabricated 0. Audit each transplanted visual's null/empty paths:

### RetentionPlayer inclusion (discretion ‚Äî gated)
Include the scrubbable video player **only if the uploaded video source resolves on permalink reload**. Verified constraint: `useUploadedVideoSource(result, pendingObjectUrl)` resolves a source from (1) `pendingObjectUrl` ‚Äî a board-store `pendingVideo.objectUrl`, **unavailable in the store-free reading cluster** ‚Äî or (2) a signed URL from `/api/videos/sign` when the persisted row is `input_mode === 'video_upload'` with a `video_storage_path` (can 404 ‚Üí `missing`). **So on a permalink reload the only available source is the signed-URL path; the fresh-upload blob path requires the board store and must NOT be reintroduced.** Recommendation: default the retention panel to **static RetentionChart + filmstrip + SegmentTable**; layer RetentionPlayer in only when `useUploadedVideoSource(data, null).status === 'ready'`.
