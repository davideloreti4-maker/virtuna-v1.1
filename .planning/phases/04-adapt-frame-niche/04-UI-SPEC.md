## Typography

All Inter, antialiased, letter-spacing 0.2px global. This phase uses exactly 4 sizes and 2 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Concept hook headline | 16px (text-base) | 600 (semibold) | 1.4 | Hook field — the bold headline of each concept card |
| Niche prompt heading + NichePicker tiles | 14px (text-sm) | 500 (medium) | 1.4 | Niche prompt heading; NichePicker tile labels (existing component, unchanged) |
| Sub-row values + chip + muted text | 12px (text-xs) | 500 (medium) | 1.4 | Angle/who-it's-for values; format_borrowed chip; empty/error body text |
| Frame section label | 12px (text-xs) uppercase, tracking-widest | 500 (medium) | 1.0 | Meta-label above concept list ("ADAPTED FOR YOUR NICHE") — uppercase+tracking gives visual smallness without leaving the 4px scale |

Weights used: medium (500) for all supporting content + semibold (600) for hook headlines only.
11px, 13px, 10px, regular (400) are eliminated from this phase.

## Color

60/30/10 split from BRAND-BIBLE.md. All values sourced from globals.css.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#07080a` (`--color-background`) | Frame body background; concept card background is `transparent` over this |
| Secondary (30%) | `#18191a` (`--color-surface`) | Concept card surface (transparent card per Raycast card pattern — blends to dominant) |
| Accent (10%) | `#FF7F50` (`--color-accent`, coral-500) | format_borrowed chip background-tint and chip text; no other element in this frame uses coral |
| Destructive | `--color-error` oklch(0.60 0.20 25) | Not used this phase — no destructive actions |

Accent reserved for: **format_borrowed chip only** (the visual signal that makes "format not content" explicit at a glance). Every concept card has exactly one coral touch. Niche picker selected state uses `bg-white/[0.08]` + `border-white/[0.12]` (existing NichePicker spec — white tint, not coral).

Border tokens used:
- Concept cards: `border border-white/[0.06]` (1px, `--color-border`)
- Concept card hover: `bg-white/[0.02]` only — no translate-y, no border change
- NichePicker tiles: existing `border-white/[0.06]` default, `border-white/[0.12]` selected
- format_borrowed chip: `bg-accent/[0.12]` fill + `text-accent` text (coral tint, not opaque)

## Surface Patterns

### Concept Card (each of 3)

```
bg-transparent
border: 1px solid rgba(255,255,255,0.06)
border-radius: 12px   (--radius-lg, Raycast card standard)
padding: 16px         (--spacing-4)
box-shadow: rgba(255,255,255,0.05) 0px 1px 0px 0px inset
hover: bg-white/[0.02]   (no translate-y, no border change — Raycast rule)
```

Internal layout (top-to-bottom):
1. Hook headline — `text-base font-semibold text-foreground` (16px, 600, #f9f9f9)
2. format_borrowed chip — `text-xs font-medium text-accent bg-accent/[0.12] rounded-full px-2 py-0.5` (12px, 500, coral, pill shape, --radius-full)
3. Divider — `border-t border-white/[0.04]` (--color-border-subtle), 8px margin top
4. Angle row — `text-xs font-medium text-white/45 uppercase tracking-widest` label + `text-xs font-medium text-foreground-secondary` value, 4px gap
5. Who-it's-for row — same pattern as angle row

### Stacked Card Container

Three concept cards stacked vertically with `flex flex-col gap-4` (16px gap). All 3 fully visible — no accordion, no tabs. Fits the tall right-column bounds inherited from Actions frame (Phase 2 D-07).

### Inline Niche Prompt State

When `niche_primary` AND `niche_sub` are both null on `creator_profile`:

```
Outer wrapper: flex flex-col gap-4 w-full
Heading: text-sm font-medium text-foreground  ("Add your niche to generate concepts")
Body: text-xs font-medium text-foreground-muted  ("Concepts adapt the source format to your niche.")
NichePicker: existing component, no restyling, compact (embedded in frame, not full-page)
CTA: Button variant="primary" size="sm" label="Generate concepts" — appears after both primary+sub are selected; disabled until sub is selected
```

NichePicker sits inside the frame body. The Decode frame (sibling) continues to render niche-free during this state — frames are independent (D-06).

### Loading / Generating State

While Adapt auto-generates (Qwen call in-flight after Decode completes, niche present):

- Three concept card skeletons: `Skeleton` component (board kit) at approximate card height (~120px each)
- No spinner over the frame — skeletons only (quieter, matches Raycast minimal aesthetic)
- Frame label visible as normal during generation

### Error State

If Adapt generation fails (D-06 — independent frame failure):

- Concept card area replaced by a single inline error message:
  - Icon: `WarningCircle` (Phosphor, 16px, `text-white/45`)
  - Heading: `text-sm font-medium text-foreground` — "Couldn't generate concepts"
  - Body: `text-xs font-medium text-foreground-muted` — "Try refreshing the page. Decode results are still available."
- No retry button this phase (D-07 — no regenerate affordance)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Concepts auto-generate | On Decode completion when niche present — no click required (D-04) |
| Niche supplied inline → generate | After PATCH to creator_profile completes, concept generation starts automatically (D-12) |
| Permalink reload | Rehydrates `variants.remix.adapt` from DB — no re-generation (D-05) |
| Concept card hover | `bg-white/[0.02]` background only — no translate-y, no border change (Raycast card rule) |
| NichePicker tile click | Pure controlled — `primary`/`sub`/`onChange` props; existing component, unchanged API |
| "Generate concepts" button | Appears only in niche-prompt state; disabled until both primary + sub are selected; triggers PATCH then adapt call |
| Mobile (<768px) | 3 concept cards in mobile card-stack order (BoardMobile.tsx slot for Adapt — D-10); same card anatomy; NichePicker same inline behavior |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Concept cards — reading order | Rendered as `<article>` elements in DOM order; no Konva (DOM overlay per Phase 2 RESEARCH Pattern 1) |
| format_borrowed chip | Plain text inside `<span>` — no icon-only elements |
| NichePicker | Existing a11y: `role="group"`, `aria-label`, `aria-pressed` on tiles — unchanged |
| Loading skeletons | `aria-live="polite"` region wrapping skeleton container; `aria-label="Generating adapted concepts…"` |
| Error state | `role="alert"` on error container; icon is decorative (`aria-hidden="true"`) |
| "Generate concepts" button | Descriptive label; `disabled` when sub-niche not yet selected |
| Reduced motion | No animated transitions in concept cards; NichePicker uses `transition-all duration-200` CSS — within tolerance |
| WCAG contrast | Hook text (#f9f9f9 on #07080a) > 15:1. Sub-row values + chip (#9c9c9d on #07080a) = 7.3:1 AAA. Chip text (#FF7F50 on transparent over #07080a) = 3.4:1 — chip is decorative classification at 12px medium; muted label at white/45 passes AA at 12px against #07080a |
