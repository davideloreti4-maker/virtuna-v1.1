## Spacing Scale

Declared values (multiples of 4 — matches the locked `globals.css` `--spacing-*` ramp; do NOT introduce off-scale values):

| Token | Value | Usage in The Reading |
|-------|-------|----------------------|
| xs | 4px | icon-to-text gaps, bar-to-value inline gap, chip inner padding |
| sm | 8px | driver-row vertical rhythm, gap between stacked fix items, dot-cloud dot gaps |
| md | 16px | default block padding, gap between a row label column and its bar |
| lg | 24px | gap between major Reading sections (hero ↔ rows ↔ Fix First ↔ Deeper read); Sheet content padding |
| xl | 32px | hero internal top/bottom breathing; thumbnail-strip ↔ hero gap |
| 2xl | 48px | thread top padding under the thumbnail; resting-state section breaks |
| 3xl | 64px | reserved — page-level only |

**Layout width:** main Reading column caps at **~760px centered** (Phase 1 D-17, carried forward). Rich visuals MAY break wider — but only in their Phase-3 Sheet, not in the Phase-2 thread.

**Exceptions:**
- Touch targets: row detail-tap and chip hit areas must be **≥44px** tall on mobile even when visual height is smaller (pad with transparent area; do not inflate the visible bar). This is the only allowed sub-scale dimension and it is a WCAG/ergonomics floor, not a spacing token.
- Arc-gauge geometry (stroke width, radius) is a visual primitive measured in px, not a spacing token — see Visual Primitives.

## Typography

Inter for ALL data. Newsreader serif is voice-only. Sizes drawn from the locked `--text-*` ramp; weights from `--font-*` (regular 400 / medium 500 / semibold 600 — **use at most two per surface**, default regular + semibold).

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Gauge number | 36px (`--text-4xl`) | 600 semibold | 1.0 (`--leading-none`) | Inter, `tabular-nums` | the `71` inside the arc — the one dominant figure |
| Band word | 16px (`--text-base`) | 500 medium | 1.2 (`--leading-tight`) | Inter (sans — it is a data label, NOT prose) | "Strong" / "Mid" / "Low" under the gauge |
| Row value | 14px (`--text-sm`) | 600 semibold | 1.2 | Inter, `tabular-nums` | the right-hand `87` / `⚠ 0:08` / `64` |
| Row label | 14px (`--text-sm`) | 500 medium | 1.2 | Inter | "Hook" / "Retention" / "Shareability" |
| Section label | 12px (`--text-xs`) | 500 medium, `uppercase tracking-[0.08em]` | 1.4 | Inter | "FIX FIRST", "Deeper read", "Hook rewrites" |
| Body / fix text | 14px (`--text-sm`) | 400 regular | 1.5 (`--leading-normal`) | Inter | fix headline + detail, rewrite variant, sheet native content |
| Watch% caption | 14px (`--text-sm`) | 400 regular | 1.4 | Inter | "57% watch" beside the cloud |
| Muted meta | 12px (`--text-xs`) | 400 regular | 1.4 | Inter | drop timestamps in sheet, "partial read" annotation, em-dash empties |

**Rationale on the band word:** brief §2.7 reserves serif for "voice moments (greeting, hero line)." The band word ("Strong") is a terse data label, not a voice sentence — render it **sans** (Inter) so it pairs visually with the gauge number and reads as instrument, not prose. The serif greeting lives in the Phase-1 shell, not in the Reading body. **No verdict/horoscope sentence appears anywhere** (D-15).

## Color

Flat-warm, LOCKED (THEME-06). Matte: no glow, no shine, no halo, no ambient lighting (D-05 / brief §2.7). Color lives **in the data** — score zones + the lone coral accent; everything else neutral charcoal + cream.

| Role | Value (token) | Usage |
|------|---------------|-------|
| Dominant (~60%) | `#262624` (`--color-charcoal-app` / `--color-background`) | the thread surface / app background |
| Secondary (~30%) | `#1e1d1b` composer (`--color-surface`) · `#2f2e2b` chips/lifted (`--color-surface-elevated`) · `#1a1a18` sidebar | Sheet/drawer surface, fix-item cards, quick-action chips, the bottom-pinned composer |
| Text — primary | `#ece7de` (`--color-cream-primary` / `--color-foreground`) | gauge number, row values, headings — never pure white |
| Text — secondary | `#c2bdb4` (`--color-foreground-secondary`) | row labels, fix body |
| Text — muted | `#8a857c` (`--color-foreground-muted`) | section labels, timestamps, captions, em-dash empties |
| Accent (~10%) | `oklch(0.68 0.13 33)` ≈ `#d97757` (`--color-accent` / coral-500) | see reserved-for list below |
| Score zone — green | `oklch(0.68 0.17 145)` (`--color-success`) | gauge fill when `overall_score ≥ 70`; the weakest-lever bar/value when that lever scores in the green band (rare) |
| Score zone — amber | `oklch(0.75 0.15 85)` (`--color-warning`) | gauge fill when `40 ≤ score < 70`; weakest-lever bar + `⚠` when that lever is the mid-band lever |
| Score zone — red | `oklch(0.60 0.20 25)` (`--color-error`) | gauge fill when `score < 70` per `bandTone` crit (`<40`); weakest-lever bar + `⚠` when the lever is weak; the gate-banner danger pole |

**Accent (coral) reserved for — EXPLICIT list (load-bearing, BRAND-BIBLE "if two things are coral, one is wrong"):**
1. The **worst-retention persona cluster** in the static hero dot-cloud (D-02 — those dots only).
2. **Primary interactive affordances** that are the brand action: the **Copy** button on a hook rewrite, the active focus ring, and the **composer send** control.
3. The **gate banner's** coral pole (the `AntiViralityHeader` gradient runs coral→warning) (D-04).

**Coral is NOT used for:** the gauge fill (that is zone-colored green/amber/red — D-01), driver-row bars (neutral cream — D-07), the weakest-lever highlight (that takes its **zone** color amber/red, NOT coral — D-07), deltas/changes, or generic links inside the Reading.

**Driver-row bar color (D-07):** bars are **neutral cream** — fill `linear-gradient` of cream/white-alpha (NOT the legacy `#FF7F50` coral gradient currently in `FactorBars.tsx`, and NOT the `boxShadow` glow — both must be removed). **Only the single weakest lever** takes its score-zone color (amber if mid, red if weak) + a `⚠` glyph in its value. The hero gauge owns the verdict color; rows stay calm so the eye lands on the one lever to fix.

| Role | Value | Usage |
|------|-------|-------|
| Destructive | `oklch(0.60 0.20 25)` (`--color-error`) | **No destructive actions exist in this phase** (see Copywriting). Token noted for completeness; the only red usage is the score-zone "weak/gated" semantic, which is a *data* color, not a destructive-action color. |

**Palette migration (REQUIRED when recomposing the existing components):** the board components were authored on the OLD Raycast palette and MUST be repointed:
- `text-emerald-400` / `text-amber-400` / `text-red-400` → `text-success` / `text-warning` / `text-error` (score-zone tokens).
- `#FF7F50` / `#E8703F` literal gradients + `boxShadow: '0 0 10px rgba(255,127,80,0.5)'` (in `FactorBars.tsx`) → neutral cream fill, **no glow** (D-05/D-07).
- `text-white` / `text-white/[0.x]` → `text-foreground` / `text-foreground-secondary` / `text-foreground-muted` (cream, never pure white — D-02).
- `border-white/[0.06]` is OK to keep (matches the locked `--color-border` hairline 6%); just ensure no inset-shine shadow rides with it.
- **Sidebar score chips** still use emerald/amber Tailwind — **unify them onto the score-zone tokens in this phase** (Phase-1 follow-up explicitly carried into D-CONTEXT).

## Copywriting Contract

User-facing noun is **"Simulation"** (D-09). Terse labels + data + the engine's own actionable fix/rewrite text ONLY; **no verdict/horoscope prose** (D-15). Exact micro-copy is delegated to build (CONTEXT "Claude's Discretion") — the strings below are the **prescribed defaults**; the executor may refine wording but MUST preserve the meaning, the band-word vocabulary, and the "no horoscope" rule.

| Element | Copy |
|---------|------|
| Section heading — Fix First | `FIX FIRST` |
| Section heading — Deeper read | `Deeper read` (inline expand affordance, D-10) |
| Band words (under gauge) | `Strong` (≥70) · `Mid` (40–69) · `Low` (<70 crit, <40) — use `bandFromScore()` from `verdict-constants.ts` (returns Strong/Mid/Low), NOT `bandLabel()` (which returns the longer "High potential/Solid contender/Needs work" — wrong vocabulary for the locked mock). |
| Watch% caption | `{n}% watch` (e.g. `57% watch`) — from `averageWatchThrough(nodes)` or `weighted_completion_pct`/`completion_pct`; shown **exactly once**, owned by the hero (D / READ-04). |
| Row labels | `Hook` · `Retention` · `Shareability` (fixed funnel order, D-08) |
| Retention value | drop timestamp `⚠ {mm:ss}` (e.g. `⚠ 0:08`) from `weighted_top_dropoff_t` — NOT watch-time (D-06). |
| Gate banner (when `anti_virality_gated`) | reuse `COPY.AV_HEADER(n)`: `⚠ Don't post yet — fixable in {n} steps` / `⚠ Low confidence — review before posting`; override link `Post anyway →` (D-04). Reason from `anti_virality_reason` (confidence / timeline_pattern / both). |
| "more fixes" affordance | `{n} more fixes →` — expands INLINE (D-10). |
| Primary CTA (rewrite) | `Copy` on each hook-rewrite (becomes `Copied` for 1.5s) — the copyable rewrite IS the product value (D-15). |
| Empty state — Fix First (`counterfactuals.suggestions` empty) | heading: `Nothing urgent to fix` · body: `This one's solid.` (a win, not a blank — D-14). |
| Empty state — no rewrites (`apollo_reasoning.rewrites` empty/null) | show the timestamped fixes **without** the rewrite/Copy chip; no placeholder (D-14). |
| Empty state — Deeper read (`apollo_reasoning: null`) | render only what's present (hero/rows still resolve from `overall_score` + gate); omit the missing Apollo dims silently — do NOT show a fabricated dimension (D-13). |
| Degraded — `analysis_unavailable` (fake 0) | dedicated state, heading: `We couldn't analyze this video` · body: `The read didn't come back. Try another video or re-upload.` — **NEVER present the 0 as a real score** (D-13 — correctness, hits on permalink reload). |
| Degraded — `partial_analysis` (one signal dead) | small annotation near the hero: `Partial read — based on half the usual signals.` (D-13). |
| Error state (fetch/permalink failure, `data: null` not from a degrade flag) | heading: `This Simulation didn't load` · body: `Refresh, or open another from the sidebar.` |
| Destructive confirmation | **None.** The Reading is read-only display + copy-to-clipboard + open/close disclosures. No delete, no irreversible action originates in this phase. (`Post anyway →` dismisses the gate banner via localStorage — reversible, non-destructive.) |

## Visual Primitives (phase-specific — prescriptive defaults for the delegated open choices)

These were explicitly left to planner/executor in `02-CONTEXT.md` "Claude's Discretion." Prescribed here so the executor implements without ambiguity; deviation is allowed only within the flat-warm + calm-motion taste bar.

### Arc gauge (hero — D-01)
- **Form:** a flat **stroked radial/arc gauge** — a 0–100 filling arc with the number centered inside. NOT a glow/halo, NOT a filled pie; a stroked track + stroked fill (matte), consistent with the matte taste bar.
- **Geometry (recommended):** ~120px diameter on desktop, ~96px on mobile; track stroke **8px**, `round` linecaps; arc sweep **270°** (gap at the bottom, −225° start) so the opening reads as a gauge dial, OR a full 360° ring — executor's call, but pick one and keep it consistent. Track color = `--color-border` (6% hairline) or `--color-surface-elevated`; **fill = the score-zone color** (`bandTone(overall_score)` → success/warning/error).
- **Number:** Inter 36px semibold `tabular-nums`, cream (`--color-foreground`); band word below in sans medium 16px, colored by the same zone token.
- **Implementation:** inline SVG `<circle>` with `stroke-dasharray`/`stroke-dashoffset` (no library). `role="img"` + an sr-only mirror (`"Score 71 of 100, Strong"`).
- **Reveal motion:** the arc fill animates from 0 → score during the **Phase-4** stage-reveal (calm `--ease-out-cubic`, ~600–800ms). In Phase 2 (static permalink) it renders at rest, filled. Respect `usePrefersReducedMotion` — no animation when reduced.

### Static persona dot-cloud (hero — D-02)
- **Form:** **lightweight static SVG/CSS dots** sized by persona weight; **no Canvas, no hover cards, no `<animate>`** (that's the Phase-3 `PersonaGraph` drill-down).
- **Data:** `buildPersonaNodes(heatmap, persona_simulation_results, badKey)` → `PersonaNode[]` (already returns `weight` 0–1, `tone: 'accent'` for the worst-retention slot). Dot radius scales with `weight`; **the `tone:'accent'` dots render coral**, all others neutral cream/charcoal-chip.
- **Watch%:** `averageWatchThrough(nodes)` (or top-level `weighted_completion_pct`) rendered once as `{n}% watch` beside the cloud.
- **A11y:** wrap in `role="img"` with an sr-only list mirroring the segments (reuse the pattern from `PersonaGraph`).
- **Phase-3 seam:** tapping the cloud opens the full `PersonaGraph` in the Sheet (the Phase-3 mount point) — wire the open affordance now, mount the rich component in Phase 3.

### Driver rows (D-05/06/07/08)
- **Form:** 3-column grid `label | bar | value` (mirror `FactorBars`'s `grid-cols-[130px_1fr_42px]`, widen value col for `⚠ 0:08`). Bar height **5–6px**, `rounded-full`, track `--color-border`.
- **Fill = the 0–100 dimension score** (Hook = `apollo_reasoning.dimensions[hook].score`; Retention = `dimensions[retention].score`; Shareability = `dimensions[share_pull].score`). Neutral cream fill; **only the single weakest** of the three takes its zone color + `⚠` (compute weakest = min score across the 3).
- **Retention value column** = `⚠ {formatTime(weighted_top_dropoff_t)}` (the drop point), while its **bar still fills by the retention score** (D-06 — bar = how good, value = where it breaks).
- **Order:** fixed funnel Hook → Retention → Shareability (D-08).
- **Detail tap:** each row has a tap/click target (≥44px) that opens the **Sheet** (D-09).

### Two-tier disclosure (D-09/D-10/D-11) — THE most-reused decision; sets the Phase-3 seam
- **Heavy drill-downs → Sheet** (`src/components/ui/sheet.tsx`): `side="bottom"` on mobile, `side="right"` on desktop. Switch via the existing `useIsMobile` hook. The Reading stays mounted behind it; dismiss returns the user in place. **This is the Phase-3 mount point** — build it as a generic container that receives any panel (`{ panelId, title, children }`); Phase 3 drops `RetentionChart` / `PersonaGraph` / filmstrip into the same surface, no structural change.
  - Reskin the Sheet to flat-warm: it currently uses `bg-background` (✓ resolves to charcoal) + `shadow-lg`. Drop/replace `shadow-lg` with `--shadow-float` only if it reads as a genuinely floating surface; overlay `bg-black/50` is fine. Border = hairline `--color-border`. No inset shine.
- **Light reveals → inline expand** (D-10): `{n} more fixes →` and `Deeper read` expand **in the thread** (short lists/text stay in the one-document flow). Use the existing `Accordion` (`ui/accordion.tsx`) or a simple `useState` height toggle — NOT a Sheet. Deliberate two-tier: light inline, heavy in the Sheet.
- **Open-panel URL behavior (recommended):** sync the open heavy panel to a **shallow `?panel={rowId}`** query param so the browser **back button closes the Sheet** (the desired mobile feel) and the panel is shareable/deep-linkable. Use Next.js shallow routing / `history.replaceState`. This is a recommendation, not a blocker — pure-ephemeral `useState` is acceptable for v1 if shallow-sync proves fiddly, but back-button-closes-the-sheet on mobile is the target.

### Phase-2↔3 seam content (D-12)
- In Phase 2 the Sheet shows the **actual engine data in simple native form** — e.g. Hook detail = the 4 hook-modality sub-scores (`hook_decomposition`) as plain rows/mini-bars + `weakest_modality`; Retention detail = drop segments (`dropoff_segment_indices` / `heatmap.segments`) as a text list. **Genuine, demoable content — NOT stubs, NOT the rich charts pulled forward.** Phase 3 swaps these for the rich charts in the same container.

### Video thumbnail strip (top of thread — D-03)
- **Recommended:** a **static poster** (single keyframe `<img>`, signed URL — plain `<img>`, not `next/image`, per the `KeyframeImage` pattern) at the top of the thread, gated on a real video (`heatmap.segments[0].keyframe_uri` or first filmstrip frame). Text/tiktok-url modes with no keyframe render **no strip and no broken placeholder** (mirror `resolveKeyframeUrl`'s real-video gate). Inline-playable is deferred (planner may revisit). Decorative `alt=""`.
