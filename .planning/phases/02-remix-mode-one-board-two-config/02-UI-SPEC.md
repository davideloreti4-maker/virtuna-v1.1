## Color

Source: BRAND-BIBLE.md ¬ßColor Tokens, CLAUDE.md ¬ßRaycast Design Language Rules.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#07080a` (`--color-background`) | Page body, canvas background |
| Secondary (30%) | `#18191a` (`--color-surface`) + `rgba(17,18,20,0.8)` glass | Cards, frame bodies, ContentForm glass, board frame backgrounds |
| Accent (10%) | `#FF7F50` (`--color-accent`, coral-500) | See reserved list below |
| Destructive | `oklch(0.60 0.20 25)` (`--color-error`) | Validation error messages only ‚Äî no destructive actions in this phase |

Accent (`#FF7F50`) reserved for:
1. Submit arrow button background (active state) ‚Äî existing
2. Active tab underline in `FrameTabs` ‚Äî existing
3. Coral weak-link `StatTile` (`tone:'accent'`) ‚Äî score-mode only, unchanged
4. Focus rings (outline) on interactive elements ‚Äî existing + intent selector

Accent is NOT used for: descriptor text, shell frame borders, mode-switcher inactive tabs, selected segment fill (selected segment = `bg-white/[0.08]` white-transparency, not coral), hover states (hover = `rgba(255,255,255,0.04‚Äì0.08)` only).

Border tokens:
- Default: `rgba(255,255,255,0.06)` (6%) ‚Äî all cards, frames, form
- Hover: `rgba(255,255,255,0.10)` (10%) ‚Äî hover state on interactive containers
- Subtle separator: `rgba(255,255,255,0.05)` (5%) ‚Äî bottom-bar divider inside ContentForm

## Component Specifications

### 1. Intent Selector (Segmented Control)

**Position:** Above the existing Video/Text/URL input tabs inside `content-form.tsx`, rendered as a top-level row before the textarea/upload area.

**Visual pattern:** Two-segment pill control. Matches the existing `MODE_CONFIG` tab style in `content-form.tsx` (same `rounded-md`, `transition-colors` pattern), promoted to a higher visual weight as the primary choice.

| State | Background | Border | Text |
|-------|------------|--------|------|
| Selected segment | `bg-white/[0.08]` | none | `text-foreground` (14px, weight 500) |
| Unselected segment | transparent | none | `text-foreground-muted` (`#848586`, 14px, weight 400) |
| Hover (unselected) | `bg-white/[0.04]` | none | `text-foreground` |
| Container | `bg-white/[0.03]` | `border border-white/[0.06]` | ‚Äî |

Selected segment fill is `bg-white/[0.08]` (white 8% transparency) ‚Äî NOT coral. Coral is reserved for focus rings only (see Color ¬ßAccent reserved for).

- Border radius: 8px on each segment pill (`rounded-md`), container 8px
- Padding per segment: `px-4 py-2` (16px / 8px)
- Gap between segments: 0px ‚Äî segments are flush inside the container pill with no gap; segments divide the container width equally via `flex-1`
- Full width of form container (`flex w-full`)
- Height: 36px (aligns with TITLE_BAR_HEIGHT)

**Labels (exact copy):**
- Segment A: "Score my content"
- Segment B: "Remix a viral video"

**Default state:** "Score my content" selected (D-02 ‚Äî preserves current behavior)

**Accessibility:** `role="tablist"` on container, `role="tab"` + `aria-selected` on each segment, `aria-controls` pointing to the tab panel. Keyboard: arrow keys cycle, Enter/Space select.

**State management:** Local `useState<'score' | 'remix'>` in `ContentForm`, default `'score'`. Threads to `formData.mode` on change. Remix selection triggers tab-coupling (D-04/D-05, see below).

### 2. Remix Input Coupling

When "Remix a viral video" is selected (D-04/D-05):

- **Visible tabs:** Video only + Link only (tab order: 1. Video, 2. Link)
- **Hidden:** Text tab ‚Äî removed from DOM (not just visually hidden) when intent = remix
- **No caption field:** Video upload shows file picker only; no `video_caption` textarea
- **URL placeholder:** "Paste a TikTok URL to decode..."
- **Default tab in Remix:** Video (index 0)

When "Score my content" is selected (D-03):
- All three tabs visible (Video, Text, URL) in current order ‚Äî unchanged
- Caption field present for Video upload ‚Äî unchanged

Tab appearance: same `MODE_CONFIG` pill pattern from existing `content-form.tsx` bottom bar, moved inline below the intent selector.

### 3. Decode Frame Shell (Konva canvas + mobile card)

**Frame bounds (desktop Konva):** Reuses `verdict` bounds exactly: `{ x: 864, y: 0, width: 360, height: 280 }` (D-07). Falls back to auto-height via `AUTO_HEIGHT_FRAMES` as Phase 3 content lands.

**Frame ID in board-constants.ts:** `'decode'` replaces `'verdict'` in remix-mode layout. Label: `'Decode'`.

**Konva-renderable spec (canvas primitives):**

```
GroupFrame chrome (existing pattern):
  - Background rect: fill #18191a, cornerRadius 12, stroke rgba(255,255,255,0.06), strokeWidth 1
  - Title bar: height 36px, border-bottom rgba(255,255,255,0.05) 1px

Title bar content (Konva Text nodes):
  - Frame label "Decode": x=16, y=10, fontSize=12, fontStyle='500', fill='rgba(255,255,255,0.45)', letterSpacing=1, textTransform='uppercase'

Body content:
  - Descriptor Text: x=16, y=52, width=328 (frame width minus 2*16 padding)
    text: "Structural breakdown of why this video worked."
    fontSize=12, fontStyle='normal', fill='rgba(255,255,255,0.35)', lineHeight=1.4, wrap='word'
```

**DOM (mobile card):** `MobileFrameCard` with label `"Decode"`. Body:

```tsx
<p className="text-xs leading-[1.4] text-white/35 px-4 py-4">
  Structural breakdown of why this video worked.
</p>
```

No skeleton, no shimmer, no spinner, no "coming soon" text (D-10/D-11).

### 5. Board Mode Layout (resolveBoardLayout extension)

`resolveBoardLayout` gains a `mode: 'score' | 'remix'` parameter (D-08).

Score mode (unchanged): returns current 6-frame set including `verdict` + `actions`.

Remix mode: replaces `verdict` with `decode` (same x/y/w/h as `verdict`), replaces `actions` with `adapt` (same x/y/w/h as `actions`). All other frames (input, engine, audience, content-analysis) are positionally identical.

`GroupId` type extended: `'decode' | 'adapt'` added as valid IDs.

`AUTO_HEIGHT_FRAMES` updated to include `'decode'` and `'adapt'`.

---

### 6. Mobile Card Order (Remix mode)

Remix-mode `MOBILE_ORDER` (D-09):

```
['input', 'decode', 'audience', 'adapt', 'content-analysis', 'engine']
```

Score mode `MOBILE_ORDER` unchanged:

```
['input', 'verdict', 'audience', 'actions', 'content-analysis', 'engine']
```

`renderBody` switch in `BoardMobile.tsx` extended with `'decode'` and `'adapt'` cases rendering the respective shell components.

## States & Interactions

### Intent Selector States

| State | Trigger | Visual |
|-------|---------|--------|
| Score selected (default) | Page load / selecting Score | Score pill: `bg-white/[0.08]` text-foreground; Remix: transparent text-muted |
| Remix selected | Click Remix segment | Remix pill: `bg-white/[0.08]` text-foreground; Text tab disappears; caption textarea hidden |
| Hover | Pointer over inactive segment | `bg-white/[0.04]` text-foreground |
| Focus-visible | Keyboard Tab | `outline: 2px solid #FF7F50`, `outline-offset: 2px` |

### Submit Button States

Unchanged from existing. In Remix mode, `isSubmitDisabled` logic:
- Remix + Video: disabled until `formData.video_file != null`
- Remix + Link: disabled until `formData.tiktok_url.trim()` passes `SOCIAL_URL_PATTERN`

### Board Shell States

Decode + Adapt shells render only in remix mode. They have no loading, hover, or interactive states this phase. Static Konva rects + text. Mobile cards: same ‚Äî static, no tap action.

### Mode Persistence

- `mode` written to `analysis_results` row via `/api/analyze` (D-12)
- `/analyze/[id]` reads `mode` from row ‚Üí drives `resolveBoardLayout(mode)` (D-15)
- Live board and permalink reload must render identical frame set
