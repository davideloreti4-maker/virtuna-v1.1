# Phase 8 — UI Design Contract

> Visual and interaction contract for **Discover & Remix→Read — the competitor/niche moat chain**.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase does NOT introduce new design tokens.** It mounts on the locked
> THEME-06 flat-warm Raycast system (`BRAND-BIBLE.md` + `src/app/globals.css @theme` =
> SSOT). Rich output ships through the **fixed numen-rework typed-renderer library**
> (`message-blocks.tsx` + `blocks.ts`), NOT model-generated UI (craft trap) and NOT
> plain text (loses the moat). This contract is therefore mostly a **reuse map +
> two new typed blocks + one new browsable grid view**, not a net-new visual language.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (own token system — deliberate; shadcn gate N/A) |
| Preset | not applicable |
| Component library | Radix primitives (via `src/components/ui/*`) + project primitives (`GlassPanel`, `Button`, `Card`, `Badge`, `Skeleton`, `Spinner`) |

**Why no shadcn:** `components.json` absent by design. The project runs a mature
own-token system (THEME-06) and a fixed typed-renderer library. Introducing a
registry would violate the "no model-generated UI" + "fixed renderer" hard
constraint (STATE.md Hard Constraints). Registry safety gate = not applicable.

## Spacing Scale

Declared values (multiples of 4 — from `BRAND-BIBLE.md` Spacing Scale; do not invent new steps):

| Token | Value | Usage |
|-------|-------|-------|
| `--spacing-1` | 4px | Icon-to-text gaps, inline tag padding |
| `--spacing-2` | 8px | Compact gaps (metrics columns, chip padding) |
| `--spacing-3` | 12px | Standard card inner padding, block gaps (`gap-3` is the MessageBlocks default) |
| `--spacing-4` | 16px | Default card padding (VideoCard uses `p-4`), grid gutter |
| `--spacing-6` | 24px | Section padding, grid view outer margin |
| `--spacing-8` | 32px | Section breaks between Discover header and grid |
| `--spacing-16` | 64px | Empty-state vertical centering (`py-20` ≈ 80px is the existing empty-state exception) |

Exceptions:
- **Empty state** uses `py-20` (80px) per the existing `CompetitorEmptyState` — keep for consistency (it is a multiple of 4).
- **Mobile touch targets:** tile primary CTA + sheet/bottom action button height = **46px** (sketch 005 `.btn`), composer/input height = **42–44px** (BRAND-BIBLE inputs = 42px; sketch composer = 44px). 44px+ satisfies the iOS touch-target floor.
- **Outlier tile min-width** in the responsive grid: target ~160–172px columns (so ~2-up at 390px, 3–4-up on wider) — tune within D-16's ~20–30 tile budget.

## Typography

Mobile-first. Sizes drawn from `BRAND-BIBLE.md` type scale + sketch 005 (which is the locked card-scale reference). **Two weights only on these surfaces: 400 (regular) + 600 (semibold).** 500 is permitted ONLY for the existing reused `Button`/nav label tokens; new P8 surfaces use 400/600.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / quote | 14px (`--text-sm`) | 400 | 1.5 |
| Card hero line (adapted hook / Read interpretation) | 16px (`--text-base`) | 600 | ~1.42 (sketch `.hook`) |
| Verdict line / Read grade | 15px | 600 | 1.2 |
| Eyebrow / section label / persona tag | 10–12px (`--text-xs`) | 600 | 1.2 — `uppercase`, `letter-spacing 0.07em–0.1em` |
| Outlier multiplier (the "12x" number) | 20px (`--text-xl`) | 600 | 1.0 — `font-variant-numeric: tabular-nums` |
| Metric values (views/saves/shares) | 12px (`--text-xs`) | 400 | 1.2 — `tabular-nums` |

Notes:
- Quotes are **italic 14px** with a 2px left border (`border-l-2 border-white/[0.12]`), matching `RemixCardRenderer` blockquote — reuse verbatim for the verbatim quote wall.
- All numeric badges/stats use `font-variant-numeric: tabular-nums` (sketch 005 + board kit convention).

## Color

THEME-06 flat-warm Raycast tokens. **60/30/10 split is already enforced by the design system** — do not re-tint surfaces.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#07080a` (`--color-background`) | Page body, grid surface base |
| Secondary (30%) | `transparent` cards on `#18191a`/`#161718` surface + `rgba(255,255,255,0.06)` borders | Outlier tiles, Read card, quote-wall rows, persona drill panels |
| Accent (10%) | `#FF7F50` (`--color-accent` / coral-500) | See reserved list below |
| Destructive | `--color-error` (`oklch(0.60 0.20 25)`) | Error banner text/border only — no destructive user action exists in this phase |

**Accent (coral) reserved for — and ONLY for:**
1. The **single primary CTA** per card/tile: tile "Remix → Read", card "Develop into hooks →" (existing remix CTA), sticky/primary buttons.
2. The **one thing to act on** — the `Lever →` line + the `.read` interpretation highlight in the multi-audience Read card (sketch 005 `.read .tx b` + `.lever b`).
3. The **"Borrowed:" format chip** on the remix card (existing — the format seam).
4. Coral focus rings / `:focus-visible` ring on interactive elements.

**Coral is NEVER used for:**
- Verdict bands → use semantic status tokens: **Strong = `--color-success`**, **Mixed = `--color-warning`**, **Weak = `--color-error`** (matches `RemixCardRenderer.BAND_COLOR` exactly — reuse the map).
- Deltas / over-vs-under comparisons in metrics → green/red, never coral (board-kit accent law).
- The **calibration/weak-signal fallback** → `--color-warning`, never coral, never error (P7 07-05 locked decision; never fabricate personas).
- More than one element per card. **If two things are coral, one is wrong** (board-kit accent law — load-bearing).

**Outlier-score / honesty color rules (phase-specific):**
- The outlier multiplier badge ("12x") is **neutral/foreground**, NOT coral — it is data, not the action. Coral stays on the CTA.
- Niche-mode multipliers MUST render the label **"vs niche"**; profile-mode renders **"vs own"** (D-05). Never show a bare "12x" without its baseline label.
- Every SIM signal carries the **"SIM-1 Flash"** provenance tag (existing band-chip pattern). Never show a fabricated 0–100 score (D-11 honesty spine — bands only).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Discover primary entry (input placeholder) | `Paste a @handle or URL, or type a niche…` |
| Discover mode hint — profile | `Profile mode · ranked vs their own baseline` |
| Discover mode hint — niche | `Niche mode · ranked vs niche median` |
| Profile sub-tag | `Your channel` (own) / `Competitor` (other) — D-15 |
| Tile primary CTA | `Remix → Read` |
| Outlier badge | `{n}× vs own` (profile) / `{n}× vs niche` (niche) — never bare `{n}×` |
| Remix card CTA (reuse) | `Develop into hooks →` |
| Multi-audience verdict line | `{Audience}: {Strong\|Mixed\|Weak} {n}/10` per audience, e.g. `Growth: Strong 8/10 · Buyers: Weak 3/10` |
| The Read (delta one-liner) | `{Grade} Read.` + the delta interpretation, e.g. `Split Read. Wins for growth, bombs for buyers — hook indexes on novelty, buyers want proof.` |
| The Lever | `Lever → {what to change}` (e.g. `Keep the novelty, add a proof beat for buyers.`) |
| Who-it's-NOT-for line | `Scrolls past: {segment}` (derived from low-disposition personas, D-10 — no extra model call) |
| Audience picker | `Compare 2 audiences` · default pair label `{Active audience} vs General` |
| Steer tag on remixed output | `as your {audience}` (only when active audience ≠ General; omit for General no-op) |
| Verbatim quote-wall section labels | `Stopped the scroll` / `Scrolled past` (grouped by verdict, each quote tagged with its audience) |
| Empty state heading | `Find what's already working` |
| Empty state body | `Paste a creator's @handle or a niche to surface their outliers — the posts beating their own baseline. Then Remix the winner for your audience.` |
| Loading state (slow Apify pull) | `Pulling outliers… this can take a few minutes.` + tile skeletons (no fake progress %) |
| Error state | `Couldn't reach that source. Check the handle or try a different niche.` + `Retry` button (reuse ScrapeErrorBanner shape) |
| Empty grid (valid pull, no results) | `No standout outliers found for this {source}. Try a broader niche or another handle.` |
| Calibration / weak-signal fallback | `Directional — your audience is still calibrating.` (warning tone, never coral, never fabricate personas) |
| Honesty label (global) | `SIM-1 Flash` provenance tag on every band; `directional` / `calibrating` framing permitted; never a fabricated numeric score |
| Destructive confirmation | none — this phase has **no destructive user action** (no delete/save-shelf; saving is P10). Discover is read-only + chain-launch. |

## New Typed Blocks (the only net-new renderers this phase introduces)

> Both register in `blocks.ts` (Zod schema → `BlockUnionSchema`) and `message-blocks.tsx`
> (`BLOCK_COMPONENTS`). Model emits validated props ONLY; the renderer owns ALL layout
> (THREAD-04). Honesty spine: bands only, no numeric score; `model: z.literal("sim1-flash")`.

### B1 · `multi-audience-read` block (D-08/D-09/D-10/D-11 — the killer feature, static card)

Layout follows the LOCKED sketch 005 `.read` spine — **static card, no live cloud/toggle/chat** (P9 owns those).

- **Top — verdict row:** per-audience verdict line, capped at 2 audiences (D-09). Each = `{Audience}: {band} {n}/10`. Band word colored by `BAND_COLOR` map (success/warning/error). Default pair = active calibrated audience vs General (D-09 — doubles as proof calibration moves the verdict).
- **The Read (coral-bordered panel, reuse sketch `.read` styling):** grade glyph (`✦` Strong / `◐` Mixed/Split / `△` Weak) + one-line DELTA interpretation (`{Grade} Read.` + prose) + `Lever →` line. Coral border `rgba(255,127,80,0.30)`, bg `rgba(255,127,80,0.08)`, radius 12px.
- **Who-it's-NOT-for line:** `Scrolls past: {segment}` derived from low-disposition personas (D-10). Shown on both single- and multi-audience Read.
- **Expand — per-audience persona drill:** collapsible panel per audience showing its personas' stop/scroll verdicts (reuse `PersonasBlock` shape: archetype + verdict + quote). Read-only (persona values now real per W0 tuning — D-02).
- Provenance: `SIM-1 Flash` tag. No 0–100 number anywhere.

### B2 · `outlier-grid` block / Discover grid view (D-13/D-14/D-16)

The grid is its **own browsable view** (not a thread card), but tiles share the typed-tile shape so they can also be referenced in-thread.

- **One entry, two modes** (D-14): single input → `@handle`/URL = profile mode; free text = niche mode. Mode hint + baseline label update live.
- **Tiles:** ~20–30 (D-16), responsive CSS grid over the `DottedGrid` surface, ranked by outlier-multiplier (primary) + save-rate/share-rate tiebreak (D-06). Each tile = `VideoCard` metrics grid + outlier-multiplier badge (`{n}× vs own|niche`) + source tag (`Your channel`/`Competitor`/niche) + primary CTA `Remix → Read`.
- **States:** loading (tile skeletons, no fake %), empty/zero-result, error+retry — all per Copywriting Contract.
- Cached per `(input, day)` — re-opening does not re-scrape (D-16); no UI affordance needed beyond instant render on cache hit.

### B3 · `remix-card` reuse + steer tag (D-03)

- Reuse `RemixCardRenderer` **as-is**. The ONLY visible change from audience steering: when the active audience ≠ General, render an `as your {audience}` tag near the adapted-hook headline (small, muted, NOT coral — coral stays on the Borrowed chip + CTA). General audience = no tag (preserves the DEFAULT no-op so the engine regression gate stays green — D-03).
