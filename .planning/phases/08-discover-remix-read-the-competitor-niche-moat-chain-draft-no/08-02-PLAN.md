<objective>
W1 backend — the de-risk spine: swap the broken clockworks Discover actors to apidojo (D-12), remap the materially-different apidojo field names into the existing `VideoData`/`ProfileData` interfaces (Pitfall 1), and build the net-new pure outlier-score module + input classifier + per-day cache (D-03/04/05/06/07/14/16). This is the only genuinely new logic in the phase; it is small, pure, and tested in isolation before the grid (Plan 03) and the killer features (W3/W4) consume it.

Purpose: De-risk the all-new Apify/outlier path before the moat features. D-12 says "only the slugs swap" — true at the interface level, FALSE at the parsing level: apidojo returns `views`/`likes`/`shares`/`comments`/`bookmarks`/`uploadedAt`/`channel.followers` where clockworks returned `playCount`/`diggCount`/etc. A blind slug swap silently zeros every metric (Pitfall 1).

Output: an apidojo-shaped Zod remap emitting `VideoData`, the apidojo Discover actor wired to `scrapeVideos`/`scrapeProfile` only (Remix `resolveVideoUrl` left on its working actor — Pitfall 2), `rankOutliers`, `classifyDiscoverInput`, the per-day cache, and a `POST /api/discover` route that returns ranked tiles.
</objective>
