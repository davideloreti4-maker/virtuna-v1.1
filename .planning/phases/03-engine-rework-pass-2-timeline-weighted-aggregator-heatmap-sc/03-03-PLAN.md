**pg_cron cleanup (mirrors niche_post_windows pattern; from 03-RESEARCH.md lines 591-617):**
```sql
CREATE OR REPLACE FUNCTION cleanup_expired_filmstrips()
  RETURNS void LANGUAGE plpgsql SECURITY DEFINER
  SET search_path = storage, pg_temp
AS $$
BEGIN
  DELETE FROM storage.objects
  WHERE bucket_id = 'filmstrips'
    AND created_at < NOW() - INTERVAL '30 days';
END;
$$;

REVOKE EXECUTE ON FUNCTION cleanup_expired_filmstrips() FROM PUBLIC;
REVOKE EXECUTE ON FUNCTION cleanup_expired_filmstrips() FROM anon, authenticated;

DO $$
BEGIN
  IF EXISTS (SELECT 1 FROM cron.job WHERE jobname = 'cleanup-expired-filmstrips') THEN
    PERFORM cron.unschedule('cleanup-expired-filmstrips');
  END IF;
END $$;

SELECT cron.schedule(
  'cleanup-expired-filmstrips',
  '0 3 * * *',
  $$SELECT cleanup_expired_filmstrips()$$
);
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service-role → storage.objects | pg_cron SECURITY DEFINER function bypasses RLS to delete >30-day objects |
| authenticated user → outcomes table | RLS restricts SELECT to outcomes whose analysis_id belongs to the user; no client INSERT/UPDATE/DELETE policy = service-role only |
| anon → outcomes / filmstrips | no policies for anon role = denied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Information Disclosure (outcomes cross-tenant read) | outcomes table | mitigate | RLS policy `outcomes_select_own` joins via analyses.creator_id = auth.uid(); no public SELECT policy |
| T-03-03-02 | Tampering (client writes outcomes) | outcomes INSERT/UPDATE | mitigate | No INSERT/UPDATE/DELETE policies on outcomes → RLS denies by default; only service role can write (used by M2-III ingestion) |
| T-03-03-03 | Information Disclosure (filmstrip object enumeration) | storage.objects bucket=filmstrips | mitigate | Bucket public=false; no anon SELECT policy; access only via signed URLs (Plan 07 mints with 30d TTL per ASVS V6) |
| T-03-03-04 | Denial of Service (storage growth) | filmstrips bucket | mitigate | 5MB file_size_limit per object; pg_cron sweep deletes >30 days; 30d × analyses × ~10 frames each is bounded |
| T-03-03-05 | Elevation of Privilege (cron function abuse) | cleanup_expired_filmstrips() | mitigate | REVOKE EXECUTE FROM PUBLIC, anon, authenticated; SECURITY DEFINER with explicit `search_path = storage, pg_temp`; only pg_cron scheduler invokes it |
| T-03-03-06 | Tampering (allowed_mime_types bypass) | filmstrips bucket | mitigate | `ARRAY['image/jpeg']` limits accepted content-type at upload; ffmpeg produces JPEG only |
</threat_model>
