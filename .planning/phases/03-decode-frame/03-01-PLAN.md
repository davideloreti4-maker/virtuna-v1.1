<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Apify token → DashScope (Omni) | The Apify download token must never appear in the URL handed to Omni/DashScope (Phase 1 Option B). |
| Third-party MP4 → Supabase Storage | Source media is derived (temp re-host) and dropped; never persisted (IP boundary). |
| Omni `content_summary` → Qwen decode prompt | Untrusted transcribed text flows into the decode LLM call. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | resolve-and-rehost.ts (token leak) | mitigate | Token only on the server-side download fetch URL; the Omni-facing URL is a Supabase signed URL (no token). Acceptance criterion asserts this. |
| T-03-02 | Compliance / IP boundary | resolve-and-rehost.ts (source mp4 persistence) | mitigate | `cleanup()` unconditionally removes the temp object; never set `video_storage_path`. grep gates assert. |
| T-03-03 | Tampering (prompt injection) | decode-prompts.ts (content_summary into prompt) | mitigate | Omni fields treated as data; output constrained to a fixed Zod-validated JSON schema; no tool-calls; low blast radius. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages this phase (milestone-locked); all modules first-party or already-installed. No install task; no legitimacy gate needed. |
