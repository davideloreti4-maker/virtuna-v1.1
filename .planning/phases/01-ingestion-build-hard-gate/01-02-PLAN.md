## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pasted TikTok URL ‚Üí Clockworks actor | user-supplied third-party URL drives a billable scrape |
| Apify-resolved mp4 URL ‚Üí downstream fetch (Plan 03 Omni/ffmpeg) | the resolved URL is attacker-influenced; it leaves this method and is fetched server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering / Info Disclosure (SSRF) | resolveVideoUrl return value | mitigate | Validate `new URL(mp4)`: require `https:` + hostname in the TikTok/Apify-CDN suffix allowlist (derived from spike-observed hosts); reject `file:`, internal IPs, non-allowlisted hosts with IngestError("ssrf_rejected") BEFORE the URL is returned. Tested in resolve-video.failures.test.ts. |
| T-01-06 | Info Disclosure (error leakage) | bare Error throws | mitigate | Replace the bare `throw new Error` (apify-provider.ts:33) with typed IngestError carrying a `kind` discriminant only ‚Äî Phase 2 maps kinds to honest copy without leaking actor internals. |

> Cost-exhaustion (per-request Apify spend) is mitigated at the route boundary in Plan 03 (T-01-05), where the 429 rate-limit pattern lives.
