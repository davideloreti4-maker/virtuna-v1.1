### 1. Magic Card

Install command:
```
npx shadcn@latest add https://magicui.design/r/magic-card.json
```
Target: src/components/magic-ui/magic-card.tsx

Dependencies declared in registry: motion (already installed), next-themes
Safety gate (2026-05-11): view passed — no flags — no fetch(), no eval(),
no process.env, no external dynamic imports. Uses standard pointer events
and motion/react only.

**Tuned defaults (override in source file after install):**

| Prop | Stock default | Tuned value | Rationale |
|------|--------------|-------------|-----------|
| gradientFrom | "#9E7AFF" (violet) | "#FF7F50" (coral-500) | Strip violet; use Raycast accent |
| gradientTo | "#FE8BBB" (pink) | "rgba(255,127,80,0.15)" | Fade coral; no pink bleed |
| gradientColor | "#262626" | "#18191a" (--color-surface) | Match surface token |
| gradientOpacity | 0.8 | 0.6 | Reduce intensity to Raycast subtlety level |
| gradientSize | 200 | 200 | Acceptable; keep |

**Visual behavior after tuning:**
- Resting: card with 1px border rgba(255,255,255,0.06), transparent bg,
  12px radius (rounded-lg), inset shadow rgba(255,255,255,0.05) 0 1px 0 0
- Hover: radial coral spotlight tracks mouse cursor at 60% opacity —
  subtle warmth, not neon glow
- Border: uses gradient border on hover tracking (the clip-path border
  technique); outer border animates via motionTemplate to coral — this
  replaces the static Raycast border on cards. Acceptable for Phase 1
  showcase; Phase 2 decides if hero tiles use this.
- Motion: uses motion/react useMotionValue + useSpring (stiffness 250,
  damping 30, mass 0.6) — spring feels natural and not maximalist
- `'use client'` required (confirmed: file ships with "use client" directive)
- useTheme dependency: next-themes must be a project dep or installed.
  Check package.json; if absent, install. The dark/light theme logic
  defaults to dark correctly for Virtuna (dark-mode-first).

**GlassPanel compatibility:**
- Magic Card does NOT use backdrop-filter, does NOT compete with GlassPanel
- These two can compose: GlassPanel wrapping Magic Card, or Magic Card
  independently on a dark bg. No conflict.

**Reduced-motion spec:**
- The component has no explicit prefers-reduced-motion guard in source.
- Executor must add: `const prefersReducedMotion = usePrefersReducedMotion()`
  (reuse `src/hooks/usePrefersReducedMotion.ts` verbatim per CONTEXT.md).
  When true: disable mouseX/mouseY spring tracking, render static card.

**SSR safety:**
- mouseX/mouseY initialized to -gradientSize (off-card) — no flash on SSR.
  No useId needed here; no module-level counters in source.

### 2. Border Beam

Install command:
```
npx shadcn@latest add https://magicui.design/r/border-beam.json
```
Target: src/components/magic-ui/border-beam.tsx

Dependencies declared in registry: motion (already installed)
Safety gate (2026-05-11): view passed — no flags — no fetch(), no eval(),
no process.env, no external dynamic imports. Pure motion/react animation.

**Tuned defaults (override in source file after install):**

| Prop | Stock default | Tuned value | Rationale |
|------|--------------|-------------|-----------|
| colorFrom | "#ffaa40" (amber) | "rgba(255,127,80,0.9)" (coral-500 at opacity) | Strip amber; use coral |
| colorTo | "#9c40ff" (violet) | "rgba(255,127,80,0)" (coral fade to transparent) | Strip violet; coral-to-transparent sweep |
| duration | 6s | 8s | Slow down; maximalist beam idiom is <6s. Raycast pace is 8-12s. |
| size | 50 | 40 | Slightly smaller beam head; less eye-catching |
| borderWidth | 1 | 1 | Keep; matches Raycast 1px border standard |
| delay | 0 | 0 | Keep |

**Visual behavior after tuning:**
- A coral-to-transparent arc sweeps clockwise along the 1px card border
  at 8s/cycle, infinite
- Effect reads as "active" or "highlighted" state, not as neon decoration
- Reserved use: CTA card borders, Pro-tier pricing card (Phase 6), hero
  tile accent (Phase 2 decides). In showcase: demonstrate on 1 card only.
- `'use client'` required (confirmed: file ships with "use client" directive)

**Reduced-motion spec:**
- Component uses `repeat: Infinity` on motion animation.
- Executor must add: `if (prefersReducedMotion) return null;` at top of
  component render. Alternatively wrap in `motion-safe:` Tailwind class.
  Source uses `motion/react` — use `useReducedMotion()` from motion/react.

**GlassPanel compatibility:**
- Border Beam is an `absolute inset-0` child overlay with pointer-events-none.
- Composes cleanly inside any `relative` container including GlassPanel.
- Ensure parent has `overflow-hidden` and `relative` (standard card pattern).

**SSR safety:**
- motion animation runs client-only ('use client' + motion/react handles this).
  No SSR flash; beam is invisible on first paint and fades in on mount.

### 3. Shine Border

Install command:
```
npx shadcn@latest add https://magicui.design/r/shine-border.json
```
Target: src/components/magic-ui/shine-border.tsx

Dependencies declared in registry: none beyond React
Safety gate (2026-05-11): view passed — no flags — no fetch(), no eval(),
no process.env, no external dynamic imports. Pure CSS mask technique.

**Tuned defaults (override in source file after install):**

| Prop | Stock default | Tuned value | Rationale |
|------|--------------|-------------|-----------|
| shineColor | "#000000" (black) | ["rgba(255,127,80,0.8)", "rgba(255,127,80,0.15)", "rgba(255,127,80,0.8)"] | Coral sweep; array form for smooth gradient arc |
| duration | 14s | 18s | Slower than stock; Raycast idiom is restrained motion |
| borderWidth | 1 | 1 | Keep; matches Raycast 1px |

**Visual behavior after tuning:**
- A rotating coral arc sweeps along the full border perimeter at 18s/cycle
- Appears as a subtle "selected" or "featured" card treatment
- Reserved use in Phase 1: Showcase demonstration only.
  Phase 6 uses it for the Pro pricing card border (pre-declared in D-01).
- No JavaScript interactivity (pure CSS radial-gradient + mask-composite)
- `'use client'` required (confirmed: file ships with "use client" directive)

**Reduced-motion spec:**
- Uses CSS `motion-safe:animate-shine` class (already in source). The
  `motion-safe:` Tailwind variant correctly suppresses the animation when
  `prefers-reduced-motion: reduce` is active. No JS change needed.
- When reduced: static 1px border rgba(255,255,255,0.06) only; no shine arc.

**@keyframes injection:**
- Registry declares `cssVars.theme["animate-shine"]` and `css["@keyframes shine"]`.
- Tailwind v4 compatibility: the shadcn CLI handles injecting these into globals.css.
- Executor must verify `@keyframes shine` is present in globals.css after install.
- If not injected automatically, manually add the keyframe from the registry spec:
  ```css
  @keyframes shine {
    0% { background-position: 0% 0%; }
    50% { background-position: 100% 100%; }
    to { background-position: 0% 0%; }
  }
  ```

**GlassPanel compatibility:**
- ShineBorder is `absolute inset-0` with `pointer-events-none`. Composes
  inside relative containers. Same rule as Border Beam: parent needs
  `overflow-hidden` and `relative`.

**SSR safety:**
- Pure CSS animation; no motion/react. No SSR flash concern. Works on
  first paint (animation just starts from 0% background-position).
