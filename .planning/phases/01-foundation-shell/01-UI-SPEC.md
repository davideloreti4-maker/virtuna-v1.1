## Spacing Scale

8px-base scale, ported verbatim from globals.css `@theme` (D-07 — carries unchanged). All values are multiples of 4.

| Token | CSS var | Value | Usage |
|-------|---------|-------|-------|
| 1 | `--spacing-1` | 4px | Icon gaps, hairline inset |
| 2 | `--spacing-2` | 8px | Compact element spacing, base unit |
| 3 | `--spacing-3` | 12px | Tight stacks, chip padding |
| 4 | `--spacing-4` | 16px | Default element spacing, body gaps |
| 5 | `--spacing-5` | 20px | — |
| 6 | `--spacing-6` | 24px | Card padding, header inline padding |
| 8 | `--spacing-8` | 32px | Layout gaps |
| 10 | `--spacing-10` | 40px | — |
| 12 | `--spacing-12` | 48px | Major section breaks |
| 16 | `--spacing-16` | 64px | Page-level section vertical rhythm |
| 20 | `--spacing-20` | 80px | Generous section breaks (calm whitespace) |
| 24 | `--spacing-24` | 96px | Hero / large section padding |

**Header height:** 56–64px sticky bar (use `--spacing-16` ≈ 64px max), hairline bottom border, inline padding `--spacing-6` (24px) desktop / `--spacing-4` (16px) mobile.
**Section vertical rhythm (scroll-skeleton stubs):** target `--spacing-20` (80px) to `--spacing-24` (96px) top/bottom per section so anchored sections feel calm (Linear/Claude.ai north star). Planner sets exact per-section values.

**Exceptions:**
- Mobile tap targets ≥ 44px (icon-only triggers, mobile menu button) — round up to 44px even though it falls off the 4px grid for accessibility (FOUND-07 / NAV-03).

## Typography

Inter scale ported verbatim from globals.css (D-07). For this chrome-only phase, declare **4 sizes + 2 weights** as the working set. The full token ramp (`--text-xs`…`--text-display`) survives in `@theme`; sections in later phases draw from it.

| Role | Size | CSS var | Weight | Line Height | Usage (Phase 1 chrome) |
|------|------|---------|--------|-------------|------------------------|
| Body | 16px | `--text-base` | 400 (`--font-regular`) | 1.5 (`--leading-normal`) | Footer copy, nav links, default text |
| Label / small | 14px | `--text-sm` | 600 (`--font-semibold`) | 1.5 | Button labels, footer column headings, mono dimension hint base |
| Subhead | 20px | `--text-xl` | 600 (`--font-semibold`) | ~1.25 (`--leading-snug`) | Footer brand line, any stub section eyebrow/label |
| Heading | 30px | `--text-3xl` | 600 (`--font-semibold`) | 1.2 (≈`--leading-tight` 1.1 / `--leading-snug` 1.25) | Stub section placeholder headings (real headings + the serif hero land Phase 2) |

**Weights:** exactly two in use — **regular 400** (body) and **semibold 600** (labels, buttons, headings). `--font-medium` (500) and `--font-bold` (700) exist in `@theme` but are NOT part of the Phase-1 chrome working set (NumenLogo wordmark already ships its own `font-bold` — leave as-is, it is brand-locked).
**Line height:** body **1.5**, headings **~1.2**.
**Letter-spacing:** global `letter-spacing: 0.2px` on `html,body` (carried from base styles); headings may use `--tracking-tight` (-0.02em); logo wordmark uses its locked `-0.02em`.
**Serif:** Newsreader is **reserved, not rendered** in Phase 1 chrome. Wire `--font-newsreader` in root layout exactly as numen-rework (`next/font/google`, `style: ["normal","italic"]`, `weight: ["400"]`, add `${newsreader.variable}` to `<html>`). Already present in this worktree's `src/app/layout.tsx` — verify, don't duplicate.

## Color

60 / 30 / 10 split, flat-warm — neutral charcoal surfaces, cream text, a single terracotta-clay coral accent. **Flat-matte: NO glass, NO backdrop-blur, NO inset white shine, NO glow/halo/ambient lighting** (D-06). Values ported verbatim from numen-rework globals.css.

| Role | Value | CSS var (semantic) | Usage |
|------|-------|--------------------|-------|
| **Dominant (60%)** | `#262624` | `--color-background` → `--color-charcoal-app` | Page background, the dominant app surface (replaces cold `#07080a`) |
| **Secondary (30%)** | `#1a1a18` | `--color-background-elevated` → `--color-charcoal-sidebar` | Header bar / footer surface, one tone-step off the page |
| Secondary (card) | `#1e1d1b` | `--color-surface` → `--color-charcoal-composer` | Card / composer surfaces |
| Secondary (lifted) | `#2f2e2b` | `--color-surface-elevated` → `--color-charcoal-chip` | Chips, lifted matte surfaces, **placeholder-slot chip surface** |
| **Accent (10%)** | `#d97757` ≈ `oklch(0.68 0.13 33)` | `--color-accent` → `--color-coral-500` | See reserved-for list below |
| Accent hover | `oklch(0.76 0.11 33)` | `--color-accent-hover` → `--color-coral-400` | Primary CTA hover |
| Accent active | `oklch(0.58 0.115 33)` | `--color-accent-active` → `--color-coral-600` | Primary CTA active/pressed |
| Accent foreground | `#1a0f0a` | `--color-accent-foreground` | Dark-brown text/label ON the terracotta CTA |
| Text primary | `#ece7de` | `--color-foreground` → `--color-cream-primary` | Brightest text (never `#fff`) |
| Text secondary | `#c2bdb4` | `--color-foreground-secondary` → `--color-cream-secondary` | Default chrome / nav links |
| Text muted | `#8a857c` | `--color-foreground-muted` → `--color-cream-muted` | Placeholder caption, footer fine print, mono dimension hint, em-dash empties |
| Border | `rgba(255,255,255,0.06)` | `--color-border` | Universal hairline (header bottom, card/slot edge, footer divider) |
| Border hover | `rgba(255,255,255,0.10)` | `--color-border-hover` | Interactive hover edge |
| Destructive | `oklch(0.60 0.20 25)` | `--color-error` | **NOT used in Phase 1** (no destructive actions). Listed for completeness; stays distinct from coral (hue 25 vs 33). |

**Accent reserved for (the ONLY places terracotta `#d97757` appears — D-04):**
1. The Numen logo mark (`NumenLogo` / `NumenMark` color)
2. The primary **"Try it free"** CTA (fill = accent, label = accent-foreground `#1a0f0a`)
3. The focus ring (`focus-visible` outline on all interactive elements — keyboard a11y)

Accent is **never** used for body text runs, secondary nav links, the "Sign in" secondary link, borders, or generic hovers. Secondary/ghost interactions use cream text + `--color-border` / `--color-hover` `rgba(255,255,255,0.05)`.

**Elevation = tone-step + hairline border only.** The single permitted shadow is `--shadow-float` (`0 10px 30px rgba(0,0,0,0.35)`), and only on genuinely floating surfaces (e.g. the mobile-nav dropdown panel, popovers). Header/footer/cards/slots rest on border + tone — no drop shadow. **Removed tokens — do NOT reintroduce:** `--gradient-navbar`, `--gradient-glass`, `--gradient-feature`, `--shadow-glass`, `--shadow-glow-accent`, and the inset-shine layer of `--shadow-button` (D-06). `--shadow-button` is now flat-matte: `rgba(0,0,0,0.5) 0 0 0 2px, rgba(0,0,0,0.2) 0 1px 2px 0` — no white-shine layers.

### Contrast verification (WCAG, computed at build-equivalent values)

| Pair | Ratio | Verdict |
|------|-------|---------|
| accent-foreground `#1a0f0a` on terracotta `#d97757` (CTA label) | **6.02:1** | Passes **AA** (≥4.5). **Below AAA** (≥7). ⚠ See note. |
| cream-primary `#ece7de` on `#262624` (body) | 12.31:1 | AAA ✓ |
| cream-secondary `#c2bdb4` on `#262624` (nav links) | 8.11:1 | AAA ✓ |
| cream-muted `#8a857c` on `#262624` (captions/fine print) | 4.13:1 | AA large/UI (≥3) ✓; below AA-normal (4.5) — use for ≥18px or non-essential text only |
| terracotta `#d97757` on `#262624` (accent text/icon, if ever) | 4.86:1 | AA ✓ |

⚠ **Contract note (resolve at build):** CONTEXT D-04 / the globals.css comment ask to "verify AA ≥7:1" on the terracotta CTA — that target conflates AA (4.5) with AAA (7). The actual ratio is **6.02:1**: it **clears WCAG AA** (the milestone's stated bar, FOUND-07) but **does not reach 7:1 AAA**. Accept as AA-passing (do not block), OR if 7:1 is genuinely wanted, darken accent-foreground (e.g. toward `#140a06`) — executor's call, but **AA is met as-is**, so no change is required for the requirements. `cream-muted` must not carry essential small body copy below 18px (use cream-secondary there).

## Component Inventory (Phase 1)

New marketing components live in **`src/components/marketing/`** (D-15) — keeps the milestone surface out of the shared design system. Follow `ui/` conventions: kebab-case files, CVA + `clsx` + `tailwind-merge` (`cn()` from `@/lib/utils`), barrel `index.ts`.

### 1. `<Placeholder>` — the reusable slot (FOUND-03, D-12/D-13/D-14)

```tsx
<Placeholder
  variant="image" | "video" | "avatar" | "logo"
  aspect="16/9"            // any CSS aspect-ratio string; locks reserved space
  label="Hero demo"        // cream-muted caption
  src?="/real.png"         // when present → renders real <img>/<video>; absent → labelled stand-in
/>
```

- **Look (flat-warm):** charcoal **chip** surface `--color-surface-elevated` (`#2f2e2b`), hairline **6% border** (`--color-border`), **12px radius** (`--radius-lg`). Centered, **low-opacity** media-type icon (lucide: `image`→`Image`, `video`→`Video`, `avatar`→`UserRound`, `logo`→`Building2`; opacity ~0.25–0.35, color `--color-foreground-muted`). Cream-muted (`#8a857c`) `label` caption beneath the icon. Faint **mono** dimension hint (font `--font-mono`, `--text-xs`, cream-muted, lower opacity) showing the resolved aspect (e.g. `16:9`).
- **NO dashed-wireframe border** (reads "cheap/unfinished"), **no glass, no shine, no glow** (D-13).
- **No layout shift (D-14, sets up FOUND-06):** lock dimensions with CSS **`aspect-ratio`** so every slot reserves its box before any asset loads. `avatar` variant defaults to a circular `aspect="1/1"` with `--radius-full`; `logo` to a wide short box; `image`/`video` default `16/9`.
- **Variant overrides:** `avatar` → circle (`rounded-full`, `1/1`); `logo` → no caption icon-only is acceptable, muted; `video` → may show a centered play-triangle affordance (static, muted) over the media-type icon.
- **Optional motion:** a very-subtle `skeleton-breathe` (keyframe already in globals.css, 0.4↔0.6 opacity) on the icon/surface, **reduced-motion gated** — `motion-reduce:animate-none` (matches existing `ui/skeleton.tsx` pattern) AND skip when `usePrefersReducedMotion()` is true. Default OFF unless the planner opts a slot in.
- **`src` is the one-prop swap** the brief requires — no central asset-registry in v1 (registry deferred, see CONTEXT `<deferred>`).

### 2. `<Header>` — flat sticky chrome (NAV-01, NAV-03, D-19/D-20/D-21)

- **NOT** the old glass pill. A **flat sticky bar**: solid secondary-tone bg (`--color-background-elevated` `#1a1a18` or app `#262624` — planner picks; must be **opaque**, no `backdrop-blur`), **hairline bottom border** (`--color-border`), **no shine, no drop shadow**.
- Contents: `NumenLogo` (left; mark + "Numen" wordmark, color inherits cream via `currentColor`) · 3–4 in-page anchor links (cream-secondary, hover cream-primary or `--color-hover`) · **"Try it free"** primary button (accent fill `#d97757`, label `#1a0f0a`, `--radius-md` 8px, `--shadow-button` flat-matte) · **"Sign in"** subtle secondary link (cream-secondary, ghost — no accent).
- **Sticky** at `--z-sticky` (200). Height 56–64px.
- **Mobile collapse (NAV-03, D-21):** simple flat collapse — a lightweight dropdown/disclosure panel (NOT a heavyweight Radix `Sheet` for 3–4 items; `ui/sheet.tsx` exists but D-21 says don't use it here). Trigger = icon button (lucide `Menu`/`X`), `aria-label` per copy table, **closes on tap** of any link. Panel = opaque charcoal, hairline border, `--shadow-float` permitted (it floats). Keep the "Try it free" CTA reachable on mobile (in-panel or persistent). Tap targets ≥ 44px.

## Motion Contract (FOUND-04, D-16/D-17/D-18)

- **One library:** `motion/react`. All new imports standardize here. The 6 existing `motion/*` wrappers already import `motion/react` — migration effectively complete. Remove redundant `framer-motion` dep only after grep confirms zero imports (else defer to Phase 5 — see `<deferred>`).
- **Global reduced-motion (two-layer):**
  1. Wrap the landing tree in **`<MotionConfig reducedMotion="user">`** (place at the marketing layout/page root) — every `motion` element auto-respects the OS setting. Not yet used anywhere in this worktree; introduce it here.
  2. Add a CSS **`@media (prefers-reduced-motion: reduce)`** block in `globals.css` for **non-Framer** animations (marquee, shimmer, `skeleton-breathe`) — set `animation: none` / minimal duration. Pair with the existing `motion-reduce:animate-none` Tailwind utility on those elements.
- **Every animation MUST have a reduced-motion fallback** (hard requirement — FOUND-04, FOUND-07). No motion is load-bearing for content or first paint.
- **Phase-1 taste bar:** calm, subtle entrances only (fade / fade-up / slide-up, gentle stagger). Heavy "wow" libs (Magic UI / Aceternity) are pulled **per-component in later phases** (the signature "crowd → score" moment is Phase 2) — not in the foundation.
- **Reserved/floating elements** use `--shadow-float` only; nothing else casts shadow.
