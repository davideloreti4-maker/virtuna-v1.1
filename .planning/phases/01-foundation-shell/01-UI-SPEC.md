# Phase 1 — UI Design Contract

> Visual and interaction contract for the **flat-warm visual system** + the **app shell** the Simulation thread lives in (home, composer, sidebar). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **DIRECTION vs PIXELS.** This contract is binding for *direction + interaction*. Per **D-08**, the exact hex ramp, exact serif typeface, and exact coral hex are decided at build and **signed at the THEME-06 human-UAT gate on the BUILT shell** — they are captured here as the approved starting point (Frame A), NOT as immutable pixel law. Fields flagged `[UAT]` lock at the gate, not here.
>
> **Source of truth:** `.planning/NUMEN-REWORK-BRIEF.md` (LOCKED) overridden on 3 Phase-1 points by `.planning/phases/01-foundation-shell/01-CONTEXT.md` (25 locked decisions D-01..D-25 — CONTEXT wins for Phase 1). Approved pre-viz: `.planning/sketches/01-base-hue/swatch.html` Frame A.

Presentation-layer only. Engine FROZEN 3.19.0. Work in `src/app/globals.css` (token SSOT), `src/components/sidebar/Sidebar.tsx`, `src/components/app/app-shell.tsx`, the new home/composer surfaces, and the named hooks/stores. Mobile-first; desktop is the same shell widened.

## Spacing Scale

Existing `@theme` scale in `globals.css` is already a superset of the 8-point system — **reuse it, do not redefine.** All multiples of 4 (with one 12px intermediary).

| Token | Value | Usage |
|-------|-------|-------|
| `--spacing-1` | 4px | Icon gaps, inline micro-padding |
| `--spacing-2` | 8px | Compact element spacing, chip/score-chip gaps |
| `--spacing-3` | 12px | Sidebar row padding, composer inner gaps |
| `--spacing-4` | 16px | Default element spacing, composer padding |
| `--spacing-5` | 20px | — |
| `--spacing-6` | 24px | Section padding, sidebar section breaks |
| `--spacing-8` | 32px | Layout gaps, greeting→composer gap |
| `--spacing-12` | 48px | Major vertical rhythm on the empty home |
| `--spacing-16` | 64px | Page-level breathing space (THEME-05 "generous space") |

**Exceptions:**
- **Touch targets:** interactive controls (composer `+`, sidebar rows, collapse toggle, mobile hamburger) maintain a **≥44px** hit area on mobile even when the visual element is smaller (pad the hit zone, not the glyph).
- **Content column cap (D-17):** composer + thread main area cap at a **~760px** centered readable column (Claude-style). This is a max-width constraint, not a spacing token. Mobile fills width with `--spacing-4` side gutters; rich visuals may break wider in later phases.
- **Sidebar width:** 220px expanded (matches current `Sidebar.tsx` `w-[220px]`); ~56–64px icon rail when collapsed (D-14).

## Typography

Two type families, strict role split (THEME-04 / D-03): **serif = voice moments ONLY** (greeting + hero line); **Inter = ALL data, UI, chrome, numbers.** Tabular numerals (`font-variant-numeric: tabular-nums`) for score chips.

Sizes drawn from the existing `--text-*` scale + the Frame A swatch greeting (38px serif). Weights restrained to **2 per family** (Linear/Things restraint, THEME-05).

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Display (greeting / hero line) | **Serif** [UAT] | **[UAT] ~38px** (`--text-4xl` 36px ↔ 40px; Frame A = 38px) | 400 (regular) — italic on the name accent | 1.1 (`--leading-tight`) | The one serif voice moment on the home (D-19). Scales down on mobile (~28–30px). |
| Body / composer input | Inter | 16px (`--text-base`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) | Composer text, placeholders, prose-free chrome copy |
| Label / UI | Inter | 14px (`--text-sm`) | 500 (`--font-medium`) | 1.4–1.5 | Sidebar items, nav labels, buttons, account row |
| Caption / meta | Inter | 12px (`--text-xs`) | 500 (`--font-medium`) | 1.4 | Section labels ("Simulations"), relative timestamps, score chips (tabular-nums) |

**Rules:**
- **No serif anywhere except greeting + hero line.** Score numbers, sidebar Simulation titles, button labels, errors — all Inter.
- **Body letter-spacing:** keep the established `0.2px` body tracking (`globals.css` base) for Inter; greeting serif at `0` tracking.
- **Weights:** Inter uses **400 + 500** as the working pair for this phase (600 available for emphasis but used sparingly; do not introduce a 3rd routine weight). Serif uses **400** (regular + italic) only.
- **Text color is never pure white** (D-02): cream/warm off-white `~#ece7de` is the brightest text. See Color.

## Color

The defining move: **surfaces are perceptually NEUTRAL charcoal, NOT warm-brown** (D-01/D-02). Warmth is delivered by **cream text + the lone coral accent**, never by the surface hue. Matte throughout — depth from tone-step + hairline borders, **no glow / shine / halo / ambient lighting** (D-05, THEME-01/05). Color appears **only in the data** (score zones) and the single coral accent (THEME-03).

All values below are the **approved Frame A starting point**; `[UAT]` = locks at THEME-06 on the built shell (D-08). **Dark surface tokens MUST be authored as exact HEX**, never oklch — oklch with L<0.15 compiles wrong in Tailwind v4 `@theme` (CLAUDE.md gotcha).

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface (60%) | **[UAT] `#262624`** (app bg) | Main content area, page background. Replaces cold `--color-background` `#07080a`. |
| Secondary surface (30%) | **[UAT] `#1a1a18`** (sidebar) · `#1e1d1b` (composer) · `#2f2e2b` (chip) | Sidebar, composer card, chips, popovers — one tone-step off the dominant for matte elevation (no shine). |
| Accent (10%) | **[UAT] `#d97757`** (terracotta/clay coral, hue ~33–37) | See reserved-for list. The lone brand accent. Matures the current coral (`oklch 0.72 0.16 40`) warmer/deeper/desaturated; stays distinct from the alert-red score-zone (D-04). |
| Accent foreground | `#1a0f0a`→**[UAT]** retune (current dark-brown, verify AA on new coral) | Text/glyph sitting ON coral (CTA label, avatar initial). |
| Primary text (cream) | **[UAT] `#ece7de`** | Brightest text — **never pure white** (D-02). Greeting, active items, primary copy. |
| Secondary text | **[UAT] `#c2bdb4`** | Default sidebar/chrome text, composer typed text. |
| Muted text | **[UAT] `#8a857c`** | Placeholders, section labels, timestamps, the "—" empty score. |
| Hairline border | `rgba(255,255,255,0.06)` (≈0.07 in Frame A) · hover `0.10` | Universal borders + dividers (matte elevation, not shadow). Keep existing `--color-border` scale. |
| **Score zone — green** | `--color-success` `oklch(0.68 0.17 145)` (Frame A chip `oklch(0.74 0.15 145)`) [UAT] | Score chips in sidebar Simulation rows. Strong band. |
| **Score zone — amber** | `--color-warning` `oklch(0.75 0.15 85)` [UAT] | Mid band score chips. |
| **Score zone — red** | `--color-error` `oklch(0.60 0.20 25)` [UAT] | Weak band / alert. **Must stay perceptually distinct from coral accent** (D-04) — coral is hue ~35, alert-red hue ~25. |

**Accent (coral) reserved for — EXPLICIT list (never "all interactive elements"):**
1. The **New Simulation** primary CTA (sidebar top).
2. The **NumenMark** stele glyph / brand mark.
3. **Focus rings** (keyboard focus state, accessibility).
4. The **active New-Simulation nav** state + account-avatar fill (per Frame A).

Everything else neutral. Score zone colors (green/amber/red) are **data colors**, semantically separate from the coral brand accent — they appear only on score chips/values, never as chrome.

**Floating-element shadow (the ONLY shadows allowed, D-05):** a whisper-soft drop shadow on *genuinely floating* surfaces — the centered composer and popovers (Frame A composer = `0 10px 30px rgba(0,0,0,0.35)`). NO inset white shine, NO glow, NO halo, NO `--shadow-glass`/`--shadow-glow-accent`/`--gradient-feature`. Resting cards/sidebar use tone-step + hairline only.

The Raycast 137deg glass must be removed at **two layers**. This is a contract item the checker/auditor verifies; the planner must task both.

**Layer A — token level (`src/app/globals.css`):**
| Token | Line (approx) | Action |
|-------|------|--------|

| `--gradient-navbar` | 194 | Remove / flatten |
| `--gradient-card-bg` (137deg) | 192 | Remove / flatten to solid charcoal |
| `--gradient-feature` (radial halo) | 196 | Remove (vetoed ambient lighting) |

**Layer B — inline styles in `Sidebar.tsx` (hardcoded glass — token swap alone will NOT fix these):**
| Spot | Line (approx) | Current | Action |
|------|------|---------|--------|

## Layout & Interaction Contract

### Home — empty state (SHELL-01 + D-18 override)
- **Greeting + composer ONLY. NO starter chips** (D-18 — intentional override of SHELL-01's `Paste · Upload · Try a demo` 3-chip spec and of the Frame A swatch's chip row). Rationale: the composer already affords paste (placeholder) + upload (`+`); chips are redundant. Clean Claude-style start.
- **No demo chip/affordance** on the P1 home (D-25 — demo is Phase 5).
- **No Simulation list under the composer** (it lives in the sidebar).
- Composer **centered** vertically in the ~760px column; serif greeting above; **NumenMark stele glyph** (not an asterisk — D-20; the swatch's `✳` is a placeholder).
- Greeting copy: serif voice moment, name from `useProfile`.

### Composer — two layouts (SHELL-04 / D-24)
- **Centered-when-empty → bottom-pinned-when-active.** Same composer component serves both states and (Phase 5) follow-ups — **no separate dock**.
- Controls: `+` = upload (D-22); paste a URL → auto-detect (SHELL-02/03).
- Floating treatment (whisper shadow) when centered; pinned to bottom of the column when a Simulation exists.
- Submit → create a new Simulation → navigate to its **permalink** where the composer drops to bottom-pinned (the Claude "new chat → /chat/[id]" model, D-23). Exact route paths = planner's call. *What renders above is Phase 2.*

### Sidebar — lean & clean (SHELL-05/06 + D-10..D-16)
- **Composition (D-10):** New Simulation CTA (coral, top) → past-**Simulations** list (score chips + remix tag) → Settings/Account at bottom. Keep the **@handle selector** (`SidebarAccountSelector`, D-12) reskinned flat.
- **Cut (D-11):** remove Pinned stub, Projects "Soon" placeholder, and the "Boards" nav. **No dead affordances.**
- **History rows (D-13):** relabel "Recent" → **"Simulations"**; keep score chips + remix tag; rows route to the new Simulation permalink (NOT `/analyze`). Reuse `useAnalysisHistory` / `use-sidebar-queries.ts`.
- **Desktop (D-14):** **persistent + collapsible** — content shifts right (no overlap); collapse to icon rail via **⌘\**. **Revive the dead collapse code** — `Sidebar.tsx` line 358 hardcodes `effectiveCollapsed = false`; the sidebar is `fixed … overlay`-only on all viewports today. Wire `app-shell.tsx` `--sidebar-offset` (currently `0px`) to the real expanded sidebar width.
- **Mobile (D-15):** slide-in **drawer** from the left with backdrop (close to today's behavior). NOT a bottom sheet.
- **Default state (D-16):** **expanded by default** on first desktop load; persist collapse/expand via the existing persisted `sidebar-store` (`isCollapsed` / `toggleCollapsed` already present, ⌘\-ready).

### Routing (D-23 — planner owns exact paths)
- The **new Numen home becomes the default authed landing** (today authed users land at `/analyze` via `src/lib/supabase/middleware.ts`). Repoint authed landing to the new home.
- `/analyze` stays **dormant-but-reachable** (not deleted).

### Mobile-first (SHELL-07)
- Built mobile-first; desktop is the same shell widened. Sidebar drawer on mobile, persistent rail on desktop. ~760px column centers on desktop, fills width with gutters on mobile.

### Motion (THEME-05)
- **Calm/soft only** (Linear/Things restraint): sidebar collapse/drawer slide, composer reflow centered→pinned. Use existing `--duration-*` / `--ease-out-cubic`. **Respect `usePrefersReducedMotion`** (hook present). No flashy reveals (stage-reveal motion is Phase 4).

- **What signs at the gate (the `[UAT]` fields above):** exact charcoal hex ramp · exact coral hex · exact serif typeface · exact score-zone green/amber/red · greeting micro-copy.
- **Until signed:** treat `[UAT]` values as the Frame A starting point; do not propagate them to other phases as final.
