**Contrast verification:**
- `text-white/55` on `#07080a` background: white at 55% opacity = rgba(255,255,255,0.55) → effective color ~`#8c8c8c` → contrast ratio ≈ 4.6:1 against `#07080a`. Passes WCAG AA (4.5:1 minimum for normal text ≥14px). At 10px section labels (small text, WCAG AA requires 4.5:1 for large text ≥18px bold or 3:1 for normal large), borderline — acceptable given label size is visually supplementary (icon + context available).
- `text-white/85` on `#07080a`: ≈ 16.2:1. Passes WCAG AAA.
- `text-emerald-300/70` on `#07080a`: emerald-300 = `#6ee7b7` at 70% → passes AA.

### Surface 4 — Optimal Post Card (OptimalPostCard)

**Dimensions:** `170 × 88` (both AV and default states)

```
┌─────────────────────────────┐
│ ⏰ When to post              │  Clock icon + text-[11px] font-medium text-white/85
│   [Tue · 6–9 PM]  [Edit]    │  GlassPill (tappable→Sheet) + Edit link text-[10px]/55
│   Your niche peaks Tue evgs │  text-[11px] text-white/65, truncate ~70 chars
│   [from your niche]  [ⓘ]    │  GlassPill size="sm" source pill + Info icon tooltip trigger
└─────────────────────────────┘
```

- Clock icon: Phosphor `Clock` weight="regular" size={12}.
- Day+time chip: `<GlassPill onClick={openEditSheet}>{day} · {hourRangeFormatted}</GlassPill>` — `size="sm"` for compact fit.
- Edit link: `text-[10px] text-white/55 hover:text-white/80` — NOT a button, a text affordance.
- Reasoning row: `text-[11px] text-white/65`, truncate with CSS `line-clamp-1`. Full text in tooltip on hover (desktop only, `title` attribute as fallback).
- Source pill: `<GlassPill size="sm">` with text per source state (see Copywriting Contract).
- Info icon on source pill row: Phosphor `Info` weight="regular" size={10}, triggers tooltip on focus/hover.

### Surface 5 — Optimal Post Edit Sheet (OptimalPostEditSheet)

**Sheet props:** `side="right"` desktop, `side="bottom"` mobile (via `isMobile` detection same as PersonaInspector).
**Sheet width:** 320px desktop (right), full-width bottom mobile.

```
┌───────────────────────────────────────────────┐
│ ✕  Edit post time                             │  Sheet header
│                                               │
│  Day                                          │  section label text-[10px]/55
│  [Mon] [Tue] [Wed] [Thu] [Fri] [Sat] [Sun]    │  7 GlassPill pills, role="radiogroup"
│                                               │
│  Start hour           End hour                │  two <select> elements
│  [6 PM ▾]             [9 PM ▾]                │  height 42px, radius 8px, bg rgba(255,255,255,0.05)
│                                               │
│  Reset to Tue 6–9 PM ←                        │  text-[10px] text-white/55 text link
│                                               │
│  [Save for this analysis]                     │  primary button, pinned bottom
└───────────────────────────────────────────────┘
```

- Day picker: 7 `<GlassPill>` pills with `active` prop on selected day. `role="radiogroup"` wrapper, arrow-key navigation.
- Hour selects: Radix `<Select>` or native `<select>`. Hours 0-23 formatted as 12-hour AM/PM. End must be > start (inline validation, no submit error).
- Save button: secondary style `bg-transparent border-white/[0.06] hover:bg-white/[0.1]`, 8px radius, full width.
- Reset link: text-only, `text-[10px] text-white/55 hover:text-white/80`. Renders `"Reset to {original_day} {original_hours}"` where values come from `result.optimal_post_window` before override.
- Focus trap: Sheet primitive owns (Radix Dialog built-in). Esc closes. Focus returns to trigger on close via `onCloseAutoFocus` + `triggerRef` (PersonaInspector pattern).
- Reduced-motion: Sheet slide animation runs on `duration-300` from Radix CSS classes. When `prefersReducedMotion === true`, override with `className` that sets `transition: none` on SheetContent.

### Surface 6 — Source Pill Tooltip (OptimalPostSourcePill)

**Trigger:** focus or hover on `Info` icon adjacent to source pill.
**Tooltip approach:** Radix `Tooltip` (already in shadcn stack), or `title` attribute as minimal fallback.

Content per source:
- `niche` → tooltip: "Based on {N} videos in your niche." (N extracted via `/\(n=(\d+)/` from reasoning string; if extraction fails → "Based on videos in your niche.")
- `fallback` → tooltip: "Niche data unavailable yet — using the default Tue evening window. Add your niche to your creator profile for tailored timing."
- `creator` → tooltip: "Edited for this analysis. Reset to use the niche recommendation."

Tooltip dismiss: Esc key, click-away, focus-out.
Keyboard accessible: focus on Info icon → Enter/Space opens tooltip, Esc dismisses.

### Surface 7 — Loading + Error States

**Script loading (phase === 'complete', useScript fetching):**
- Skeleton = subtle pulsing opacity on section label text only: `animate-pulse` Tailwind class applied to section label spans. NOT full-card shimmer.
- ScriptBody renders section labels immediately with pulsing, body text appears when data arrives.
- Skeleton duration: typically <100ms (cache hit), up to 500ms (cache miss).

**Script error state:**
```
┌─────────────────────────────────────────────────┐
│  Couldn't generate script                       │  text-xs text-white/85
│  [Try again]                                    │  secondary button, calls refetch()
└─────────────────────────────────────────────────┘
```

Rendered by `<ScriptEmptyState variant="error" />`. Retry calls TanStack Query `refetch()`.

**Optimal post loading (result === null, skeleton):**
- Single-line dim placeholder: `<div className="h-4 w-20 bg-white/[0.06] rounded animate-pulse" />`.
- Renders inside OptimalPostCard where day/time chip would be.

**Pre-complete phase (streaming, boardState !== 'complete'):**
- Script slot: Phase 5 PlaceholderCard remains (dashed border, "Reshoot script" label) — Phase 6 does NOT render ScriptBody until `phase === 'complete'`.
- Optimal post slot: PlaceholderCard OR skeleton if `result?.optimal_post_window` is available.

## Accessibility Contract

### Per-Section Copy Buttons
- Element: `<button type="button">`
- `aria-label="Copy {SectionName} section"` (e.g., `"Copy New Opening section"`)
- Keyboard: Tab-reachable in DOM order. Enter fires copy + announces via aria-live.
- Focus indicator: `outline-2 outline-offset-2 outline-white/40`
- Touch target: min 44×44px padding even if visual icon is smaller.

### Copy-all Pill
- Element: `<button type="button">` inside GlassPill
- `aria-label="Copy full reshoot script"`
- Same keyboard behavior as per-section buttons.

### Day Picker (OptimalPostEditSheet)
- Wrapper: `<div role="radiogroup" aria-label="Day of week">` containing 7 `<GlassPill>` rendered as `<button role="radio" aria-checked={isSelected}>`.
- Arrow navigation: ArrowLeft/ArrowRight cycle through pills; Home/End jump to Mon/Sun.
- Space/Enter: selects focused pill.
- Selected state visually distinct via `GlassPill active` prop (NOT color-only — GlassPill adds border/bg change).

### Edit Sheet
- Focus trap: Radix Dialog built-in (Sheet uses Radix `Dialog.Content`).
- Esc: closes Sheet, focus returns to trigger element.
- Focus return: `onCloseAutoFocus` prop on `SheetContent` + `triggerRef` pattern (PersonaInspector.tsx line 121 precedent).
- Sheet title: `<SheetTitle>Edit post time</SheetTitle>` rendered as visually visible heading.

### Source Pill Tooltip
- Trigger: `Info` icon (`Phosphor Info`) with `tabIndex={0}` and `role="button"` or Radix `Tooltip` trigger.
- Focus → Enter/Space opens tooltip (or Radix handles this).
- Esc: dismisses tooltip.
- Screen reader: tooltip text is `aria-describedby` target.

### Aria-live Region
- Region: existing `aria-live="polite"` on `<ActionsNode>` (Phase 5 wired).
- Phase 6 writes to this region for 3 announcements (script-ready, empty-state, copy-confirmation).
- All announcements debounced 500ms to prevent spam on rapid copy clicks.
- Only fires when content actually changes (useRef guard for empty-state — fires once per analysis render).

### Color + Contrast
- All text meets WCAG AA:
  - `text-white/85` on `#07080a`: ~16:1 ratio — AAA.
  - `text-white/65` on `#07080a`: ~9:1 ratio — AAA.
  - `text-white/55` on `#07080a`: ~6.3:1 ratio — AA.
  - `text-emerald-300/70` on `#07080a`: ~5.8:1 ratio — AA.
- "Copied!" state uses `text-coral-500` (`#FF7F50`) on `#07080a`: ~4.6:1 — passes AA (≥14px text).
- Copy confirmation: NOT color-only — `CheckCircle` icon + "Copied!" text accompanies color change.
- Source pill state change: NOT color-only — pill text label changes per state ("from your niche" / "default" / "yours").

### prefers-reduced-motion
- Sheet slide animation: when `usePrefersReducedMotion() === true`, override SheetContent `className` with `transition-none`. Radix's `data-[state=open]:slide-in-from-bottom` still fires but duration is 0.
- AV grid-template-rows grow transition: inherits Phase 5 `ActionsNode.tsx:43` pattern — already guards with `prefersReducedMotion ? 'none' : 'grid-template-rows 200ms ease-out'`.
- Copy "Copied!" feedback: opacity transition `duration-150` only — acceptable under reduced-motion (brief, non-decorative). No spring animation.
- Skeleton pulse (`animate-pulse`): set to `opacity-100` (no pulse) when `prefersReducedMotion === true`.

**Critical:** Reshoot hero chrome does NOT use orange/coral styling. Verdict node already "screams" with the orange band (Phase 5 D-04). Reshoot is the CONSTRUCTIVE next step — `text-white/85` headline, `text-white/55` subhead. Subdued, actionable.

### Grid Transition
- From default → AV: `grid-template-rows` from `ACTIONS_GRID_DEFAULT_ROWS = '88px 88px'` to `ACTIONS_GRID_AV_ROWS = '160px 1fr'`
- Transition: `200ms ease-out` (per `ActionsNode.tsx:43`)
- `prefersReducedMotion` guard: already in ActionsNode

### Cross-Group Ripple
- Phase 6 subscribes to `getFrameAntiViralityState('actions', boardState)` — no new state, no new selector.
- Coordinate: Verdict orange band (Phase 5) → Audience curve highlights (Phase 4) → Reshoot hero promoted (Phase 6). Same engine signal, three distinct constructive treatments.

### AV + Empty State Conflict Resolution
- Precedence rule: **empty state body wins**. If `is_empty_state === true` AND `boardState === 'anti-virality'` simultaneously (theoretically impossible per business logic — AV implies low confidence, high confidence triggers empty state — but render must be defensive):
  - `<ScriptEmptyState />` renders in the AV hero slot space (col-span-2, 360×160)
  - AV chrome (headline "Try this instead" + subhead) is suppressed
  - Empty state check + "Your video is solid" renders centered in the grown space
  - `<ScriptEmptyState />` uses no special AV styling — same component, same copy, stretched container
