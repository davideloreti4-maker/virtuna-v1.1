## Spacing Scale

8px base, already declared in `globals.css` (`--spacing-*`). All multiples of 4. Reuse verbatim — no phase exceptions.

| Token | Value | Usage (this phase) |
|-------|-------|--------------------|
| xs | 4px | Inline chip gaps, checklist row icon-to-label gap (`gap-1`) |
| sm | 8px | Compact element spacing, chat-turn internal rhythm (`gap-2`) |
| md | 12–16px | Card face padding (`px-4`), brief padding, chat bubble padding (`--spacing-3`/`--spacing-4`) |
| lg | 24px | Inter-card / inter-turn gap in the thread column (`gap-6`), thread vertical rhythm |
| xl | 32px | Section breaks within the Test surface |
| 2xl | 48px | Major Test-surface section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none. Touch targets: interactive rows (chips, CTAs, disclosure, refine affordances) keep a ≥44px hit area via padding even when the visible glyph is small (matches the existing chip `px-2.5 py-1` + the card CTA `px-4 py-3` pattern).

## Typography

Reuse the declared `--text-*` / `--font-*` / `--leading-*` scale. Phase 5 declares exactly **2 weights**: regular 400 + semibold 600.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / chat turn / card prose | 16px (`--text-base`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) |
| Hook/idea line (card face dominant) | 16px (`--text-base`) | 600 (`--font-semibold`) | 1.25 (`--leading-snug`) |
| Label / chip / progress step / band / timestamp | 12–14px (`--text-xs` / `--text-sm`) | 600 (`--font-semibold`) | 1.25 (`--leading-snug`) |
| Test hero title ("Test") | 20–24px (`--text-xl`/`--text-2xl`) | 600 (`--font-semibold`) | 1.1–1.25 |

> `--font-medium` (500) is an **inherited THEME-06 system token, not actively varied as a Phase 5 typography choice** — it remains in the system for pre-existing chip/tag surfaces but is not enumerated as a Phase 5 weight declaration. New Phase 5 labels/chips/progress steps render at 600 semibold, keeping the phase contract at 2 weights.

Rules:
- Chat turns render through `MarkdownBlockRenderer` (`prose prose-invert prose-sm`) — body inherits the 1.5 line-height + cream-primary foreground; do not override per-turn.
- The model-authored follow-up turn (D-03) is body text, not a card — never bold-styled as a heading.
- Progress checklist labels are 12–14px semibold, muted until the stage completes (then cream-secondary + ✓).

**Primary visual anchor:** the thread column cards (`max-w-[760px] mx-auto` — idea/hook/refined cards + interleaved chat turns) are the focal point of the open thread / home surface; the composer is the secondary persistent focal element pinned at the bottom (the universal entry door). On the Test surface, the score gauge in the reframed hero is the primary anchor.

## Color

60/30/10 split mapped to the THEME-06 charcoal/cream/coral system. Cream text is **never pure white** (D-02); coral is **never glass/glow** (flat matte only).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#262624` (`--color-charcoal-app` → `--color-background`) | App/thread surface behind chat + cards |
| Secondary (30%) | `#1e1d1b` composer / `#1a1a18` sidebar / `#2f2e2b` chip (`--color-charcoal-composer`/`-sidebar`/`-chip`) | Card surfaces (`bg-transparent` + 6% border + inset hairline), composer, the Test brief panel, chip/lifted surfaces |
| Accent (10%) | `#d97757` terracotta coral (`--color-coral-500` → `--color-accent`) | See reserved-for list below |
| Score-zone data | `--color-success` green / `--color-warning` amber / `--color-error` red | SIM-1 band chips on cards (Strong/Mixed/Weak) ONLY — these are data, not chrome (THEME-03); they are NOT a destructive-action color |
| Destructive | none this phase | No destructive actions exist in Phase 5 (no delete/discard) |

**Accent (coral) reserved for — explicit list, nothing else:**
- The active tool chip (active state: `border-accent/60 bg-accent/10 text-accent`) — including the newly-live **Chat** chip (D-05).
- Chain CTAs: "Test full →", "Develop this →", and the new suggested chain-step CTA ("Turn this into hooks →") (D-05/D-09).
- The audience-archetype tag chip on hook cards (existing).
- Focus-visible rings (`ring-accent`).
- The "powered by SIM-1 Max" accent treatment on the Test hero tag (subtle — tag text/border may use coral; the hero title itself stays cream).

Coral is NOT used for: body text, card bodies, chat turns, progress checkmarks (✓ uses cream-secondary, NOT coral), the "Simulating your audience" stage label, surfaces, or borders other than the active-chip/CTA accents above.

**Cream text roles:** `--color-foreground` (cream-primary `#ece7de`) for primary text/card lines; `--color-foreground-secondary` (`#c2bdb4`) for chat body / typed text; `--color-foreground-muted` (`#8a857c`) for placeholders, labels, timestamps, progress-pending steps, the em-dash empty state.

**Borders/states (existing, reuse):** resting border `rgba(255,255,255,0.06)`, hover `rgba(255,255,255,0.1)`, card inset hairline `rgba(255,255,255,0.05) 0 1px 0 0 inset`. No glow, no glass gradient, no inset white-shine (D-05).

## Copywriting Contract

Voice = **Numen co-pilot** (D-08): concise, opinionated, creator-craft-focused, always tying back to **"your audience"**; honest about the **Flash-ceiling → Max-realized** ladder (never promises views); proactively offers the next chain step when relevant. Honesty spine: no fake progress, no untested refined output, no fabricated score.

| Element | Copy |
|---------|------|
| Primary CTA (chain endpoint) | **Test full →** (carries a hook/idea into the Test surface) |
| Secondary chain CTA | **Develop this →** (idea → hooks, existing) |
| Suggested chain-step CTA (pure-chat, D-05) | **Turn this into hooks →** (tappable suggestion only — never auto-fires) |
| Test hero title (TEST-01 / D-06) | **Test** with tag **powered by SIM-1 Max** |
| Test landing brief (above upload, D-06) | Heading: the carried hook/idea line, verbatim. Sub: **Shoot this hook → upload → SIM-1 Max scores the real thing.** |
| Progress checklist stages (STUDIO-01 / D-02 — must map to REAL pipeline phases) | **Generating… → Self-judge → Simulating your audience (SIM-1 Flash) → Ranking** (each flips to a ✓ as the real SSE stage completes; no fake timers) |
| Model follow-up turn (D-03) | Model-authored per run, references THIS run. Pattern: a one-line observation + an offered next step, e.g. *"Hook #2 leans hardest on the skeptic — want it punchier, or should I script the top one?"* (generated, not a fixed string) |
| Empty state heading (open chat, no turns yet) | **Ask anything about your content.** |
| Empty state body | **Numen grounds every answer on your niche and your audience — not a generic chatbot. Try "what should I post this week?" or send an idea to test it.** |
| Cold-start nudge (thin/no profile, D-08) | one-time soft line: **Heads up — your profile is light, so I'm leaning on {niche}/{platform} baselines. Fill in your profile for sharper, you-specific reads.** |
| Refine intent (NL, D-04) — user-typed, not our copy | recognized patterns e.g. "make hook 1 punchier", "tighten idea 2" → scoped re-run. No confirmation dialog. |
| Refined-card note (D-04) | model drops a one-line chat note after the new card, e.g. *"Re-ran #1 punchier — fresh SIM-1 score below."* (generated) |
| Error state — skill run failed | **Couldn't finish that run.** body: **The generation or SIM-1 pass dropped out. Tap to retry — nothing was charged.** |
| Error state — chat turn failed | **That answer didn't come through.** body: **Send it again, or rephrase.** |
| Error state — Test landing missing brief | (graceful) render the upload affordance with no brief; no error shown. |

**Destructive actions:** none in this phase. No delete, discard, or irreversible action is introduced — refine **adds** a new card (does not overwrite the original), chat **appends**. No confirmation dialogs required. (If the planner chooses to let a refine *replace* rather than *stack* the original card — Claude's discretion per D-04 — that is a visual stacking choice, not a data deletion; still no confirmation needed since the thread is append-only persistence.)

## Interaction Contract (Phase-specific affordances)

These are the new interaction surfaces; all render through the fixed typed-renderer library (THREAD-04) — **never model-generated UI**.

| Affordance | Behavior |
|-----------|----------|
| Progress checklist (STUDIO-01) | Transient UI driven by named SSE stage events (Claude's discretion: ephemeral SSE-driven element vs a transient typed block — D-02). Rows reveal top-down; pending = muted label + outline dot; complete = cream-secondary label + coral-free ✓. Replaced by the streamed cards on completion. `aria-live="polite"`. Calm motion only (reuse `reading-reveal` 0.42s fade-up; respect `prefers-reduced-motion`). |
| Cards-in-chat (STUDIO-02) | Skill cards append in the single open thread interleaved with markdown chat turns. Column width `max-w-[760px] mx-auto`, inter-item gap `gap-6` (matches `IdeasThreadView`). A short model-authored markdown turn (D-03) renders directly below the card group. |
| Chat-to-refine (STUDIO-02 / D-04) | Refine = scoped skill re-run → a NEW freshly-SIM-scored card inline + a one-line chat note. The refined card is visually a normal card; stacking-vs-replace is planner discretion. The new card MUST show its own fresh band chip (never reuse the prior score — moat honesty). |
| Test reframe (TEST-01 / D-06) | `reading-hero.tsx` "The read" / hero title relabeled to **Test** + **powered by SIM-1 Max** tag; entry/chip language matches. Carried hook/idea renders as a brief panel (secondary surface, 6% border) above the upload. Presentation/wiring only — no `ENGINE_VERSION` bump, video path untouched. Rename depth bounded (D-06a): hero + entry/chip + brief by default; internal nomenclature (`reading_id`, routes) unchanged. |
| Chat chip live (D-05) | Flip the existing disabled Chat chip (`tool-chips.tsx`, `enabled:false // P5`) to live; model label already maps chat → "SIM-1 Flash". Active state uses the established coral active-chip treatment. |
| No silent auto-fire (D-05) | A skill runs ONLY on an explicit chip-send or a tapped chain CTA. Pure chat may *suggest* a tappable CTA but never auto-launches a skill. |
