<objective>
**Wave 0 Smoke Test — Gemini 2.5 Flash Audio Reliability.**

Purpose: Phase 6's load-bearing assumption (D-A1 + Assumption A1 in RESEARCH) is that Gemini reliably emits voice_clarity_0_10, audio_hook_first_2s_0_10, three ratios summing to ~1.0, and a 50-150 char audio_description when the existing `gemini_video_analysis` prompt is extended with an `audio_signals` schema block. If reliability is poor (<90% emission rate or malformed fields), `signal_availability.audio = false` would fire often and the entire audio signal becomes near-useless in production.

This plan creates a standalone script the developer runs ONCE against 3 fixture videos (talking_head, slideshow, music-heavy) BEFORE writing any production audio code. The output is a human-reviewable JSON dump showing what Gemini actually emits for each video. The developer signs off (or aborts the phase) based on the results.

Output: A `scripts/smoke-test-gemini-audio.ts` script (tsx-runnable) plus a fixture-directory README. The developer drops 3 .mp4 files into `tests/fixtures/audio-smoke/` per the README, then runs the script. Pass criterion (per VALIDATION.md "Manual-Only Verifications"): ≥9/9 fields emit valid values within expected ranges across the 3 fixtures.

This plan gates Phase 6 SC#1 — if Gemini reliability is poor, Plans 02-05 should not ship as written; the planner returns to discuss-phase to revisit D-A1.
</objective>
