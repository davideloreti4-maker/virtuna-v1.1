## Known Responsive Decisions

These are intentional responsive behaviors documented in previous phases:

- **Showcase sidebar hidden on mobile:** Uses `md:block` (768px breakpoint) -- Phase 43 decision
- **App pages are desktop-only:** /app routes show "expand window" message on mobile -- by design
- **Homepage sections stack vertically on mobile:** Grid layouts collapse to single column
- **Font sizes scale down on mobile:** Headings use responsive Tailwind classes (text-3xl md:text-5xl etc.)
- **Dark theme consistent across viewports:** No light/dark switching per viewport

## Methodology

- **Tool:** Playwright with viewport override per test
- **Animations:** Disabled via injected CSS + `reducedMotion: 'reduce'`
- **Overflow detection:** `scrollWidth > clientWidth` on document element
- **Content clipping:** Bounding rect check on all main/footer children
- **Touch targets:** Flagged interactive elements smaller than 32x24px (mobile only)
- **Font readability:** Flagged text elements below 12px font-size (mobile only)
- **Screenshots:** Full-page captures (not viewport-height limited)
