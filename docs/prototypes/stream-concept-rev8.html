Design laws in play

One action zone per turn — plus per-row quick actions on hover, so the primary's default pick never traps you.
The group states the shared fact. “× vs their usual” once per strip, not once per tile. A row states only what its siblings don't.
Band color once per item. Dot + word. Never a fill, never a label, never twice.
Absence is prose. Zero results = a sentence with the basis, ghost actions only. Never padded structure, never a cream primary on nothing.
Honesty unchanged. Bands only · evidence from tool rows · “original — not drawn from a retrieved video” said plainly · provenance in the ✓ receipt.

The vocabulary — 17 primitives, closed at runtime, versioned over time

Every structured moment above is one of these — including every shape the active /dev/cards skills need (Ideas renders as ranked results like Hooks; citations are evidence rows; band and markdown are already here). Skill runners compose them deterministically; the chat agent composes them ad-hoc. Nothing outside the set can render. Hover quick-actions and the empty state are behaviors of the set, not extra primitives. The extension guarantee (rev 7): adding a primitive = one schema + one renderer case + one fixture, versioned. When no shape fits, the composer falls back to prose and logs the gap — the vocabulary grows from evidence, not speculation. Deferred with tripwires: chart (first skill needing a trend picture) · timeline (first multi-day-status skill).
