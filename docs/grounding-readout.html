Your question, answered

"So does this mean we stop relying on the reference library?"

No — you stop transplanting from it and start remixing it. The 532 videos and the retrieval that finds them are good. The failure was that the writer got a bare skeleton and no way to judge whether it fit — so it filled in the blank. Give it the full anatomy (the beats, the "when to use this", the tension, the real reasoning) and let it decide, per hook, how hard to borrow: clone a line that already fits, swap the subject, take only the angle, or drop a structure that doesn't belong. That is exactly how the remix skill already treats a video a creator hands it.

And it lives in its own adapt stage, not the writer's prompt — so the full data never competes with, and never evicts, the creator's own voice.

How this was run — and where to be skeptical

  • Same ask, same creator profile, same model in every arm. The only thing that changed was what the writer saw first. Ungrounded always ran first so a warm cache couldn't flatter the library.
  • Before each run we probed the retrieval directly: all 8 asks pulled 6 real examples across 6 distinct hook shapes — the library genuinely engaged, this isn't a dead-pipeline artifact.
  • One sample per arm per case. A narrow gap would be noise; a clean sweep plus a machine-countable copy-tell (0 vs 14 vs 12) is not. Winners above are a considered read, not a blind panel — worth your own eyes.
  • The "structure-only" arm is a one-line switch (GROUNDING_HOOKS_SURFACE=structure), off by default. Nothing is committed; all 88 grounding tests still pass; prod is untouched.
